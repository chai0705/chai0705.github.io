1.shell脚本中的特殊变量

  1. 位置参数（命令行参数）：
    • $0：当前脚本的名称。
    • $1、$2、$3、…：位置参数，表示脚本或函数接收的参数。
    • $#：位置参数的数量。
    • $*：所有位置参数的列表，作为一个单个字符串。
    • $@：所有位置参数的列表，作为多个独立的字符串。
  2. 退出状态：
    • $?：上一个命令的退出状态（返回值）。
  3. 进程信息：
    • $$：当前脚本的进程 ID。
    • $!：后台运行的最后一个进程的进程 ID。
  4. 用户和环境信息：
    • $USER：当前用户的用户名。
    • $HOME：当前用户的主目录路径。
    • $PATH：可执行文件的搜索路径。
    • $SHELL：当前正在使用的 Shell 解释器的路径。
    • $PWD：当前工作目录的路径。
    • $OLDPWD：上一个工作目录的路径。
  5. 文件名和位置信息：
    • $BASENAME：给定路径的文件名部分。
    • $DIRNAME：给定路径的目录部分。
    • $PWD：当前工作目录的路径。
    • $LINENO：正在执行的当前行的行号。
    • $FUNCNAME：当前正在执行的函数的名称。
    • $BASH_SOURCE：包含当前脚本和函数的路径的数组。
    • ${BASH_SOURCE[0]}：当前正在执行的脚本文件的路径。
  6. 时间和日期：
    • $SECONDS：脚本运行的秒数。
    • $EPOCHSECONDS：自 Unix 纪元以来的秒数。
    • $EPOCHREALTIME：自 Unix 纪元以来的实时秒数。
  7. Shell 版本和选项：
    • $BASH_VERSION：Bash 版本号。
    • $BASH_VERSINFO：包含 Bash 版本详细信息的数组。
    • $RANDOM：一个随机整数。

2.grep 命令

grep是一个非常有用的命令。grep命令用于在文本文件或标准输入中查找匹配指定模式的行，并将它们输出到标准输出。

下面是对grep命令的详细解释：

基本语法：

参数：

  • pattern：要搜索的模式或正则表达式。
  • file：要搜索的文件名。可以指定多个文件，或者使用通配符来匹配多个文件。

常用选项：

  • -i：忽略大小写，不区分大小写地进行匹配。
  • -v：反向匹配，只输出不匹配的行。
  • -r 或 --recursive：递归地搜索目录下的文件。
  • -l 或 --files-with-matches：只输出包含匹配项的文件名，而不显示匹配的行。
  • -n 或 --line-number：显示匹配行的行号。
  • -c 或 --count：只显示匹配行的总数。
  • -A num 或 --after-context=num：显示匹配行及其后续的num行内容。
  • -B num 或 --before-context=num：显示匹配行及其前面的num行内容。
  • -C num 或 --context=num：显示匹配行及其前后的num行内容。

3.awk命令

awk 是一种文本处理工具，用于逐行处理和操作结构化文本数据。它从输入文件或标准输入读取文本行，并根据预定义的规则执行操作，如查找、替换、计算、格式化等。awk 是一个强大而灵活的工具，常用于文本处理、数据提取和报告生成等任务。

以下是 awk 命令的一般语法：

复制

  • pattern 是一个模式，用于匹配输入行。当输入行与模式匹配时，将执行相应的 action。
  • action 是一系列的命令或操作，用于处理匹配的行。可以是打印、计算、替换等任意操作。
  • input_file 是要处理的输入文件名。如果不指定文件名，则从标准输入读取数据。

awk 的基本工作流程如下：

  1. 从输入文件或标准输入逐行读取文本行。
  2. 对于每一行，根据指定的模式进行匹配。
  3. 如果匹配成功，则执行相应的操作或命令。
  4. 处理完所有行后，输出最终结果。
