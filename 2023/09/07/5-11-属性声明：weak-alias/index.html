5.11 属性声明：weak & alias

2023-09-07

字数统计: 1.2k   |   阅读时长≈ 4 分钟

在 C 语言中，符号（Symbol）是指变量、函数、常量等在编译过程中生成的标识符。符号具有不同的属性，其中包括强符号（Strong Symbol）和弱符号（Weak Symbol）。

  1. 强符号（Strong Symbol）：
    • 强符号是指在编译过程中具有高优先级的符号。
    • 当存在多个强符号的定义时，链接器会选择其中一个作为最终的定义，并且会忽略其他的定义。
    • 通常情况下，全局变量和初始化的静态变量都被视为强符号。
    • 在不同的源文件中定义同名的强符号会引发链接错误。
  2. 弱符号（Weak Symbol）：
    • 弱符号是指在编译过程中具有低优先级的符号。
    • 当存在多个弱符号的定义时，链接器会选择其中一个作为最终的定义，但如果没有找到强符号的定义，链接器会接受弱符号的定义。
    • 通常情况下，未初始化的静态变量和函数都被视为弱符号。
    • 在不同的源文件中定义同名的弱符号不会引发链接错误。

根据您提供的示例代码，可以进行如下解释：

  • int a; 和 int b; 是两个未初始化的静态变量，它们被视为弱符号。
  • int a = 1; 是对变量 a 的初始化，它被视为强符号，因为它是一个具有初始值的全局变量。
  • int b = 2; 是对变量 b 的初始化，它也被视为强符号，因为它是一个具有初始值的全局变量。
  • void func() 和 -void func() 是函数 func 的声明和定义，它们都被视为强符号。

在链接过程中，如果存在多个对同一符号的定义，链接器将根据符号的强度规则来选择最终的定义。在您的示例中，如果在 main.c 和 func.c 中都包含了同名的变量或函数定义，链接器将会选择其中一个作为最终的定义，并且会忽略其他的定义（对于强符号）或接受其中一个定义（对于弱符号）。

在链接过程中，符号决议（Symbol Resolution）是指解决多个源文件中对同一符号的定义或引用的过程。根据符号的强度和规则，可以发生以下三种场景：

  1. 强符号与强符号：
    • 如果存在多个源文件中对同一符号的强符号定义，链接器将发生冲突，并且会报告链接错误。这是因为强符号具有高优先级，只能有一个最终的定义。
  2. 强符号与弱符号、弱符号与弱符号：
    • 如果存在强符号与弱符号或者多个弱符号的定义，根据一般规则，强符号和弱符号可以共存。
    • 在这种情况下，强符号将覆盖弱符号，并成为最终的定义。体积更大的符号通常被视为强符号，因此它将胜出。

需要注意的是，具体的符号决议规则和行为可能因编译器、操作系统和链接器的不同而有所差异。上述规则是一般情况下的基本规律，但在特定的环境中可能会有特殊的规则适用。

在实际开发中，为了避免符号冲突和不确定的行为，应遵循以下原则：

  • 避免在不同的源文件中定义具有相同名称的强符号。
  • 在需要共享符号的情况下，使用弱符号进行定义，并确保在链接时只有一个强符号的定义。这可以通过将符号定义放在一个源文件中，或者使用特定的链接器选项来实现。
