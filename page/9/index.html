在这个例子中，typeof(f()) 返回函数 f 的返回类型。然后，该类型被用于声明变量 k，即 k 具有与函数 f 的返回类型相同的类型。

使用示例

  1. typeof (int*);
    这段代码使用 typeof 获取 int* 的类型，但没有将其用于任何声明或赋值操作。因此，它本身并没有实际意义。
  2. typeof (int)*y;
    这段代码将 y 定义为指向 int 类型的指针。由于 typeof (int) 返回 int 类型，因此这行代码等效于 int *y;。

    这段代码将 y 定义为指针 x 指向的数据类型。 typeof(*x) 返回 x 指向的数据类型，因此这行代码等效于 int y;。

  6. typeof (typeof (char *)[4]) y;
    这段代码定义了一个名为 y 的字符指针数组，数组大小为 4。由于 typeof (char*) 返回 char* 类型，因此这行代码等效于 char *y[4];。
  7. typeof(int x[4]) y;
    这段代码定义了一个名为 y 的数组，数组元素的类型是 int，大小为 4。因此，这行代码等效于 int y[4];。

5.2.1 预定义标识符

  1. __DATE__：该标识符表示当前源文件被编译时的日期，以字符串形式表示，格式为”MMM DD YYYY”。
  2. __TIME__：该标识符表示当前源文件被编译时的时间，以字符串形式表示，格式为”HH:MM:SS”。
  3. __FILE__：该标识符表示当前源文件的文件名，以字符串形式表示。
  4. __func__：该标识符表示当前函数的名称，以字符串形式表示。该标识符在C99标准中引入。
  5. __FUNCTION__：与__func__类似，也表示当前函数的名称，但在某些编译器中使用。
  6. __LINE__：该标识符表示当前代码行的行号，以整数形式表示。
  7. include：这不是一个预定义标识符，而是一个预处理指令，用于包含头文件。
  8. define：也是一个预处理指令，用于定义宏。
  9. _STDC_：该标识符用于指示编译器是否遵循ANSI C标准。如果编译器遵循ANSI C标准，则该标识符被定义为整数常量1。
  10. printf/scanf：这些是C语言中的标准库函数，用于格式化输入和输出。

5.1 C语言的标准

C语言的标准版本主要有以下几个：

  1. K&R C：指的是《The C Programming Language》一书中描述的C语言，由Brian Kernighan和Dennis Ritchie编写。这本书是C语言开发的里程碑，但它不是正式的标准。
  2. ANSI C：也称为C89或C90，是由美国国家标准学会（ANSI）于1989年采纳的C语言标准。它是第一个被广泛接受的C语言标准，并且成为后续标准的基础。
  3. C99：它是于1999年采纳的C语言标准，也称为C89的修订版。该标准引入了一些新特性，如新的数据类型（比如布尔类型和复数类型）、变长数组、单行注释等。
  4. C11：于2011年采纳的C语言标准，它是C语言的最新标准。C11引入了一些新特性，如泛型选择表达式、多线程支持、原子操作和_Static_assert等。

5.2 C标准和编译器

不同的编译器对C标准的支持程度可能会有所差异。以下是一些常见的编译器以及它们对C标准的支持情况的概述：

GCC编译器：
GCC（GNU Compiler Collection）是一个开源的编译器套件，支持多种编程语言，包括C语言。GCC对C标准的支持相对较好，可以支持多个C标准版本，如C89、C99和C11。您可以使用命令行选项来指定所需的C标准版本，例如使用”-std=c89”来编译符合C89标准的代码。

ARM编译器：
ARM公司提供了针对ARM架构的编译器套件，其中包括ARM Compiler和GNU ARM Embedded Toolchain。这些编译器通常也支持多个C标准版本，具体取决于所使用的工具链版本和配置。

C51编译器：
C51是由Keil Software提供的针对8051系列微控制器的编译器。C51编译器主要支持ANSI C标准，即C89标准，但并不支持后续的C99或C11标准。

VC系列编译器：
VC（Visual C++）系列编译器是Microsoft Visual Studio提供的编译器套件。VC编译器对C标准的支持程度在不同版本中可能有所差异。早期版本的VC编译器主要支持C89标准，而较新的版本逐渐增加了对C99和部分C11特性的支持。
