3.Linux的优势（学习Linux操作系统最主要的原因）

1.1 Unix的诞生

时间：上个世纪60年代

地点：美国贝尔实验室

目的：开发一个多用户多任务的操作系统

结果：由于该项目过于复杂，进展十分缓慢，贝尔实验室就退出了该项目。

人物：肯.汤姆森和丹尼斯.里奇

目的：重新设计一个更简单、更实用的操作系统

结果：在1969年成功开发出了第一版的UNIX操作系统。

后续发展：而在20世纪70年代，由于半导体和集成电路技术的普及，大大拉低了内存制造成本，这使得C语言等高级语言的应用成为了可能，丹尼斯·里奇又着手B语言的改进，就这样直到今天仍被我们使用的C语言就诞生了，于是在C语言的基础上肯·汤姆森和丹尼斯·里奇又对Unix不断更新迭代，最终于1975年发布了Unix第6版本，第6版本的unix内核由9000行的C语言和700行的汇编语言所实现，让Unix系统的维护和修改变的容易，也让其可以方便的移植到不同的计算机上。

1.2 GNU和Linux的发展

时间：20世纪70年代末

事情的起因：贝尔实验室的总公司AT&T对UNIX操作系统源代码使用做出了限制，禁止将其供给学生使用，并不在开源。

人物：理查德.斯托曼（对当时UNIX操作系统的私有化和限制使用感到担忧)

行为：决定创建一个完全由免费软件按组成的类Unix计算机操作系统，

名称：GNU项目（GNU意味着”GNU’s not Unix”，强调了GNU和Unix的区别）

核心：开发一系列自由软件工具，如GNU编译器（GCC）、GNU调试器（GDB）、命令行解释器GNU bash等。

挑战：缺少操作系统内核这一核心组件。

时间：1991年

人物：林纳斯.拖瓦兹

事件：发布了一个名为Linux的操作系统内核（采用了Unix的设计理念，具有高度的可移植性和可扩展性）

最终结果：Linux内核与GNU项目相结合，为用户提供了一个完整而自由的操作系统（GNU/Linux操作系统）。

Linux可以分为内核版本和发行版本。

  • 内核版本：是操作系统的核心、负责管理硬件资源并提供进程管理、内存管理等操作系统的核心功能。
  • 发行版本：是基于GNU/Linx构建的完整操作系统，包含了内核、源代码以及相关应用程序，常见的发行版本有Ubuntu、red hat、debian等。

由于Ubuntu的图形化界面相对于初学者更加友好，所以本套课程选用ubuntu进行Linux的学习。

早期：Linux只适用于386处理器

发展：后来经过全世界的网友的帮助，使得Linux最终能够兼容多种架构，包括x86、ARM、PowerPC等。

结果：Linux服务器占据了如今整个服务器市场的高达85% – 90%的份额，而数以十亿计个人手机所运行的安卓系统使用的也是Linux内核。

1.3 Linux的优势

优势描述
开源免费Linux是开源软件，用户可以免费获取和使用，无需支付额外的费用。同时，用户可以自由地修改和定制Linux的源代码，满足个性化需求。
稳定性、可靠性和安全性由于Linux的开源特性，Linux得到了广泛的测试、审查和改进，可以长时间运行而且不容易崩溃或故障，并且具有极高的安全性
硬件兼容性Linux可以运行在多种硬件平台上，具有很好的硬件兼容性。与传统的UNIX系统相比，Linux更容易安装和运行在各种硬件设备上。
高度可定制性用户可以根据自己的需求和偏好进行自定义配置。这使得Linux适用于各种不同的应用场景，无论是服务器、嵌入式设备还是个人桌面。
应用生态系统和支持Linux拥有庞大而活跃的应用生态系统，提供了各种开源软件和工具。用户可以充分利用已有的软件资源，无需从头开发。同时，Linux社区提供了广泛的支持和文档，用户可以轻松获取帮助和解决问题。

7.5 Linux目录结构

7.1 命令的基本格式

在Linux中，命令的格式通常遵循以下结构：

command [options] [arguments]

  1. 命令（command）：表示要执行的具体命令或程序。命令可以是系统自带的内置命令，也可以是可执行程序或脚本文件。
  2. 选项（options）：表示对命令的配置，可选的。选项通常以短格式（单个字母前缀，如 -a）或长格式（完整的单词前缀，如 --all）提供。
  3. 参数（arguments）：参数通常紧跟在选项之后，用于命令的输入或操作对象。参数是命令的操作数，可以是文件、目录、文本、数字等。

注：这里的选项和参数用[]进行描述，表示可选、非必填

示例1：

ls -la /home/topeet
  • 命令：ls
  • 选项：-la
  • 参数：/home/topeet

这个命令的含义是列出/home/topeet目录下包括隐藏文件的所有文件和目录，并显示它们的详细信息。

示例2：

cd /home/topeet
  • 命令：cd
  • 参数：/home/topeet

这个命令的含义是将当前工作目录切换到/home/topeet目录。

7.2 列出文件和目录：ls命令

命令：ls

英文：list

命令功能：列出指定目录下的文件和子目录

基本格式：ls [options] [file(s) or directory]

详细解释：

  • options：选项用于控制 ls 命令的行为和输出格式。以下是常用的选项：

  • -l：以长格式（详细信息）显示文件和目录。输出包括文件的权限、所属用户和组、大小、创建日期等详细信息。

  • -a：显示所有文件和目录，包括以 . 开头的隐藏文件（默认情况下，隐藏文件不会显示）。

  • -h：以人类可读的格式显示文件大小，例如以 KB、MB、GB 等单位显示。

  • -r：以相反的顺序显示文件和目录。

  • -t：按修改时间排序，最近修改的文件或目录显示在前面。

  • file(s) or directory：可以指定一个或多个文件或目录作为参数。如果不指定参数，则默认为当前目录。

7.3 打印当前工作目录：pwd命令

命令：pwd

英文：print working directory

命令功能：显示当前工作目录的路径

  1. /：第一个斜杠代表根目录，根目录是 Linux 文件系统的最顶层目录，所有其他目录和文件都是从根目录开始的
  2. home：home 是根目录下的一个目录，用于存放用户的主目录。
  3. /：第二个斜杠代表层次结构，代表topeet 是 home 目录下的一个子目录。
  4. topeet：表示用户名。每个用户都有一个主目录，通常以其用户名命名，用于存放用户的个人文件和配置。

7.4 切换目录：cd命令

命令：cd

英文：change directory

命令功能：显示当前工作目录的路径

基本格式：cd [directory]

详细解释： directory：要切换到的目录。可以是绝对路径（从根目录开始的完整路径）或相对路径（相对于当前目录的路径）

​ 相对路径：是相对于当前工作目录或参考位置的路径。

​ ==绝对路径：是从文件系统的根目录（通常是 /）开始的完整路径。==

特殊目录：

  • .：表示当前目录。
  • ..：表示当前目录的父目录。
  • ~：表示当前用户的主目录。

Linux 目录结构

/lost+found由文件系统恢复工具保留的文件和目录
/media用于挂载可移动媒体设备
/mnt用于挂载临时文件系统
/opt用于安装可选的第三方软件
/proc包含有关系统和进程的动态信息
/root系统管理员的主目录
/run存放在系统引导过程中创建的临时文件和运行时数据
/sbin存放系统管理员使用的系统命令
/snap存放基于 Snap 包管理器安装的应用程序
/srv存放系统提供的服务数据
/swapfile交换文件，用于虚拟内存的扩展
/sys提供与内核和硬件相关的信息
/tmp存放临时文件
/usr包含系统的大部分应用程序和文件
/var包含经常变化的文件和目录，如日志文件、缓存文件、临时文件等

​ 命令：rm

​ 基本格式：rm [选项] 文件名…

命令：find

命令功能：按照指定的条件在指定的路径下递归地搜索文件，并根据条件对搜索结果进行操作

命令格式：find [路径] [表达式]

  • 路径：指定要搜索的起始路径。如果未指定路径，则默认从当前目录开始搜索。
  • 表达式：指定搜索的条件和操作。表达式可以包括各种选项和用于匹配文件属性的测试条件，以及对搜索结果执行的操作。

  • -ctime n：按文件的创建时间查找，n 表示天数。
  • -atime n：按文件的访问时间查找，n 表示天数。

11 链接命令

链接：链接是文件系统中的一个概念，它允许一个文件或目录引用另一个文件或目录，使得多个文件名指向同一个实际的文件或目录。

命令：ln

英文：link

命令功能：用于在Linux系统中创建硬链接或软链接

硬链接：硬链接与原始文件有相同的 inode 号，它们共享文件的内容和属性。因此，对于用户来说，硬链接看起来就像是原始文件的另一个名称。删除原始文件不会影响硬链接，只有当所有指向该文件的硬链接和原始文件都被删除时，文件的内容才会被释放。

软链接：是一个特殊类型的文件，它包含了指向另一个文件或目录的路径。软链接实际上是一个指向目标文件或目录的快捷方式，类似于 Windows 系统中的快捷方式。删除原始文件或目录不会影响符号链接，但如果删除了符号链接所指向的目标文件或目录，则符号链接将变为坏链接。

语法：ln [选项] <目标文件> [链接文件]

常用选项：

  • -s：创建符号链接。如果不指定该选项，则创建硬链接。

12.1 普通用户和root用户

普通用户

系统中具有有限权限和资源访问权限的用户。普通用户在登录时会获得一个独立的用户账号，该账号有一个用户名和密码。普通用户可以在自己的主目录中创建和管理文件，运行属于自己的进程，但无法修改系统级别的配置文件和进行敏感操作，如安装软件、更改系统设置等。普通用户通常使用命令行终端或图形界面登录，并使用自己的用户名和密码进行身份验证。 root用户: 指Linux 系统中具有完全访问权限和控制权的特殊用户。root 用户(也称系统管理员用户 )具有对系统的完全控制权，可以执行系统级别的操作，如修改配置文件、安装软件包、创建和删除用户账号等。

12.4 文件权限介绍

文件权限组：

  • 所有者权限：所有者权限适用于文件或目录的所有者。文件的所有者通常是创建该文件的用户。所有者权限控制着对文件的读、写和执行操作。
  • 用户组权限：组权限适用于文件或目录所属的用户组。用户组是一组用户的集合，可以在创建文件时指定。组权限控制着属于同一用户组的成员对文件的读、写和执行操作。
  • 其他用户权限：其他用户权限适用于除了文件所有者和所属用户组之外的其他用户。其他用户权限控制着非所有者和非所属用户组成员对文件的读、写和执行操作。

文件权限状态：

  • 读权限（r）：允许用户查看文件内容、目录中的文件列表和文件属性。
  • 写权限（w）：允许用户修改文件内容、创建、重命名或删除文件。
  • 执行权限（x）：对于文件，允许用户执行文件（如果是脚本或可执行程序）。对于目录，允许用户进入目录并访问其中的文件。

符号表示法： chmod 命令可以使用符号表示法来指定要设置的权限，符号表示法由三部分组成分别为要操作的权限组、操作符以及要指定的权限，具体如下所示：

命令功能：用于显示目录或文件的磁盘使用情况。它可以递归地计算目录及其子目录的大小，并提供各个目录和文件的磁盘空间使用情况。

常用的选项包括：

  • -h：以人类可读的格式显示磁盘使用情况，例如以K、M、G为单位，更易于阅读。
  • -s：仅显示总计信息，而不显示每个目录和文件的详细情况。

如果不指定目录或文件参数，默认情况下 du 命令将显示当前目录的磁盘使用情况。

14.3 磁盘分区：fdisk命令

命令：fdisk

命令功能：用于分区磁盘的命令行工具，可用于创建、修改和删除磁盘分区。

命令格式：fdisk [选项] <设备>

常用的选项包括：

  • -l：列出指定设备的分区表信息，而不进行修改。可以使用fdisk -l来查看系统中所有磁盘和分区的信息。
  • -n：创建一个新的分区。使用该选项可以进入分区创建模式，按照提示逐步创建新的分区。
  • -d：删除指定的分区。根据提示选择要删除的分区序号，然后确认删除操作。
  • -p：打印指定设备的分区表信息，包括分区的起始扇区、大小、文件系统类型等。

14.4 磁盘挂载和移除：mount 和umount 命令

命令：mount

命令功能：用于将磁盘或文件系统挂载到指定的挂载点（目录）上，使得磁盘中的数据可在该挂载点处访问

命令格式：mount [选项] <设备> <挂载点>

常用的选项包括：

  • -t <文件系统类型>：指定要挂载的文件系统类型，如ext4、ntfs、vfat等。如果未指定该选项，则mount命令会尝试自动检测文件系统类型。
  • -o <选项列表>：指定文件系统的挂载选项，如读写权限、访问控制等。选项列表以逗号分隔，常见的选项包括ro（只读）、rw（读写）、noexec（禁止执行程序）等。具体的选项取决于文件系统类型和操作系统。

命令：umount

命令功能：用于卸载（解除挂载）已挂载的磁盘和文件系统

命令格式：umount [选项] <挂载点>

16,1 什么是shell？

Shell（外壳）是Linux操作系统的一种命令行解释器，它提供了用户与操作系统内核进行交互的接口。

Shell 两种常见的工作模式：交互式模式和批处理模式。

交互式模式（Interactive Mode）：用户可以逐行输入命令，并立即查看和处理命令的输出结果。Shell 在接收到用户输入后会立即解释和执行相应的命令，并将结果输出到终端。

批处理模式（Batch Mode）：Shell 从一个脚本文件中逐行读取并执行命令，而不需要用户的实时输入和交互。用户事先编写好一系列的命令，并将它们保存在一个脚本文件中。然后，用户可以通过执行脚本文件来自动运行其中的命令。批处理模式适用于用户需要自动化执行一系列命令、进行批量处理或重复性或逻辑判断任务的情况。

16.2 更改默认的 Shell

命令：chsh

英文：Change Shell

命令功能：更改用户的默认登录 Shell（解析器）

命令格式：chsh -s [shell路径]

常用选项：

  • -s, --shell SHELL: 指定要更改为的新 Shell。可以提供 Shell 的完整路径或简单的 Shell 名称。例如，-s /bin/zsh 或 -s zsh。

16.3 Shell内置命令

在Shell中，存在两种类型的命令：内置命令（Built-in Commands）和外部命令（External Commands）。

内置命令：直接嵌入在 Shell 解释器中的命令。这意味着 Shell 解释器本身提供了这些命令的实现，无需调用外部程序。

外部命令：由独立的可执行程序提供的命令。外部命令可以是系统自带的工具，也可以是用户自己安装的软件。在执行外部命令时，Shell 会搜索系统的可执行文件路径，以找到并执行相应的命令。用户可以通过指定命令的完整路径或将可执行文件所在的目录添加到 PATH 环境变量中来执行外部命令。

区分内置命令和外部命令的一个简单方法是使用 type 命令。例如，type cd 将显示 “cd is a shell builtin”，而 type ls 将显示 “ls is /bin/ls”，表明 cd 是一个内置命令，而 ls 是一个外部命令。

18.1 变量的组成和使用

变量：用于存储和表示数据的一种命名实体，包含各种类型的数据，如字符串、数字、命令输出等

命名规范：

  • 变量必须以字母或下划线开头，并由字母、数字和下划线组成
  • 变量名对大小写敏感。
  • 建议使用大写字母来表示常量或全局变量，使用小写字母来表示局部变量。

变量的赋值：使用等号（=）将值赋给变量。需要注意的是，等号两边不能有空格，否则空格会被解释成命令和参数的一部分。例如：var=Hello, World!

变量的使用：在Shell脚本中，可以通过在变量名前加$符号来引用变量的值。例如：echo $var。

界定符： （1）单引号

。在单引号字符串中，所有字符都被视为普通字符，不进行变量扩展、命令替换或特殊字符转义。

(2）双引号

，并支持变量扩展和特殊字符转义。

18.2 变量的分类

（1）本地变量：在Shell脚本中定义和使用的变量

​ 全局变量

，不需要任何修饰符进行声明。无论是在函数内部还是函数外部，全局变量的作用范围都是整个Shell进程。全局变量的生命周期从声明语句开始，一直持续到bash进程结束。

​ 局部变量

local关键字进行修饰，只能在函数内部声明。局部变量的作用范围仅限于声明它的函数内部，在函数结束后，局部变量将不再可用。局部变量的生命周期从声明语句开始，一直持续到函数执行结束。

（2）环境变量：是操作系统自带的变量，每个bash进程都有自己的环境变量集合。当启动一个bash子进程时，子进程会从父进程继承一份相同的环境变量副本。这意味着子进程拥有与父进程相同的初始环境变量。

使用env打印当前bash进程的环境变量。

使用export命令将一个本地变量导出为环境变量

（3）特殊变量

。特殊变量提供了对脚本执行环境、命令行参数、退出状态等信息的访问。

  1. $0：表示当前脚本的名称。在脚本中使用$0可以获取脚本的名称，包括路径。
  2. $1、$2、$3…：表示脚本的位置参数或命令行参数。当脚本被调用时，可以通过$1、$2等来获取传递给脚本的参数值。$1表示第一个参数，$2表示第二个参数，以此类推。
  3. $#：表示命令行参数的个数。$#用于获取传递给脚本的参数的总数。
  4. $*：表示所有命令行参数的列表。$*将所有命令行参数作为单个字符串返回。
  5. $@：表示所有命令行参数的列表。与$*不同，$@将命令行参数作为多个独立的字符串返回。
  6. $?：表示上一个命令的退出状态。当命令执行完毕后，可以使用$?来获取其退出状态。通常，如果命令成功执行，则$?的值为0，否则为非零值。

19.2 if分支结构

​ 分支结构名称：if 分支结构

​ 适用范围：用于根据条件的真假执行不同的代码块，处理复杂的条件逻辑和数值/字符串比较。

基本语法1：

if condition
then
    # 代码块1，条件为真时执行的代码
else
    # 代码块2，条件为假时执行的代码（可选）
fi

基本语法2：

if condition1
then
    # 代码块1，条件1为真时执行的代码
elif condition2
then
    # 代码块2，条件2为真时执行的代码
else
    # 代码块3，以上条件都为假时执行的代码（可选）
fi

​

示例：

#!/bin/bash

# 编写一个成绩判断的脚本，将脚本的第一个参数作为要判断的分数
# 如果分数大于等于90则为优秀
# 如果分数大于等于80则为良好
# 如果分数大于等于60则为及格
# 如果分数小于60则为不及格
score=$1
if [ $score -ge 90 ]
then
        echo 优秀
elif [ $score -ge 80 ]
then
        echo 良好
elif [ $score -ge 60 ]
then    
        echo 及格
else 
        echo 不及格
fi

        # 匹配 模式3 时执行的代码块
        ;;

        ;;

#!/bin/bash
