==拜托一份幻觉比发现一个真理更能使人明智==

==当你在凝视深渊的时候，深渊也在凝视你==

  1. --variant=buildd: 这个参数指定了 debootstrap 的运行模式。buildd 是一种特殊的模式,它是为构建服务器设计的。这种模式会选择一些特定的软件包来构建一个基本的 Debian 系统。
  2. --components="${components[${release}]}": 这个参数指定了要安装的 Debian 软件包的组件。${release} 表示当前的 Debian 版本,${components[${release}]} 是一个包含了该版本所有组件的字符串,比如 “main contrib non-free”。
  3. --arch="${arch}": 这个参数指定了目标系统的架构,比如 “amd64”、“i386” 等。
  4. $DEBOOTSTRAP_OPTION: 这是一个环境变量,它可以包含额外的 debootstrap 选项。
  5. --foreign: 这个参数告诉 debootstrap 只下载并解压基本的系统文件,但不会配置它们。这样可以让你自定义系统的其他部分。
  6. --include="${includes}": 这个参数指定了需要额外安装的软件包,以逗号分隔的列表形式提供。
  7. "${release}": 这个参数指定了要安装的 Debian 版本,比如 “buster”、“bullseye” 等。
  8. "${target_dir}": 这个参数指定了要安装 Debian 系统的目标目录。
  9. "${mirror_addr}": 这个参数指定了 Debian 软件包的镜像地址,debootstrap 会从这个地址下载软件包。
  1. 命令名称和简介
    • 命令名称: debootstrap
    • 命令作用: 在指定的目标目录中引导安装一个基本的 Debian 系统环境。
  2. 命令语法
    • debootstrap [OPTION...] SUITE TARGET [MIRROR [SCRIPT]]
    • debootstrap [OPTION...] --second-stage
  3. 参数解释
    • SUITE: 要安装的 Debian 发行版本代号,如 “sid”、“stretch”、“jessie” 等,或者使用符号名称如 “unstable”、“testing”、“stable” 等。
    • TARGET: 要安装 Debian 系统的目标目录。
    • MIRROR: Debian 软件包源的镜像地址,可以是 http://、https://、file:///、ssh:/// 格式的 URL。
    • SCRIPT: 可选的,自定义的启动脚本。
  4. 选项解释
    • --arch=ARCH: 设置目标体系结构,在dpkg未安装时可使用。
    • --include=alpha,beta: 添加要下载和提取的软件包列表。
    • --exclude=alpha,beta: 从下载和提取列表中删除指定的软件包。需要小心使用,可能会排除关键软件包。
    • --extra-suites=alpha,beta: 使用存档的其他suites中的软件包。
    • --components=alpha,beta: 使用存档的指定components中的软件包。
    • --no-resolve-deps: 禁用debootstrap自动解决依赖的行为。不建议使用,最好指定完整的基础系统。
    • --log-extra-deps: 在解析依赖关系时,记录额外的依赖项到debootstrap.log。
    • --variant=minbase|buildd|fakechroot: 选择不同的bootstrap脚本变体。
    • --merged-usr 和 --no-merged-usr: 控制是否创建 /{bin,sbin,lib}/ 到 /usr/ 的符号链接。
    • --keyring=KEYRING: 设置用于检查获取的Release文件签名的密钥环。
    • --no-check-gpg: 禁用检查获取的Release文件的GPG签名。
    • --force-check-gpg: 强制检查Release文件签名,禁用自动回退到HTTPS。
    • --verbose: 产生更多的下载信息。
    • --print-debs: 显示要安装的软件包并退出。
    • --download-only: 只下载软件包,不执行安装。
    • --foreign: 只执行引导的初始解压缩阶段。
    • --second-stage: 完成引导过程的第二阶段。
    • --second-stage-target=DIR: 在子目录中运行第二阶段。
    • --keep-debootstrap-dir: 不删除目标中的/debootstrap目录。
    • --cache-dir=DIR: 使用指定目录缓存 .deb 文件。
    • --unpack-tarball=FILE: 从gzipped tarball文件中获取 .debs,而不是通过HTTP(S)下载。
    • --make-tarball=FILE: 制作下载的软件包的gzipped tarball,而不是执行引导过程。
    • --debian-installer: 供内部使用的debian-installer选项。
    • --extractor=TYPE: 覆盖自动选择的 .deb 提取器类型。
    • --no-check-certificate: 不检查证书的有效性。
    • --certificate=FILE: 使用指定的客户端证书文件。
    • --private-key=FILE: 使用指定的私钥文件。
    • --inrelease-path: 确定suite的InRelease文件的相对路径。
  5. 用途
    • 在没有安装盘的情况下引导安装 Debian 系统
    • 在 chroot 环境中安装不同版本的 Debian 系统,用于测试等场景
    • 创建基于 Debian 的软件包构建环境,与 pbuilder 工具配合使用
    • 执行交叉编译安装 Debian 系统
    • 定制化安装 Debian 系统,包括安装额外软件包、排除某些软件包等
