一、整体架构（先确认我的理解）                                                                             
                                                                                                             
  这是 RK3588 上的多路 AI NVR，三个可执行实体共用一个核心 nvr::run()：                                       
                                                                                                             
  - nvr_core（库）：每路视频一个 stream_worker 线程（解码→clone→入队），npu_workers 个共享 npu_worker        
  线程从有界队列取帧推理（letterbox→BGR2RGB→rknn_run→解码→NMS），主循环每 50ms 拼宫格 + 生成 status          
  JSON。                                                                                                     
  - ai_nvr（无界面）：把宫格/状态写磁盘（mosaic.jpg / status.json），配 Python web/server.py 读盘提供        
  Web。                                                                                                      
  - nvr_qt_console（Qt 主控台，被脚本标为首选）：核心进程内跑，用 LiveFeed 内存交接（已去磁盘往返），内嵌    
   C++ HTTP 服务供局域网访问。

  1. 先插桩测量，再动手。在 npu_worker 里给每个阶段记时间：decode / letterbox / cvtColor / rknn_run /        
  outputs_get / decode+NMS，按 worker 累计平均值打到日志。没有这组数你就是盲调。                             
  2. 定位真瓶颈：用现成的 NPU 占用表 + CPU% 判断 9 路时到底卡在哪——大概率是软解码吃满 CPU而 NPU              
  没喂饱。如果 NPU 利用率低、CPU 满，那再多 NPU worker 也没用，杠杆在解码侧（MPP）。                         
  3. 按杠杆大小排序，而不是按好不好写：                                                                      
    - 大杠杆（推迟）：MPP 硬解 → RGA 预处理。                                                                
    - 中杠杆（可做）：输入零拷贝、去冗余拷贝（第三节 A 的 1/2/3）、对齐建图节奏、ai_nvr 内嵌 Web 去磁盘。    
    - 模型侧：YOLOv8n、更小输入尺寸（已是 int8）。                                                           
    - 调度侧（step.md 的 V3）：非重点通道降频、告警通道升频、按画面变化动态调推理频率。                      
  4. 每次只改一处、用第 1 步的数对比，确认收益再进行下一处。

      - 对客户很有用，因为很多问题其实是网络没配好。

      - 测 CPU 多核、内存带宽、磁盘读写、温度变化。

        CPU 多核: 正常
        eMMC 写入速度: 145 MB/s
        当前温度: 48°C
        是否降频: 否

      - 检查 RKNN/NPU 驱动、设备节点、运行库是否存在。
      - 能跑一个简单 demo，确认 NPU 可用。
      - 对 AI 客户价值很高。

  5. 一键摄像头检测工具
      - 自动列出 USB 摄像头、MIPI 摄像头、/dev/video*。
      - 测试能不能取流、分辨率、帧率。
      - 可以生成中文结果，减少客户问“摄像头为什么没画面”。

  6. 一键显示屏/HDMI 检测工具
      - 检查 HDMI、MIPI 屏、分辨率、当前桌面环境。

      - 对工控/广告屏/触摸屏客户很实用。

  7. 一键串口/USB/外设检测工具
      - 列出串口、USB 设备、GPIO/I2C/SPI 基础状态。

      - 自动收集 dmesg、系统日志、网络状态、驱动状态、温度、磁盘状态。
