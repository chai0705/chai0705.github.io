# 发明名称



**一种基于GPIO电气特征分析的嵌入式系统板级自动识别与配置方法**



## 一、 技术领域



本发明涉及嵌入式系统软硬件协同设计领域，具体涉及一种利用普通GPIO引脚的电气充放电特征来识别硬件版本，并在Bootloader阶段动态适配设备树（Device Tree）的方法。



## 二、 背景技术



在嵌入式产品的全生命周期中，供应链的不确定性常导致硬件设计的微小变更。例如：

1. **元器件替换**：由于缺货，不同批次的PCBA可能混用不同厂商的DDR内存颗粒、Flash芯片或不同型号的传感器（Sensor）。
2. **衍生版本**：同一块核心板（SOM）可能搭配不同的底板（Baseboard），或者同一底板通过不同的电阻配置区分“标准版”与“高配版”。

**现有技术的缺点：**

- **ADC电阻ID法**：需要占用宝贵的ADC专用引脚，且需要高精度的分压电阻，增加BOM成本。
- **GPIO编码法**：若要区分4种状态，需要占用2个GPIO引脚，资源浪费严重。
- **EEPROM存储法**：需要额外的存储芯片记录版本号，增加了硬件成本和烧录工序。
- **多固件维护**：针对每种硬件版本维护不同的U-Boot或内核镜像，生产管理混乱，容易烧错固件。

因此，需要一种无需额外增加专用芯片、不占用ADC资源、仅利用现有GPIO即可实现硬件精准识别，并能自动适配软件配置的方法。



## 三、 发明内容（核心技术方案）





### 1. 发明目的



本发明旨在提供一种通过软件算法测量普通GPIO引脚的模拟电气特征（充放电时间/阻抗），从而识别硬件版本，并在系统启动早期自动修补设备树（FDT）的方法，实现“一套固件兼容多种硬件”。



### 2. 技术方案



本发明包含硬件特征构建与软件识别适配两个部分：



#### (1) 硬件特征构建原理



利用GPIO引脚的寄生电容或外围电路固有的**RC充放电特性**。

- **差异源**：不同的外设芯片（如不同型号的Sensor），其引脚的输入电容（Input Capacitance）和漏电流不同；或者在PCB设计时，在特定GPIO（如LED控制脚或空闲脚）上贴装不同阻值的上拉/下拉电阻（R）或不同容值的电容（C）。
- **特征量**：当GPIO从输出模式切换为高阻抗输入模式时，电平不会瞬间跳变，而是呈现指数衰减或上升。该过程的耗时 $\Delta t$ 与 $R \times C$ 成正比。



#### (2) 软件执行流程（运行于U-Boot或Pre-loader阶段）



1. **放电/充电阶段**：
   - 将目标GPIO配置为**输出模式**。
   - 输出特定电平（例如高电平 Logic 1），并保持微秒级时间，使线路电容充满电。
2. **高阻测量阶段**：
   - 将该GPIO迅速切换为**输入模式（高阻态）**。
   - 此时，电荷通过外部电阻或漏电流路径泄放。
   - 软件立即启动高精度定时器（Timer）或循环计数器。
3. **特征捕获阶段**：
   - 轮询GPIO的输入电平状态寄存器。
   - 当读取到的电平由“1”翻转为“0”时，停止计时。
   - 记录此时的时间计数值 $T_{decay}$（衰减时间）。
4. **指纹匹配阶段**：
   - 将 $T_{decay}$ 与预置的“特征阈值表”进行比对。
   - 例如：$0 < T < 10us$ 判定为版本A（无外挂电容）；$50us < T < 100us$ 判定为版本B（挂载10nF电容）。
5. **动态加载阶段**：
   - 根据判定结果，Bootloader在内存中定位到设备树（FDT/DTB）的对应节点。
   - 执行设备树修补操作（Device Tree Fixup）：例如修改 `status = "okay"` 或调整 `compatible` 属性，或者加载对应的 `.dtbo` 叠加包。
   - 启动Linux内核。



### 3. 有益效果



- **零成本**：不需要ADC，不需要EEPROM，仅利用现有的电阻/电容或器件寄生参数。
- **高兼容性**：一套镜像文件即可自动适配生产线上的混料情况。
- **资源节约**：复用现有的功能引脚（如复位脚、中断脚）在启动瞬间进行检测，不占用系统运行时的引脚资源。



## 四、 具体实施方式（可以作为专利说明书的详细描述）





### 实施例：兼容两种不同型号的触摸屏（IC-A 与 IC-B）



假设嵌入式设备可能贴装两种触摸屏IC，它们连接在同一个I2C总线上，复位引脚（RST）连接到SoC的 `GPIO_X`。

**步骤 1：硬件差异化**

- **方案A（利用器件差异）**：IC-A的RST引脚内部集成$50k\Omega$下拉电阻，IC-B的RST引脚为高阻输入。
- **方案B（利用PCB差异）**：针对IC-A的PCB版本，在`GPIO_X`处不贴电容；针对IC-B的PCB版本，在`GPIO_X`处并联一颗100nF电容。

**步骤 2：软件探测逻辑（U-Boot C代码逻辑）**

C

```
// 伪代码流程
int identify_hardware_version(void) {
    int gpio_pin = GPIO_X;
    unsigned long start_time, end_time, delta_us;

    // 1. 充电阶段：输出高电平，给电容充电
    gpio_direction_output(gpio_pin, 1);
    udelay(100); // 确保电荷充满

    // 2. 切换阶段：配置为输入，关闭内部上下拉
    // 此时外部电路开始通过电阻放电
    gpio_direction_input(gpio_pin); 
    gpio_set_pull(gpio_pin, PULL_DISABLE); 

    // 3. 测量阶段：轮询直到电平变低
    start_time = get_timer_us();
    while (gpio_get_value(gpio_pin) == 1) {
        // 设置超时防止死循环
        if (get_timer_us() - start_time > TIMEOUT_US) break;
    }
    end_time = get_timer_us();
    
    delta_us = end_time - start_time;

    // 4. 判定阶段
    if (delta_us < 10) {
        return VERSION_A; // 放电极快，认为是内部有下拉电阻的IC-A
    } else if (delta_us > 50) {
        return VERSION_B; // 放电慢，认为是高阻输入的IC-B
    }
    return VERSION_UNKNOWN;
}
```

```
// U-Boot ft_board_setup 函数中
int ft_board_setup(void *blob, bd_t *bd) {
    int hardware_id = identify_hardware_version();
    
    if (hardware_id == VERSION_A) {
        // 启用 IC-A 节点，禁用 IC-B
        do_fixup_by_path(blob, "/i2c@1000/touchscreen_a", "status", "okay", 5, 1);
        do_fixup_by_path(blob, "/i2c@1000/touchscreen_b", "status", "disabled", 9, 1);
        printf("Detected Hardware: Touchscreen IC-A\n");
    } else {
        // 启用 IC-B，禁用 IC-A
        do_fixup_by_path(blob, "/i2c@1000/touchscreen_a", "status", "disabled", 9, 1);
        do_fixup_by_path(blob, "/i2c@1000/touchscreen_b", "status", "okay", 5, 1);
        printf("Detected Hardware: Touchscreen IC-B\n");
    }
    return 0;
}
```

## 五、 权利要求书（核心保护点，帮你提炼好了）



1. 一种基于IO电气特征分析的板级自动识别与配置方法，其特征在于，包括以下步骤：
   - **步骤S1：** 在嵌入式系统启动阶段，控制目标GPIO引脚输出第一电平，对该引脚连接的线路负载进行充电或放电操作；
   - **步骤S2：** 将所述目标GPIO引脚切换为高阻抗输入模式，并关闭芯片内部的上下拉电阻；
   - **步骤S3：** 实时监测所述目标GPIO引脚的电平状态，并利用计时器测量该引脚电平从第一电平翻转至第二电平所经历的衰减时长；
   - **步骤S4：** 将所述衰减时长与预设的特征阈值区间进行比对，确定当前硬件的板级版本标识；
   - **步骤S5：** 根据所述板级版本标识，在内存中动态修改设备树（Device Tree）文件或加载对应的设备树叠加包（Overlay），随后加载操作系统内核。
2. 根据权利要求1所述的方法，其特征在于，所述目标GPIO引脚为复用引脚，在完成步骤S3的测量后，将该引脚恢复为默认的功能模式（如I2C、SPI或复位功能），不影响系统启动后的正常业务功能。
3. 根据权利要求1所述的方法，其特征在于，所述硬件差异的构建方式包括：在不同版本的PCB上，于所述目标GPIO引脚处贴装不同容值的电容器，或者利用不同型号外设芯片引脚的输入阻抗/电容差异。

~~~c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/i2c.h>
#include <linux/gpio.h>
#include <linux/gpio/consumer.h>
#include <linux/interrupt.h>
#include <linux/delay.h>

//定义ft5x06设备的GPIO描述符
//用于保存reset和interrupt引脚的GPIO描述符
struct gpio_desc *reset_gpio, *irq_gpio;

//保存ft5x06设备的i2c客户端对象指针
struct i2c_client *ft5x06_client;

//读取ft5x06设备寄存器的函数
int ft5x06_read_reg(u8 reg_addr)
{
    u8 data;
    //定义两个i2c_msg结构体,分别表示写操作和读操作的数据包
    struct i2c_msg msgs[2] = {
        [0] = {
            .addr = ft5x06_client->addr, //设备地址
            .flags = 0,        //写操作
            .len = sizeof(reg_addr),
            .buf = &reg_addr,  //写入要读取的寄存器地址
        },
        [1] = {
            .addr = ft5x06_client->addr,
            .flags = i2c_M_RD, //读操作
            .len = sizeof(data),
            .buf = &data,      //读取到的数据存储位置
        },
    };

   //使用i2c_transfer函数进行i2c总线读取操作
   //如果读取失败，返回-EIO错误码
   if (i2c_transfer(ft5x06_client->adapter, msgs, ARRAY_SIZE(msgs)) != ARRAY_SIZE(msgs)){
        return -EIO;
    }

    return data; //返回读取到的寄存器值
}

//向ft5x06设备写入寄存器的函数
void ft5x06_write_reg(u8 reg_addr, u8 *data, u16 len)
{
    u8 buff[256];
//构建写操作的数据包
    struct i2c_msg msgs[] = {
        [0] = {
            .addr = ft5x06_client->addr, //设备地址
            .flags = 0,        //写操作
            .len = len + 1,    //数据长度 + 寄存器地址长度
            .buf = buff,       //数据缓冲区
        },
    };

    buff[0] = reg_addr;       //写入寄存器地址
    memcpy(&buff[1], data, len); //写入数据

    //使用i2c_transfer()函数进行i2c总线写入操作
    //如果写入失败，直接返回
    if (i2c_transfer(ft5x06_client->adapter, msgs, ARRAY_SIZE(msgs)) != ARRAY_SIZE(msgs)) {
        return;
    }
}

// ft5x06中断处理函数
irqreturn_t ft5x06_handler(int irq, void *args)
{
    printk("This is ft5x06 handler\n");
    //返回中断已处理标志
    return IRQ_RETVAL(IRQ_HANDLED);
}

// ft5x06设备的探测函数
int ft5x06_probe(struct i2c_client *client, const struct i2c_device_id *id)
{
    int ret,value;
    printk("This is ft5x06 probe\n");
    ft5x06_client = client; //保存i2c客户端对象指针

    //获取reset GPIO描述符
    reset_gpio = gpiod_get_optional(&client->dev, "reset", 0);
    if (!reset_gpio) {
        printk("gpiod_get_optional reset gpio is error\n");
        return -1;
    }

    //获取irq GPIO描述符
    irq_gpio = gpiod_get_optional(&client->dev, "interrupts", 0);
    if (!irq_gpio) {
        printk("gpiod_get_optional irq gpio is error\n");
        return -1;
    }

    //设置reset GPIO为输出，并拉低5ms后拉高
    gpiod_direction_output(reset_gpio, 0);
    msleep(5);
    gpiod_direction_output(reset_gpio, 1);

    //写寄存器
    ft5x06_write_reg(0x80, &(u8){0x4b}, 1);

    //读寄存器
    value = ft5x06_read_reg(0x80);
    printk("reg 0x80 is %x\n", value);

    return 0;
}

// ft5x06设备的移除函数
//参数client是i2c客户端对象指针
int ft5x06_remove(struct i2c_client *client)
{
    //释放中断
    free_irq(client->irq, NULL);
    //释放GPIO资源
    gpiod_put(reset_gpio);
    gpiod_put(irq_gpio);
    return 0;
}
~~~

~~~c
#include <linux/module.h>
#include <linux/init.h>
#include <linux/input.h>
#include <linux/gpio.h>
#include <linux/interrupt.h>
#include <linux/platform_device.h>

#define GPIO_PIN        70          // 按键GPIO引脚
#define KEY_CODE        KEY_1       // 要上报的按键键值
#define DEVICE_NAME     "key_gpio70"// 设备名称
#define DRIVER_NAME     "key_gpio70_driver"// 驱动名称

struct key_dev {
    struct input_dev *input_dev;    // 输入设备结构体
    int irq_num;                    // 中断号
};

static struct key_dev key_device;   // 设备私有数据

// 中断处理函数：按键按下时上报事件
static irqreturn_t gpio_key_irq_handler(int irq, void *dev_id)
{
    struct key_dev *dev = (struct key_dev *)dev_id;
    
    // 上报按键按下事件（value=1表示按下）
    input_event(dev->input_dev, EV_KEY, KEY_CODE, 1);
    // 上报同步事件，通知内核事件完整
    input_sync(dev->input_dev);
    
    // 延时短暂时间后上报按键释放事件（可选，根据需求决定是否需要）
    // 若只需要按下事件，可删除以下3行
    input_event(dev->input_dev, EV_KEY, KEY_CODE, 0);
    input_sync(dev->input_dev);
    
    printk(KERN_INFO "GPIO%d key pressed, reported KEY_1 event\n", GPIO_PIN);
    return IRQ_HANDLED;
}

static int __init gpio_key_init(void)
{
    int ret;
    struct key_dev *dev = &key_device;
    
    printk(KERN_INFO "Initializing GPIO%d key driver\n", GPIO_PIN);
    
    // 1. 申请GPIO引脚
    if (!gpio_is_valid(GPIO_PIN)) {
        printk(KERN_ERR "GPIO%d is not a valid GPIO pin\n", GPIO_PIN);
        return -EINVAL;
    }
    
    ret = gpio_request(GPIO_PIN, DEVICE_NAME);
    if (ret < 0) {
        printk(KERN_ERR "Failed to request GPIO%d\n", GPIO_PIN);
        return ret;
    }
    
    // 2. 配置GPIO为输入模式
    gpio_direction_input(GPIO_PIN);
    
    // 3. 将GPIO映射到中断号
    dev->irq_num = gpio_to_irq(GPIO_PIN);
    if (dev->irq_num < 0) {
        printk(KERN_ERR "Failed to map GPIO%d to IRQ\n", GPIO_PIN);
        ret = dev->irq_num;
        goto err_gpio_free;
    }
    printk(KERN_INFO "GPIO%d mapped to IRQ%d\n", GPIO_PIN, dev->irq_num);
    
    // 4. 分配输入设备
    dev->input_dev = input_allocate_device();
    if (!dev->input_dev) {
        printk(KERN_ERR "Failed to allocate input device\n");
        ret = -ENOMEM;
        goto err_gpio_free;
    }
    
    // 5. 配置输入设备
    dev->input_dev->name = DEVICE_NAME;
    dev->input_dev->phys = "key_gpio70/input0";
    dev->input_dev->id.bustype = BUS_HOST;
    dev->input_dev->id.vendor = 0x0001;
    dev->input_dev->id.product = 0x0001;
    dev->input_dev->id.version = 0x0100;
    
    // 设置支持的事件类型和按键
    set_bit(EV_KEY, dev->input_dev->evbit);  // 支持按键事件
    set_bit(EV_SYN, dev->input_dev->evbit);  // 支持同步事件
    set_bit(KEY_CODE, dev->input_dev->keybit);  // 支持KEY_1按键
    
    // 6. 注册输入设备
    ret = input_register_device(dev->input_dev);
    if (ret < 0) {
        printk(KERN_ERR "Failed to register input device\n");
        goto err_input_free;
    }
    
    // 7. 请求中断
    ret = request_irq(dev->irq_num,
                     gpio_key_irq_handler,
                     IRQF_TRIGGER_RISING | IRQF_ONESHOT | IRQF_NO_SUSPEND,
                     DRIVER_NAME,
                     dev);
    if (ret < 0) {
        printk(KERN_ERR "Failed to request IRQ%d\n", dev->irq_num);
        goto err_input_unregister;
    }
    
    printk(KERN_INFO "GPIO%d key driver initialized successfully\n", GPIO_PIN);
    return 0;
    
err_input_unregister:
    input_unregister_device(dev->input_dev);
err_input_free:
    input_free_device(dev->input_dev);
err_gpio_free:
    gpio_free(GPIO_PIN);
    return ret;
}

static void __exit gpio_key_exit(void)
{
    struct key_dev *dev = &key_device;
    
    printk(KERN_INFO "Exiting GPIO%d key driver\n", GPIO_PIN);
    
    // 释放中断
    free_irq(dev->irq_num, dev);
    // 注销输入设备
    input_unregister_device(dev->input_dev);
    // 释放输入设备资源
    input_free_device(dev->input_dev);
    // 释放GPIO引脚
    gpio_free(GPIO_PIN);
    
    printk(KERN_INFO "GPIO%d key driver exited successfully\n", GPIO_PIN);
}

module_init(gpio_key_init);
module_exit(gpio_key_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("topeet");
MODULE_DESCRIPTION("GPIO70 Key Event Driver");
~~~
