### 第一步：配置 GNSS 参数 (`AT+QGPSCFG`)

这一步主要是设置 GNSS 的工作模式（如支持 GPS、北斗、格洛纳斯等混合定位）。

**打开 AT 指令终端**（假设你的 AT 端口是 `/dev/ttyUSB2`）：

```
# 使用 microcom 进入交互模式，退出按 Ctrl+X
microcom -s 115200 /dev/ttyUSB2
```

**发送配置指令**（在终端内输入）：

**配置支持的卫星系统：** 建议开启 GPS + 北斗。

```
AT+QGPSCFG="gnssconfig",1
```

配置 NMEA 语句输出端口：** 确保输出到 USB 端口。

```
AT+QGPSCFG="outport","usbnmea"
```

------

### 第二步：打开 GNSS (`AT+QGPS`)

官方文档提到这是启动定位的核心步骤。

**启动定位：**

```
AT+QGPS=1
```

*注：执行后如果返回 `OK`，说明模块已开始搜星。如果返回 `ERROR` 且错误代码是 504，说明 GNSS 已经是在开启状态了。*

------

### 第三步：获取定位信息

你有三种方式可以获取数据，在 RK3588 的 Linux 系统下，建议如下操作：

#### 方式 1：通过 NMEA 端口读取（最常用，适合导航软件）

这是最推荐的方式。在 Linux 下，`usbnmea` 端口通常对应 `/dev/ttyUSB1`。

**操作：** 另开一个 shell 窗口，执行：

```
cat /dev/ttyUSB1
```

你会看到类似 `$GPRMC...` 的原始数据流。

#### 方式 2：通过 AT 指令主动查询（适合简单获取经纬度）

如果你只需要偶尔获取一次坐标，不需要解析复杂的数据流：

- **操作：** 在 AT 端口（`/dev/ttyUSB2`）发送：

  ```
  AT+QGPSLOC?
  ```

  **返回示例：** `+QGPSLOC: 084852.0,3114.5367N,12121.3175E,1.0,44.2,2,0.0,0.0,0.0,230323,06`（分别代表时间、纬度、经度、高度等）。

#### 方式 3：获取指定类型的 NMEA 语句

如果你只需要某一种特定的报文（比如只想要 `$GPGSV`）：

1. **启用功能：** `AT+QGPSCFG="nmeasrc",1`
2. **获取数据：** `AT+QGPSGNMEA="GSV"`
