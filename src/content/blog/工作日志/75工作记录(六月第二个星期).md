# 工作记录(星期二)



# 在 RK3588 开发板上安装 Claude Code（API 登录，跳过 login）

## 环境说明

- **登录方式**：通过 `~/.claude/settings.json` 配置 `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN` 直连中转网址，跳过官方 OAuth 登录
- **中转网址**：`http://192.168.1.199:41016`（局域网地址，RK3588 必须和它在同一内网才能访问）
- **Node 版本**：v24.16.0（npm 11.13.0）
- **架构差异**：当前电脑是 x86_64，RK3588 是 aarch64（ARM64），不能直接拷贝二进制，要在板子上重新装

---

## 第 0 步：在当前电脑上拿到 token

在**当前这台 x86 电脑**上执行，把输出记下来（一会儿要填到板子上）：

```bash
echo $ANTHROPIC_AUTH_TOKEN
```

---

## 第 1 步：RK3588 上安装 Node.js（用 nvm）

SSH 登录到 RK3588 后执行：

```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# 让 nvm 生效（或者重新打开终端）
source ~/.bashrc

# 安装和当前环境一致的 Node 版本
nvm install 24
nvm use 24
nvm alias default 24

# 验证
node -v
npm -v
uname -m   # 应输出 aarch64
```

> 如果板子访问 GitHub 慢，nvm 那行可换成 gitee 镜像，或直接用系统包管理器装 Node 20+。Claude Code 要求 Node ≥ 18。

---

## 第 2 步：安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

装完验证：

```bash
claude --version
which claude
```

---

## 第 3 步：配置 API（跳过登录的关键）

把配置写进 Claude Code 自己的配置文件 `~/.claude/settings.json`，这样配置只对 Claude Code 生效，不污染全局环境。

先创建目录（如果还没有）：

```bash
mkdir -p ~/.claude
```

然后编辑 `~/.claude/settings.json`（用 nano、vim 都行）：

```bash
nano ~/.claude/settings.json
```

填入以下内容。`ANTHROPIC_AUTH_TOKEN` 改成第 0 步拿到的真实值：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "<API_KEY_REDACTED>", 
    "ANTHROPIC_BASE_URL": "http://192.168.1.199:41016",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-opus-4-8",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL_NAME": "claude-opus-4-8",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-opus-4-8[1M]",
    "ANTHROPIC_DEFAULT_OPUS_MODEL_NAME": "claude-opus-4-8",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-opus-4-8[1M]",
    "ANTHROPIC_DEFAULT_SONNET_MODEL_NAME": "claude-opus-4-8",
    "ANTHROPIC_MODEL": "claude-opus-4-8"
  },
  "skipDangerousModePermissionPrompt": true
}
```

> 注意：
>
> - 必须是合法的 JSON——逗号、引号别漏，最后一项后面不要多加逗号。
> - 这是文件 `~/.claude/settings.json`，不是把 `export` 命令丢进 `.claude` 目录。`.claude` 目录不会自动执行 shell 脚本。
> - `settings.json` 方式和 `.bashrc` 的 `export` 方式**二选一**，不要同时配，避免混淆。

`settings.json` 的配置在每次 `claude` 启动时自动加载，不需要 `source`。

---

## 第 4 步：验证网络能通到中转服务

`192.168.1.199` 是内网地址，先在板子上测能不能连上：

```bash
curl -v http://192.168.1.199:41016
```

- 能返回内容（哪怕是 404/401）→ 网络通，继续
- `Connection refused` / 超时 → RK3588 和中转服务不在同一内网，或防火墙挡了。需确保板子和 `192.168.1.199` 在同一路由器/网段下

---

## 第 5 步：启动

```bash
cd ~   # 进到任意工作目录
claude
```

配置正确的话会直接进入对话界面，不会要求登录。

---

## 常见问题

| 现象                        | 原因 / 解决                                                  |
| --------------------------- | ------------------------------------------------------------ |
| 启动后还是让我登录          | `settings.json` 没生效，检查 JSON 格式是否正确（用 `cat ~/.claude/settings.json` 看一下），token 是否填了 |
| `Connection refused`        | 板子连不上 `192.168.1.199`，检查是否同一内网                 |
| `command not found: claude` | nvm 的 PATH 没加载，重开终端或 `source ~/.bashrc`            |
| 模型报错                    | 确认 4 个模型变量都设了，且中转服务支持 `claude-opus-4-8`    |
