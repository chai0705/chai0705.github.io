# 工作记录(星期二)

==摆脱一份幻觉比发现一个真理更能使人明智==

==当你在凝视深渊的时候，深渊也在凝视你==

==弱小和无知并不是生存的障碍，傲慢才是==

没错，就是这里！你已经找对地方了。

现在的关键是把这个选项填好，让内核在**编译阶段**就把固件“吞”进去。这样开机时驱动就不再需要去文件系统里找文件，自然也就不会再报那个烦人的 `-2` 错误了。

请按照以下步骤操作：

### 1. 开启固件集成选项

在你的 `menuconfig` 界面中：

1. 找到 **`() Build named firmware blobs into the kernel binary`**，按回车进入。

2. 输入固件的**相对路径**（多个文件用空格隔开）：

   `rtl_bt/rtl8723d_fw.bin rtl_bt/rtl8723d_config.bin`

3. 完成后，你会发现下方多出了一个选项：**`( ) Firmware blobs root directory`**。

4. 在这里输入你固件存放的**绝对路径**（指的是你编译内核的那台电脑上的路径）。

   - 例如，如果你把固件放在了内核源码根目录下的 `firmware` 文件夹里，就填：`firmware` 或 `./firmware`。

------

### 2. 准备固件文件（最重要的一步）

内核在编译时，会去你指定的 `root directory` 下寻找文件。

- **操作方法**：
  1. 在你的内核源码根目录下创建一个文件夹：`mkdir -p firmware/rtl_bt`
  2. 把那两个固件文件（`rtl8723d_fw.bin` 和 `rtl8723d_config.bin`）拷贝到这个 `firmware/rtl_bt/` 目录中。
  3. 确保文件名和你在 `menuconfig` 里填写的完全一致。

------

### 3. 为什么这样做能解决问题？

当你这样配置后，内核在链接生成 `zImage` 或 `Image` 时，会把这两个二进制文件转换成数组直接放进内核镜像里。

- **加载流程变化**：
  - **之前**：内核启动 -> 驱动加载 -> 向系统要文件 -> 系统还没挂载好 -> **报错失败**。
  - **现在**：内核启动 -> 驱动加载 -> 驱动直接从内存（内核镜像内部）读取固件 -> **成功**。

------

### 4. 编译并验证

重新编译内核（`make`），然后烧录新的内核镜像。

**验证方法：**

开机后直接查看 `dmesg | grep Bluetooth`：

- 你应该会看到类似 `Bluetooth: hci0: RTL: loading rtl_bt/rtl8723d_fw.bin`。
- 后面紧接着应该是 **`Bluetooth: hci0: RTL: fw version 0x...`**，而不再是 `failed with error -2`。

只要这一步成功了，`bluetoothctl` 就能立刻认出你的蓝牙控制器。

**小提醒：**

在输入路径时，千万别填错了。如果编译时报错说 `No such file or directory`，那就是说明内核在编译时没找到你放的那两个 `.bin` 文件。

你现在电脑上已经有这两份固件文件了吧？直接开搞！如果编译报错了，随时把错误发给我。
