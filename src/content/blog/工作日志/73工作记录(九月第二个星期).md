默认的SDK内容是没有蜂鸣器的，需要进行配置，现在就以蜂鸣器的配置为例进行海思平台复用的讲解。

~~~c
/* BEEP-配置GPIO10_3引脚为PWM0_12功能 */
static void beep_gpio_pin_mux(void)
{
    void *iocfg2_base = sys_config_get_reg_iocfg2();

    /* 设置寄存器偏移 0x0108 的管脚复用为PWM12 */
    sys_writel(iocfg2_base + 0x0108, 0x0005);
}
~~~

这是田修改好的配置函数，首先在第四行通过sys_config_get_reg_iocfg2函数获取到了cfg2配置寄存器的基地址，然后来看第7行，调用了sys_writel向对应的寄存器里面写入数据从而决定相应的复用功能，首先找到原理图中的蜂鸣器控制引脚如下所示：、

![image-20250908155500767](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202509081555825.png)

然后通过GPIO10_3找到核心板对应的引脚JTAG_TMS/P_IRIS_CONTROL4_1V8，具体如下所示：

![image-20250908155529407](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202509081555510.png)

接下来需要确定该引脚对应的寄存器地址，这时候需要打开官方文档中提供的21AP10_PINOUT_CN.xlsx excel表格，找到其中的管脚控制寄存器，然后搜索JTAG_TMS对应的内容。

![image-20250908155655373](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202509081556427.png)

该引脚确定的内容如下所示，address地址为0x0102F0108，其中0x0102F是由sys_config_get_reg_iocfg2函数来确定的，后四位的0108是要确定的偏移地址，而关于复用功能相关的可以来看该寄存器的低四位：

# BSP 顶层 Makefile 使用说明

## 一、总体说明

1. 由于 Makefile 中文件系统编译依赖组件较多，不能保证单独编译的文件系统可用，建议采用 `make all` 编译
2. 可单独编译 uboot、kernel
3. 本说明针对 hi3516cv610 进行说明
4. 支持选用 `arm-v01c02-linux-musleabi`、`arm-v01c02-linux-gnueabi` 32bit 工具链进行编译
5. `arm-v01c02-linux-musleabi` 工具链对应 musl 库
6. 总体编译时，如需使用 musl 库工具链，请配置 `LIB_TYPE=musl`
7. 单独编译时，如需使用 musl 库工具链，请配置 `CROSS_COMPILE=arm-v01c02-linux-musleabi-`

### 开源工具下载注意事项

由于开源工具整改，部分开源工具不再提供源码包，客户编译 OSDRV 时需要自行下载：

- `linux-5.10.xx.tar.gz`（下载地址：https://www.kernel.org/pub/），将下载的文件存放到 `open_source/linux` 目录中，需与 patch 版本匹配

## 二、编译命令详解

### (1) 编译整个 bsp 目录

bash

```bash
make all
```

**默认编译命令**：

```bash
make LIB_TYPE=musl CHIP=hi3516cv610 DEBUG=0 all
```

**编译参数说明**：

1. `BOOT_MEDIA`：默认选择 spi 启动方式，可指定 `BOOT_MEDIA=emmc`（emmc 启动）或 `BOOT_MEDIA=spi_nand`（release 版本 nand 介质）
2. `LIB_TYPE`：默认使用 musl 库编译
3. `CHIP`：默认编译 `hi3516cv610`，可指定 `hi3516cv608`
4. `DEBUG`：默认 `DEBUG=0`（release 版本），`DEBUG=1` 为 debug 版本
5. 编译结果存放在 `bsp/pub` 目录

### (2) 清除编译文件

```bash
make clean
```

### (3) 彻底清除中间文件

```bash
make distclean
```

### (4) 单独编译 kernel image

```bash
make kernel
```

**修改 config 并重编 kernel 镜像步骤**：

1. 进入 `open_source/linux/linux-5.10.y` 目录，执行：

```bash
make ARCH=arm CROSS_COMPILE=arm-v01c02-linux-musleabi- menuconfig
```

生成 defconfig 文件：

```bash
make ARCH=arm CROSS_COMPILE=arm-v01c02-linux-musleabi- savedefconfig
```

拷贝为新的 deconfig 文件：

```bash
cp defconfig arch/arm/configs/hi3516cv610_new_defconfig
```

返回 bsp 目录重新编译（eMMC 介质需加 `BOOT_MEDIA=emmc`）：

```bash
make KERNEL_CFG=hi3516cv610_new_defconfig kernel
```

### (5) 单独编译非安全启动镜像

```bash
make gslboot_build -j
```

**自定义表格编译**：

```bash
make gslboot_build REGBIN_XLSM=xxxx.xlsm -j
```

（需将表格拷贝到 `tools/pc/boot_tools` 目录）

## 三、制作文件系统镜像（以 hi3516cv610 为例）

### 1. SPI Flash（jffs2 格式）

```bash
./bsp/pub/bin/pc/mkfs.jffs2 -d bsp/pub/rootfs_musl_arm -l -e 0x10000 -o bsp/pub/rootfs_hi3516cv610_64k.jffs2
```

**参数说明**：`-e 0x10000` 为块大小 64KB（需根据 uboot 启动打印信息修改）

### 2. NAND Flash（ubifs 格式）

```bash
./bsp/tools/pc/ubi_sh/mkubiimg.sh hi3516cv610 2k 128k bsp/pub/rootfs_musl_arm 32M bsp/pub/bin/pc
```

**参数说明**：`2k` 为 pagesize，`128k` 为 block size（需根据 uboot 启动打印信息修改）

### 3. eMMC（ext4 格式）

```bash
# 1. 创建镜像文件
dd if=/dev/zero of=bsp/pub/hi3516cv610_emmc_image_musl/rootfs_hi3516cv610_96M.ext4 bs=512 count=196608

# 2. 格式化 ext4
./bsp/pub/bin/pc/mkfs.ext4 bsp/pub/hi3516cv610_emmc_image_musl/rootfs_hi3516cv610_96M.ext4

# 3. 填充文件系统
cd open_source/e2fsprogs/out/pc/contrib
./populate-extfs.sh ../../../../../smp/a7_linux/source/bsp/pub/rootfs_musl_arm ../../../../../smp/a7_linux/source/bsp/pub/hi3516cv610_emmc_image_musl/rootfs_hi3516cv610_96M.ext4
```

## 四、注意事项

### (1) 源码包操作限制

- 禁止在 Windows 下复制源码包，可能导致：
  - Linux 可执行文件变为非可执行文件
  - u-boot / 内核中的符号链接文件变为实际文件，导致源码包膨胀

### (2) 芯片编译选项配置

此芯片具有浮点运算单元和 neon，文件系统库采用兼容软浮点调用接口的硬浮点和 neon 编译，所有板端代码编译时需在 Makefile 中添加：

```makefile
CFLAGS += -mcpu=cortex-a7 -mfloat-abi=softfp -mfpu=neon-vfpv4
CXXFLAGS += -mcpu=cortex-a7 -mfloat-abi=softfp -mfpu=neon-vfpv4
```
