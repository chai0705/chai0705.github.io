// 🚀 主命令函数：执行文件拷贝（从文件系统 → 裸设备偏移）
int do_cpfile(cmd_tbl_t *cmdtp, int flag, int argc, char *const argv[]) {
    // 必须传入两个参数：<源描述> <目标描述>
    if (argc != 3) {
        printf("用法: %s\n", CMD_CPFILE_HELP);
        return CMD_RET_USAGE;
    }

    char *src_str = argv[1]; // 源描述，如 "mmc 1:1 /u-boot.bin"
    char *dst_str = argv[2]; // 目标描述，如 "mmc 0 0x0"

    rw_handle_t hsrc, hdst;

    // 解析源句柄（必须是文件系统）
    if (parse_handle(&hsrc, src_str) != 0) {
        printf("错误：无效的源描述: %s\n", src_str);
        return CMD_RET_FAILURE;
    }

    // 解析目标句柄（必须是裸设备）
    if (parse_handle(&hdst, dst_str) != 0) {
        printf("错误：无效的目标描述: %s\n", dst_str);
        return CMD_RET_FAILURE;
    }

    // 使用预定义缓冲区（避免动态分配，稳定高效）
    char *buffer = (char *)CPFILE_BUFFER_ADDR;
    loff_t buffer_size = CPFILE_BUFFER_SIZE;

    // 获取源文件大小（这是要烧写的总字节数）
    loff_t total_size = hsrc.size;
    if (total_size <= 0) {
        printf("错误：源文件为空或无法获取大小。\n");
        return CMD_RET_FAILURE;
    }

    // 打印烧写信息
    printf("正在烧写 '%s' (%lld 字节) → 设备偏移 0x%llx\n", hsrc.filename, total_size, hdst.offset);

    loff_t copied = 0;
    loff_t chunk_read;

    // 循环读取源文件 → 写入目标设备
    while ((chunk_read = handle_read(&hsrc, buffer, buffer_size)) > 0) {
        loff_t chunk_written = handle_write(&hdst, buffer, chunk_read);
        if (chunk_written != chunk_read) {
            printf("\n错误：在偏移 0x%llx 处写入失败\n", hdst.offset);
            return CMD_RET_FAILURE;
        }
        copied += chunk_written;
        print_progress_bar(copied, total_size); // 更新进度条
    }

    // 检查是否因错误提前退出
    if (chunk_read < 0) {
        printf("\n错误：读取源文件失败。\n");
        return CMD_RET_FAILURE;
    }

    // 完成提示
    printf("\n✅ 成功烧写 %lld / %lld 字节。\n", copied, total_size);
    return CMD_RET_SUCCESS;
}

U-Boot 的命令行系统是其核心功能之一，允许用户在启动阶段通过控制台输入命令，实现环境变量管理、设备操作、内核启动等功能。理解其命令行格式和命令编写方式，对于定制 U-Boot 至关重要。

### 一、U-Boot 命令行基本格式

U-Boot 命令行的语法规则与 Linux Shell 类似，基本格式为：

```bash
<命令名> [选项] [参数1] [参数2] ...
```

#### 核心要素：

1. **命令名**：区分大小写（如 `printenv` 不能写作 `PrintEnv`）。
2. **选项**：通常以 `-` 开头（如 `fatload -b 0x42000000 mmc 1:1 uImage` 中的 `-b`）。
3. **参数**：根据命令功能不同，参数可以是数值（如地址 `0x42000000`）、设备名（如 `mmc 1:1`）、字符串（如环境变量名）等。
4. **分隔符**：命令名、选项、参数之间用**空格**分隔；若参数包含空格，需用双引号包裹（如 `setenv bootargs "console=ttyAMA0,115200"`）。

### 二、U-Boot 命令的代码实现框架

U-Boot 的所有命令都通过统一的结构体和注册机制实现。核心步骤包括：

1. 定义命令结构体（描述命令的名称、功能、参数等）；
2. 实现命令执行函数（命令的具体逻辑）；
3. 注册命令（将命令加入 U-Boot 的命令表，使其可被解析执行）。

#### 1. 命令结构体 `struct cmd_tbl_s`

每个命令都由 `struct cmd_tbl_s` 结构体描述，定义在 `include/command.h` 中：

```c
struct cmd_tbl_s {
    char        *name;        // 命令名（如 "printenv"）
    int         maxargs;      // 最大参数数量（含命令名本身）
    int         repeatable;   // 是否支持重复执行（按回车重复上一条命令）
    int         (*cmd)(struct cmd_tbl_s *, int, int, char *const []);  // 命令执行函数
    char        *usage;       // 简短用法说明（如 "print environment variables"）
#ifdef CONFIG_SYS_LONGHELP
    char        *help;        // 详细帮助信息（使用 "help 命令名" 查看）
#endif
#ifdef CONFIG_AUTO_COMPLETE
    int         (*complete)(int argc, char *const argv[], char last_char, int maxv, char *cmdv[]);  // 自动补全函数
#endif
};

typedef struct cmd_tbl_s cmd_tbl_t;  // 类型别名
```

- 关键成员：
  - `name`：命令的唯一标识，必须与用户输入的命令名一致。
  - `maxargs`：限制参数总数（含命令名），超过则报错（如 `maxargs=3` 表示命令名 + 2 个参数）。
  - `cmd`：函数指针，指向命令的具体实现逻辑（核心）。

#### 2. 命令执行函数原型

命令执行函数的原型固定为：

```c
int cmd_func(cmd_tbl_t *cmdtp, int flag, int argc, char *const argv[]);
```

- 参数说明：
  - `cmdtp`：指向当前命令的 `cmd_tbl_t` 结构体（可获取命令名、帮助信息等）。
  - `flag`：命令执行标志（如 `CMD_FLAG_REPEAT` 表示重复执行），通常无需处理。
  - `argc`：实际参数数量（含命令名，如 `printenv bootargs` 中 `argc=2`）。
  - `argv`：参数数组（`argv[0]` 为命令名，`argv[1]` 为第一个参数，以此类推）。

#### 3. 命令注册宏 `U_BOOT_CMD`

定义完结构体和执行函数后，需用 `U_BOOT_CMD` 宏将命令注册到 U-Boot 命令表中。该宏定义在 `include/command.h` 中，格式为：

```c
U_BOOT_CMD(
    name,          // 命令名（字符串）
    maxargs,       // 最大参数数（含命令名）
    repeatable,    // 是否可重复执行（1 允许，0 禁止）
    cmd,           // 命令执行函数
    usage,         // 简短用法说明
    help           // 详细帮助信息（可选）
);
```

### 三、实战：编写一个自定义 U-Boot 命令

以实现一个 `hello` 命令为例，功能是打印问候语，支持自定义名称参数（如 `hello topeet` 打印 `Hello, topeet!`）。

#### 步骤 1：创建命令文件

在 U-Boot 源码的 `common/` 目录下创建 `cmd_hello.c`（命令文件通常以 `cmd_xxx.c` 命名）。

#### 步骤 2：实现命令执行函数

```c
#include <common.h>
#include <command.h>

// 命令执行函数：hello [名称]
static int do_hello(cmd_tbl_t *cmdtp, int flag, int argc, char *const argv[])
{
    // 检查参数数量（最少1个：命令名本身；最多2个：命令名+名称）
    if (argc > 2) {
        // 打印用法（使用 cmdtp->usage）
        printf("Usage: %s\n", cmdtp->usage);
        return 1;  // 错误返回非0
    }

    // 执行逻辑：无参数则默认问候，有参数则使用指定名称
    if (argc == 1) {
        printf("Hello, World!\n");
    } else {
        printf("Hello, %s!\n", argv[1]);  // argv[1] 为用户输入的名称
    }

    return 0;  // 成功返回0
}
```

#### 步骤 3：定义命令结构体并注册

在 `cmd_hello.c` 中继续添加：

```c
// 定义命令结构体（通过宏注册）
U_BOOT_CMD(
    hello,          // 命令名：hello
    2,              // 最大参数数：2（hello + 名称）
    1,              // 支持重复执行（按回车重复）
    do_hello,       // 执行函数：do_hello
    "Print a greeting message",  // 简短用法
    "hello [name] - Print 'Hello, name!' (default: Hello, World!)\n"  // 详细帮助
);
```

#### 步骤 4：配置编译选项

需确保该命令被编译进 U-Boot：

1. 在common/Makefile中添加：

   ```makefile
   obj-y += cmd_hello.o  # 编译 cmd_hello.c 为目标文件
   ```

   

2. （可选）若需通过 Kconfig 控制，在common/Kconfig 中添加：kconfig

   ```kconfig
   config CMD_HELLO
       bool "Enable 'hello' command"
       default y
       help
           Add support for the 'hello' command, which prints a greeting message.
   ```

   

   然后在 configs/xxx_defconfig中启用 CONFIG_CMD_HELLO=y。

#### 步骤 5：编译与测试

1. 重新编译 U-Boot：

   ```bash
   make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf-  # 根据目标架构调整
   ```

   

2. 将新 U-Boot 烧录到设备，启动后在命令行测试：

### 四、高级用法：参数解析与设备交互

实际命令常需解析复杂参数（如地址、设备名）或操作硬件（如读写 Flash、加载内核）。以下是一个更实用的示例：实现 `mmc_read` 命令，从指定 MMC 设备读取数据到内存。

#### 示例：`mmc_read` 命令

```c
#include <common.h>
#include <command.h>
#include <mmc.h>  // MMC设备操作接口

static int do_mmc_read(cmd_tbl_t *cmdtp, int flag, int argc, char *const argv[])
{
    // 参数格式：mmc_read <dev> <sector> <addr> <count>
    // 如：mmc_read 1 0x100 0x42000000 0x10（从MMC设备1的0x100扇区读0x10个扇区到0x42000000）
    if (argc != 5) {
        printf("Usage: %s\n", cmdtp->usage);
        return 1;
    }

    // 解析参数（字符串转数值）
    int dev = simple_strtoul(argv[1], NULL, 0);  // MMC设备号（如1）
    ulong sector = simple_strtoul(argv[2], NULL, 0);  // 起始扇区
    ulong addr = simple_strtoul(argv[3], NULL, 0);  // 目标内存地址
    ulong count = simple_strtoul(argv[4], NULL, 0);  // 扇区数量

    // 初始化MMC设备
    struct mmc *mmc = find_mmc_device(dev);
    if (!mmc) {
        printf("MMC device %d not found!\n", dev);
        return 1;
    }
    if (mmc_init(mmc) != 0) {
        printf("Failed to initialize MMC device %d!\n", dev);
        return 1;
    }

    // 读取数据（U-Boot提供的mmc_read函数）
    if (mmc->block_dev.block_read(dev, sector, count, (void *)addr) != count) {
        printf("MMC read failed!\n");
        return 1;
    }

    printf("Read %lu sectors from MMC %d, sector %lu to 0x%lx\n", count, dev, sector, addr);
    return 0;
}

// 注册命令
U_BOOT_CMD(
    mmc_read,
    5,  // 参数数量：mmc_read + 4个参数
    0,  // 不支持重复执行
    do_mmc_read,
    "Read data from MMC device",
    "mmc_read <dev> <sector> <addr> <count>\n"
    "  - Read <count> sectors from MMC <dev> starting at <sector> to memory <addr>"
);
```

### 五、命令行解析流程（补充原理）

当用户在 U-Boot 命令行输入命令后，流程如下：

1. **命令行解析**：`common/cli.c` 中的 `run_command()` 函数将输入字符串按空格分割为 `argc` 和 `argv`。
2. **命令查找**：遍历全局命令表（`__u_boot_cmd_start` 到 `__u_boot_cmd_end`），根据 `argv[0]` 匹配命令名。
3. **参数校验**：检查实际参数数 `argc` 是否在命令定义的 `maxargs` 范围内。
4. **执行命令**：调用命令对应的 `cmd` 函数（如 `do_hello`），传入 `argc`、`argv` 等参数。

### 总结

U-Boot 命令行的核心是 `cmd_tbl_t` 结构体和 `U_BOOT_CMD` 注册宏。编写自定义命令的步骤为：

1. 实现命令执行函数（处理 `argc` 和 `argv`）；
2. 用 `U_BOOT_CMD` 宏注册命令，指定名称、参数限制和帮助信息；
3. 配置编译选项，确保命令被包含进 U-Boot。

实际开发中可参考 U-Boot 源码中已有的命令（如 `common/cmd_env.c` 的 `printenv`、`common/cmd_mmc.c` 的 `mmc`），快速理解复杂命令的实现逻辑。

### U-Boot 命令系统核心结构体 `cmd_tbl_s` 详解

#### 一、结构体整体作用

`struct cmd_tbl_s` 是 U-Boot 命令系统的基础数据结构，用于描述每个命令的元信息（名称、参数、功能函数等）。U-Boot 通过维护该结构体的数组（命令表）实现命令的注册、查找和执行，是嵌入式系统中命令行交互的核心组件。

#### 二、结构体成员详细解析

```c
struct cmd_tbl_s {
    char        *name;          /* 命令名称（如 "boot"、"mmc"） */
    int         maxargs;       /* 命令允许的最大参数数量 */
    int         repeatable;    /* 是否允许命令自动重复（用于脚本场景） */
    int         (*cmd)(struct cmd_tbl_s *, int, int, char * const []);  /* 命令执行函数指针 */
    char        *usage;        /* 简短用法说明（显示于help命令） */
#ifdef CONFIG_SYS_LONGHELP
    char        *help;         /* 详细帮助信息（可选，需配置开启） */
#endif
#ifdef CONFIG_AUTO_COMPLETE
    /* 命令参数自动补全函数（提升交互体验） */
    int         (*complete)(int argc, char * const argv[], char last_char, int maxv, char *cmdv[]);
#endif
};
```

#### 1. `char *name` - 命令名称

- **作用**：命令的字符串标识（如 `"printenv"`），用户在命令行输入该字符串触发命令执行。
- **示例**：`name = "reset"` 对应命令行输入 `reset` 重启系统。
- **设计要点**：名称需唯一，通常为小写字母，简洁易记。

#### 2. `int maxargs` - 最大参数数量

- **作用**：限制命令可接收的参数个数（不包含命令名本身），用于参数合法性检查。
- 示例：
  - `maxargs = 1`：命令如 `echo arg`（仅 1 个参数）
  - `maxargs = -1`：表示参数数量无限制（如 `ls /dir/*`）
- **运行时检查**：若输入参数数 > `maxargs`，U-Boot 会提示用法错误。

#### 3. `int repeatable` - 命令重复属性

- 取值：
  - `0`：不允许自动重复（默认）
  - `1`：允许自动重复（如在循环中持续执行）
- 应用场景：
  - 监控命令（如 `watchdog reset`）需设置为可重复
  - 一次性操作（如 `reboot`）设为不可重复

#### 4. `int (*cmd)(...)` - 命令执行函数指针

- 核心功能：指向命令的实际处理函数，原型为：

  ```c
  int command_function(struct cmd_tbl_s *cmdtp, int flag, int argc, char *const argv[]);
  ```

  

- 参数解析：

  - `cmdtp`：当前命令的结构体指针（可用于获取命令元信息）
  - `flag`：命令执行标志（如 `CMD_FLAG_REPEAT` 表示重复执行）
  - `argc`：参数数量（包含命令名，如 `reset` 的 `argc=1`）
  - `argv[]`：参数字符串数组（`argv[0]` 为命令名）

- 返回值：

  - `CMD_RET_SUCCESS`（0）：执行成功
  - `CMD_RET_FAILURE`（非 0）：执行失败
  - `CMD_RET_USAGE`：参数错误，显示用法

#### 5. `char *usage` - 简短用法说明

- **作用**：在 `help` 命令中显示的简洁用法（单行描述）。

- 示例：

  ```
  usage = "printenv [varname]"
  ```

  

  ```
  help printenv
  ```

  ```plaintext
  printenv - print environment variables
  usage:
  printenv [varname]
  ```

  

#### 6. `char *help` - 详细帮助信息（条件编译）

- **启用条件**：需在 U-Boot 配置中定义 `CONFIG_SYS_LONGHELP`。

- **作用**：提供命令的详细说明（多行），如参数含义、示例等。

- 示例

  ```c
  help = "printenv - print environment variables\n"
         "Usage:\n"
         "  printenv [varname]\n"
         "If varname is given, print that variable, otherwise print all.\n";
  ```

  

#### 7. `int (*complete)(...)` - 自动补全函数（条件编译）

- **启用条件**：需定义 `CONFIG_AUTO_COMPLETE`（提升交互体验的功能）。

- **功能**：当用户输入命令时按下 Tab 键，自动补全未输入的参数或命令。

- 参数解析

  ：

  - `argc`：当前已输入的参数数量
  - `argv[]`：已输入的参数字符串
  - `last_char`：最后输入的字符（用于过滤补全项）
  - `maxv`：补全结果最大数量
  - `cmdv[]`：补全结果数组（由函数填充）

- **返回值**：实际补全的数量。





#### 步骤 2：实现命令执行函数

```c
#include <common.h>
#include <command.h>

// 命令执行函数：hello [名称]
static int do_hello(cmd_tbl_t *cmdtp, int flag, int argc, char *const argv[])
{
    // 检查参数数量（最少1个：命令名本身；最多2个：命令名+名称）
    if (argc > 2) {
        // 打印用法（使用 cmdtp->usage）
        printf("Usage: %s\n", cmdtp->usage);
        return 1;  // 错误返回非0
    }

    // 执行逻辑：无参数则默认问候，有参数则使用指定名称
    if (argc == 1) {
        printf("Hello, World!\n");
    } else {
        printf("Hello, %s!\n", argv[1]);  // argv[1] 为用户输入的名称
    }

    return 0;  // 成功返回0
}
```

#### 步骤 3：定义命令结构体并注册

在 `cmd_hello.c` 中继续添加：

```c
// 定义命令结构体（通过宏注册）
U_BOOT_CMD(
    hello,          // 命令名：hello
    2,              // 最大参数数：2（hello + 名称）
    1,              // 支持重复执行（按回车重复）
    do_hello,       // 执行函数：do_hello
    "Print a greeting message",  // 简短用法
    "hello [name] - Print 'Hello, name!' (default: Hello, World!)\n"  // 详细帮助
);
```
