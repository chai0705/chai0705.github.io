| ------------ | ----------------- | ------------ | --------------------------------------- | -------------------------- |
| **uboot**    | 0x00002000 (4 MB) | 0x00004000   | 存储 U-Boot 引导程序                    | 系统启动阶段引导           |
| **misc**     | 0x00002000(4 MB)  | 0x00006000   | 存储系统状态信息（如 OTA 标记等）       | 辅助系统更新与恢复         |
| **boot**     | 0x00020000(64 MB) | 0x00008000   | 存储内核、设备树和uboot和内核logo       | bootable：可引导分区       |
| **recovery** | 0x00020000(64 MB) | 0x00028000   | 存储恢复模式的镜像和工具                | 系统故障时用于恢复         |
| **backup**   | 0x00010000(32 MB) | 0x00048000   | 存储系统备份数据                        | 系统安全机制的一部分       |
| **rootfs**   | 动态扩展 (`-`)    | 0x00078000   | 存储 Linux 根文件系统，包括所有应用程序 | **grow**：使用所有剩余空间 |





| **分区名称** | **大小**                 | **起始地址** | **用途**                                    | **备注**                   |
| ------------ | ------------------------ | ------------ | ------------------------------------------- | -------------------------- |
| **uboot**    | 0x00004000 字节 (8 MB)   | 0x00004000   | 存储 U-Boot 引导加载程序                    | 系统的初始启动阶段         |
| **boot**     | 0x00040000 字节 (128 MB) | 0x00008000   | 存储内核镜像、启动配置文件、设备树overlay等 | **bootable**：可引导分区   |
| **rootfs**   | 动态扩展 (`-`)           | 0x00048000   | 存储根文件系统，包括所有应用和配置文件      | **grow**：使用剩余存储空间 |





| **对比维度**       | **rk传统分区镜像**                                           | **定制扩展分区镜像**                                         |
| ------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **设备树管理**     | 每个 rk传统分区镜像与设备树 (dtb) 紧密绑定，若硬件变化需重构镜像。 | 支持集成多个设备树，根据硬件 ID 动态加载对应设备树，适配多板卡环境。 |
| **多设备适配**     | 无法实现多设备共享，一个镜像只能对应一个硬件型号。           | 一个镜像适配多个设备，通过加载不同的设备树实现多板卡共享。   |
| **设备树插件支持** | 不支持设备树插件，无法动态调整硬件配置。                     | 支持 `.dtbo` 插件，动态更新设备树或配置文件，无需重烧镜像    |
| **启动速度**       | 启动较快，直接加载内核和设备树，减少启动步骤                 | 启动略慢，需要在启动时加载相应设备树和 overlay               |
| **分区结构**       | 分区结构传统，可能存在冗余部分（如 recovery、backup 分区）   | 分区简化，仅包含 uboot**、**boot 和 rootfs 分区，减少冗余，提高存储效率 |
| **OTA 更新支持**   | 支持基础的 OTA 更新，可通过 TF 卡、U 盘或远程升级            | 不支持 OTA 更新，但可以不需重启更新内核、驱动、设备树。      |
| **存储利用率**     | 存储效率较低，冗余结构导致空间浪费                           | 存储效率高，采用动态扩展的 **rootfs**，占用所有剩余空间      |
| **适用场景**       | 适用于硬件配置固定且对启动速度要求高的系统，如工业控制设备   | 适用于多设备环境和需要频繁更新的嵌入式系统，如物联网应用     |
| **维护与更新**     | 维护复杂，需针对硬件变化重构镜像                             | 动态切换设备树，支持在线更新，减少系统停机和维护成本         |





| **应用场景**           | **适合 rkboot**                | **适合 extboot**                      |
| ---------------------- | ------------------------------ | ------------------------------------- |
| **固定硬件的工业系统** | 硬件配置稳定、启动速度要求高   | 若有多板卡共享需求，则选择 extboot    |
| **消费电子产品**       | 启动时间短且硬件不频繁变化     | 若需频繁更新应用程序，选择 extboot    |
| **物联网设备**         | 基础功能的 IoT 设备            | 复杂 IoT 系统，需要在线更新和动态适配 |
| **多板卡项目**         | 不适用，每个硬件需单独构建镜像 | 适用于多设备共享镜像的项目            |
| **长期维护项目**       | 若更新频率低且不需要动态设备树 | 适合需频繁更新的系统                  |
