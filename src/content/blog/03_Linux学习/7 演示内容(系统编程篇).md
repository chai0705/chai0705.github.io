## 2.1 系统调用

==系统调用:是操作系统提供的一组API接口，允许用户空间的应用程序与内核进行直接交互。是用户程序进入内核模式的唯一途径，用于请求操作系统内核执行特定任务。==

#### **常见的系统调用**

| 分类     | 系统调用   | 功能描述                                                   |
| -------- | ---------- | ---------------------------------------------------------- |
| 文件操作 | `open()`   | 打开文件，返回文件描述符。                                 |
|          | `read()`   | 从文件描述符读取数据。                                     |
|          | `write()`  | 向文件描述符写入数据。                                     |
|          | `close()`  | 关闭文件描述符。                                           |
| 进程管理 | `fork()`   | 创建一个新进程。                                           |
|          | `exec()`   | 在当前进程中执行一个新程序。                               |
|          | `wait()`   | 等待子进程结束。                                           |
| 内存管理 | `mmap()`   | 将文件或设备映射到进程的地址空间中，进行内存映射文件 I/O。 |
| 设备管理 | `ioctl()`  | 控制设备操作，提供更灵活的设备操作接口。                   |
| 网络通信 | `socket()` | 创建一个网络通信的套接字，用于网络连接和数据传输。         |

##  2.2 C库函数

==C 库函数（C Library Functions）是 C 语言提供的一组标准化函数，用于执行常见的任务，如文件 I/O、字符串处理、数学运算、内存分配等。C 库函数往往对底层系统调用进行了封装，提供了更高层次的接口，开发者可以通过这些函数轻松实现日常开发任务。==

#### **C 库函数的作用**

- **简化编程**：C 库函数封装了复杂的系统调用和操作系统功能，使得开发者不必直接调用底层系统调用，只需调用更高层次的函数。
- **跨平台兼容**：C 库函数通常设计为跨平台的，它们的接口和行为在不同操作系统中保持一致。这使得使用 C 库函数编写的代码更具可移植性。
- **提供缓冲和优化**：C 库函数通常带有缓冲机制。例如，标准输入输出函数（如 `printf()`、`fread()` 等）通过缓冲区来提高 I/O 性能。

####  **常见的 C 库函数**

| 类别           | 函数名     | 描述                           |
| -------------- | ---------- | ------------------------------ |
| 输入/输出函数  | `printf()` | 格式化输出到标准输出           |
|                | `scanf()`  | 格式化输入，从标准输入获取数据 |
|                | `fopen()`  | 打开一个文件，返回文件指针     |
|                | `fread()`  | 从文件中读取数据到内存中       |
|                | `fwrite()` | 将数据写入文件                 |
| 字符串操作函数 | `strlen()` | 计算字符串的长度               |
|                | `strcpy()` | 复制字符串                     |
|                | `strcmp()` | 比较两个字符串                 |
| 内存管理函数   | `malloc()` | 动态分配内存                   |
|                | `calloc()` | 动态分配并初始化内存           |
|                | `free()`   | 释放内存                       |
| 数学函数       | `sqrt()`   | 计算平方根                     |
|                | `pow()`    | 计算幂                         |
| 其他实用函数   | `time()`   | 获取当前时间                   |
|                | `rand()`   | 生成随机数                     |

至此，关于C语言库函数相关的介绍也就完成了，最后来看本节视频的第三部分系统调用与 C 库函数的对比。

## 2.3 系统调用与 C 库函数的对比

|            | 系统调用                                           | C 库函数                                       |
| ---------- | -------------------------------------------------- | ---------------------------------------------- |
| 抽象层次   | 底层接口，直接与操作系统内核交互                   | 高层封装，简化开发者操作                       |
| 性能与效率 | 高效，但涉及用户模式和内核模式的上下文切换         | 通过缓冲优化性能，可能带来额外开销             |
| 使用场景   | 与操作系统核心功能直接交互，精确控制硬件、设备文件 | 处理常见文件、字符串、数学操作，跨平台应用开发 |
| 可移植性   | 依赖于特定操作系统，较难移植                       | 跨平台标准接口，具有更好的可移植性             |
| 错误处理   | 基础错误处理，开发者需自行处理所有错误             | 提供高级错误处理机制，处理部分错误             |

# 学习环境的搭建

 [VS Code官网](https://code.visualstudio.com/) 



安装完成后，按照以下步骤进行 SSH 连接配置：

1. 打开 VS Code 的 **命令面板**（快捷键：`Ctrl + Shift + P`），输入并选择 `Remote-SSH: Connect to Host`。
2. 在弹出的输入框中，输入你虚拟机的 IP 地址和 SSH 用户名，格式为：`ssh user@IP_address`。例如：`ssh user@192.168.1.100`。
3. 如果是首次连接，系统会提示你输入 SSH 密码，并且询问是否信任该主机，输入密码并选择“是”。
4. 连接成功后，VS Code 界面将会自动切换到远程工作区，你现在就可以在 Ubuntu 环境中编写和运行代码了。
