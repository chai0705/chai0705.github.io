## 1.1 Unix的诞生

时间：上个世纪60年代 

地点：美国贝尔实验室

目的：开发一个多用户多任务的操作系统

结果：由于该项目过于复杂，进展十分缓慢，贝尔实验室就退出了该项目。

____

人物：肯.汤姆森和丹尼斯.里奇

<img src="https://chai-1301855619.cos.ap-beijing.myqcloud.com/202401292058910.png" alt="image-20240127150722157" style="zoom:200%;" />

目的：重新设计一个更简单、更实用的操作系统

结果：在1969年成功开发出了第一版的UNIX操作系统。

后续发展：而在20世纪70年代，由于半导体和集成电路技术的普及，大大拉低了内存制造成本，这使得C语言等高级语言的应用成为了可能，丹尼斯·里奇又着手B语言的改进，就这样直到今天仍被我们使用的C语言就诞生了，于是在C语言的基础上肯·汤姆森和丹尼斯·里奇又对Unix不断更新迭代，最终于1975年发布了Unix第6版本，第6版本的unix内核由9000行的C语言和700行的汇编语言所实现，让Unix系统的维护和修改变的容易，也让其可以方便的移植到不同的计算机上。

## 1.2 GNU和Linux的发展

时间：20世纪70年代末

事情的起因：贝尔实验室的总公司AT&T对UNIX操作系统源代码使用做出了限制，禁止将其供给学生使用，并不在开源。

---

人物：理查德.斯托曼（==对当时UNIX操作系统的私有化和限制使用感到担忧==)

行为：决定创建一个完全由免费软件按组成的类Unix计算机操作系统，

名称：GNU项目（GNU意味着"GNU's not Unix"，强调了GNU和Unix的区别）

核心：开发一系列自由软件工具，如GNU编译器（GCC）、GNU调试器（GDB）、命令行解释器GNU bash等。

挑战：缺少操作系统内核这一核心组件。

----

时间：1991年

人物：林纳斯.拖瓦兹

事件：发布了一个名为Linux的操作系统内核（采用了Unix的设计理念，具有高度的可移植性和可扩展性）

最终结果：Linux内核与GNU项目相结合，为用户提供了一个完整而自由的操作系统（**GNU/Linux操作系统**）。

____

==Linux可以分为内核版本和发行版本。==

- 内核版本：是操作系统的核心、负责管理硬件资源并提供进程管理、内存管理等操作系统的核心功能。
- 发行版本：是基于GNU/Linx构建的完整操作系统，包含了内核、源代码以及相关应用程序，常见的发行版本有**Ubuntu**、red hat、debian等。

![image-20240127161852653](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202401292059512.png)

==由于Ubuntu的图形化界面相对于初学者更加友好，所以本套课程选用ubuntu进行Linux的学习==。

-----

早期：Linux只适用于386处理器

发展：后来经过全世界的网友的帮助，使得Linux最终能够兼容多种架构，包括x86、ARM、PowerPC等。

结果：Linux服务器占据了如今整个服务器市场的高达85% – 90%的份额，而数以十亿计个人手机所运行的安卓系统使用的也是Linux内核。

## 7.1 命令的基本格式

在Linux中，命令的格式通常遵循以下结构：

>command [options] [arguments]

1. 命令（command）：表示要执行的具体命令或程序。命令可以是系统自带的内置命令，也可以是可执行程序或脚本文件。
2. 选项（options）：表示对命令的配置，可选的。选项通常以短格式（单个字母前缀，如 `-a`）或长格式（完整的单词前缀，如 `--all`）提供。
3. 参数（arguments）：参数通常紧跟在选项之后，用于命令的输入或操作对象。参数是命令的操作数，可以是文件、目录、文本、数字等。

==注：这里的选项和参数用[]进行描述，表示可选、非必填==

**示例1：**

~~~shell
ls -la /home/topeet
~~~

- 命令：ls
- 选项：-la
- 参数：/home/topeet

这个命令的含义是列出`/home/topeet`目录下包括隐藏文件的所有文件和目录，并显示它们的详细信息。

**示例2：**

~~~shell
cd /home/topeet
~~~

- 命令：cd
- 参数：/home/topeet

这个命令的含义是将当前工作目录切换到`/home/topeet`目录。

## 7.2 列出文件和目录：ls命令

>命令：ls
>
>英文：list
>
>命令功能：列出指定目录下的文件和子目录
>
>基本格式：ls [options] [file(s) or directory]
>
>详细解释：
>
>- `options`：选项用于控制 `ls` 命令的行为和输出格式。以下是常用的选项：
>  - `-l`：以长格式（详细信息）显示文件和目录。输出包括文件的权限、所属用户和组、大小、创建日期等详细信息。
>  - `-a`：显示所有文件和目录，包括以 `.` 开头的隐藏文件（默认情况下，隐藏文件不会显示）。
>  - `-h`：以人类可读的格式显示文件大小，例如以 KB、MB、GB 等单位显示。
>  - `-r`：以相反的顺序显示文件和目录。
>  - `-t`：按修改时间排序，最近修改的文件或目录显示在前面。
>- `file(s) or directory`：可以指定一个或多个文件或目录作为参数。如果不指定参数，则默认为当前目录。
>
>

## 7.3 打印当前工作目录：pwd命令	

>命令：pwd
>
>英文：print working directory
>
>命令功能：显示当前工作目录的路径

![image-20240305145020458](C:\Users\Administrator\AppData\Roaming\Typora\typora-user-images\image-20240305145020458.png)

>1. `/`：第一个斜杠代表根目录，根目录是 Linux 文件系统的最顶层目录，所有其他目录和文件都是从根目录开始的
>2. `home`：`home` 是根目录下的一个目录，用于存放用户的主目录。
>3. `/`：第二个斜杠代表层次结构，代表`topeet` 是 `home` 目录下的一个子目录。
>4. `topeet`：表示用户名。每个用户都有一个主目录，通常以其用户名命名，用于存放用户的个人文件和配置。

## 7.4 切换目录：cd命令

> 命令：cd
>
> 英文：change directory
>
> 命令功能：显示当前工作目录的路径
>
> 基本格式：cd [directory]
>
> 详细解释：
> 		directory：要切换到的目录。可以是绝对路径（从根目录开始的完整路径）或相对路径（相对于当前目录的路径）
>
> ​	==相对路径：是相对于当前工作目录或参考位置的路径。==
>
> ​	==绝对路径：是从文件系统的根目录（通常是 `/`）开始的完整路径。==
>
> 特殊目录：
>
> - `.`：表示当前目录。
> - `..`：表示当前目录的父目录。
> - `~`：表示当前用户的主目录。

## 7.5 Linux目录结构

>

# 11 链接命令:ln

链接：链接是文件系统中的一个概念，它允许一个文件或目录引用另一个文件或目录，使得多个文件名指向同一个实际的文件或目录。

>命令：ln
>
>英文：link
>
>命令功能：用于在Linux系统中创建硬链接或软链接
>
>> 硬链接：硬链接与原始文件有相同的 inode 号，它们共享文件的内容和属性。因此，对于用户来说，硬链接看起来就像是原始文件的另一个名称。删除原始文件不会影响硬链接，只有当所有指向该文件的硬链接和原始文件都被删除时，文件的内容才会被释放。
>>
>> 软链接：是一个特殊类型的文件，它包含了指向另一个文件或目录的路径。软链接实际上是一个指向目标文件或目录的快捷方式，类似于 Windows 系统中的快捷方式。删除原始文件或目录不会影响符号链接，但如果删除了符号链接所指向的目标文件或目录，则符号链接将变为坏链接。
>
>语法：ln [选项] <目标文件> [链接文件]
>
>常用选项：
>
>- `-s`：创建符号链接。如果不指定该选项，则创建硬链接。

## 12.1 普通用户和root用户

普通用户:指在Linux 系统中具有==有限权限==和==资源访问权限==的用户。普通用户在登录时会获得一个独立的用户账号，该账号有一个用户名和密码。普通用户可以在自己的主目录中创建和管理文件，运行属于自己的进程，但无法修改系统级别的配置文件和进行敏感操作，如安装软件、更改系统设置等。普通用户通常使用命令行终端或图形界面登录，并使用自己的用户名和密码进行身份验证。
	root用户: 指Linux 系统中具有完全访问权限和控制权的特殊用户。root 用户(也称系统管理员用户 )具有对系统的完全控制权，可以执行系统级别的操作，如修改配置文件、安装软件包、创建和删除用户账号等。

**文件权限组：**

- 所有者权限：所有者权限适用于文件或目录的所有者。文件的所有者通常是创建该文件的用户。所有者权限控制着对文件的读、写和执行操作。
- 用户组权限：组权限适用于文件或目录所属的用户组。用户组是一组用户的集合，可以在创建文件时指定。组权限控制着属于同一用户组的成员对文件的读、写和执行操作。
- 其他用户权限：其他用户权限适用于除了文件所有者和所属用户组之外的其他用户。其他用户权限控制着非所有者和非所属用户组成员对文件的读、写和执行操作。

**文件权限状态：**

- 读权限（r）：允许用户查看文件内容、目录中的文件列表和文件属性。
- 写权限（w）：允许用户修改文件内容、创建、重命名或删除文件。
- 执行权限（x）：对于文件，允许用户执行文件（如果是脚本或可执行程序）。对于目录，允许用户进入目录并访问其中的文件。

## 16,1 什么是shell？

Shell（外壳）是Linux操作系统的一种命令行解释器，它提供了用户与操作系统内核进行交互的接口。

![image-20240327131440158](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202403271314211.png)

Shell 两种常见的工作模式：交互式模式和批处理模式。

交互式模式（Interactive Mode）：用户可以逐行输入命令，并立即查看和处理命令的输出结果。Shell 在接收到用户输入后会立即解释和执行相应的命令，并将结果输出到终端。

批处理模式（Batch Mode）：Shell 从一个脚本文件中逐行读取并执行命令，而不需要用户的实时输入和交互。用户事先编写好一系列的命令，并将它们保存在一个脚本文件中。然后，用户可以通过执行脚本文件来自动运行其中的命令。批处理模式适用于用户需要自动化执行一系列命令、进行批量处理或重复性或逻辑判断任务的情况。

## 16.2 更改默认的 Shell

可以通过以下命令打印当前系统所使用的shell解析器：

~~~shell
echo $SHELL
~~~

查看系统中全部的shell解析器

~~~shell
cat /etc/shells
~~~

可以通过以下命令安装zsh 解析器：

~~~shell

~~~

## 16.3 Shell内置命令

在Shell中，存在两种类型的命令：内置命令（Built-in Commands）和外部命令（External Commands）。

内置命令：直接嵌入在 Shell 解释器中的命令。这意味着 Shell 解释器本身提供了这些命令的实现，无需调用外部程序。

外部命令：由独立的可执行程序提供的命令。外部命令可以是系统自带的工具，也可以是用户自己安装的软件。在执行外部命令时，Shell 会搜索系统的可执行文件路径，以找到并执行相应的命令。用户可以通过指定命令的完整路径或将可执行文件所在的目录添加到 PATH 环境变量中来执行外部命令。

区分内置命令和外部命令的一个简单方法是使用 `type` 命令。例如，`type cd` 将显示 "cd is a shell builtin"，而 `type ls` 将显示 "ls is /bin/ls"，表明 `cd` 是一个内置命令，而 `ls` 是一个外部命令。

## 18.1 变量的组成和使用

> 变量：用于存储和表示数据的一种命名实体，包含各种类型的数据，如字符串、数字、命令输出等
>
> 命名规范：
>
> - 变量必须以字母或下划线开头，并由字母、数字和下划线组成
> - 变量名对大小写敏感。
> - 建议使用大写字母来表示常量或全局变量，使用小写字母来表示局部变量。
>
> 变量的赋值：使用等号（=）将值赋给变量。需要注意的是，等号两边不能有空格，否则空格会被解释成命令和参数的一部分。例如：`var=Hello, World!`
>
> 变量的使用：在Shell脚本中，可以通过在变量名前加`$`符号来引用变量的值。例如：`echo $var`。

> 界定符：
> 	（1）单引号:单引号用于创建保持字符串的字面值的字符串。在单引号字符串中，所有字符都被视为普通字符，不进行变量扩展、命令替换或特殊字符转义。
>
> (2）双引号:双引号用于创建字符串，并支持变量扩展和特殊字符转义。

## 18.2 变量的分类

> （1）本地变量：在Shell脚本中定义和使用的变量
>
> ​		全局变量:在Shell的解析环境中存在的变量，不需要任何修饰符进行声明。无论是在函数内部还是函数外部，全局变量的作用范围都是整个Shell进程。全局变量的生命周期从声明语句开始，一直持续到bash进程结束。
>
> ​		局部变量:使用`local`关键字进行修饰，只能在函数内部声明。局部变量的作用范围仅限于声明它的函数内部，在函数结束后，局部变量将不再可用。局部变量的生命周期从声明语句开始，一直持续到函数执行结束。
>
> （2）环境变量：是操作系统自带的变量，每个bash进程都有自己的环境变量集合。当启动一个bash子进程时，子进程会从父进程继承一份相同的环境变量副本。这意味着子进程拥有与父进程相同的初始环境变量。
>
> 使用`env`打印当前bash进程的环境变量。
>
> 使用`export`命令将一个本地变量导出为环境变量
>
> （3）特殊变量:是环境变量中的一组预定义的特殊变量。特殊变量提供了对脚本执行环境、命令行参数、退出状态等信息的访问。
>
> 1. `$0`：表示当前脚本的名称。在脚本中使用`$0`可以获取脚本的名称，包括路径。
> 2. `$1`、`$2`、`$3`...：表示脚本的位置参数或命令行参数。当脚本被调用时，可以通过`$1`、`$2`等来获取传递给脚本的参数值。`$1`表示第一个参数，`$2`表示第二个参数，以此类推。
> 3. `$#`：表示命令行参数的个数。`$#`用于获取传递给脚本的参数的总数。
> 4. `$*`：表示所有命令行参数的列表。`$*`将所有命令行参数作为单个字符串返回。
> 5. `$@`：表示所有命令行参数的列表。与`$*`不同，`$@`将命令行参数作为多个独立的字符串返回。
> 6. `$?`：表示上一个命令的退出状态。当命令执行完毕后，可以使用`$?`来获取其退出状态。通常，如果命令成功执行，则`$?`的值为0，否则为非零值。

## 18.3 命令代换

> 命令代换:将命令的输出作为字符串嵌入到变量中
>
> 可以使用反引号 ` 或美元符号加圆括号 $() 来表示命令代换。

## 19.2 if分支结构

> ​	分支结构名称：`if` 分支结构
>
> ​	适用范围：用于根据条件的真假执行不同的代码块，处理复杂的条件逻辑和数值/字符串比较。

基本语法1：

~~~shell
if condition
then
    # 代码块1，条件为真时执行的代码
else
    # 代码块2，条件为假时执行的代码（可选）
fi
~~~

基本语法2：

~~~shell
if condition1
then
    # 代码块1，条件1为真时执行的代码
elif condition2
then
    # 代码块2，条件2为真时执行的代码
else
    # 代码块3，以上条件都为假时执行的代码（可选）
fi
~~~

​	

示例：
~~~shell
#!/bin/bash

# 编写一个成绩判断的脚本，将脚本的第一个参数作为要判断的分数
# 如果分数大于等于90则为优秀
# 如果分数大于等于80则为良好
# 如果分数大于等于60则为及格
# 如果分数小于60则为不及格
score=$1
if [ $score -ge 90 ]
then
        echo 优秀
elif [ $score -ge 80 ]
then
        echo 良好
elif [ $score -ge 60 ]
then    
        echo 及格
else 
        echo 不及格
fi
~~~



## 19.3 case 分支结构

> ​	分支结构名称：`case` 分支结构
>
> ​	使用范围：适用于根据模式匹配执行不同的操作，可以根据不同的条件执行不同的代码块。		

​	基本语法：

```shell
case expression in
    模式1)
        # 匹配 模式1 时执行的代码块
        ;;
    模式2)
        # 匹配 模式2 时执行的代码块
        ;;
    模式3)
        # 匹配 模式3 时执行的代码块
        ;;
    *)
        # 默认情况下执行的代码块
        ;;
esac
```





~~~shell
#!/bin/bash
# 编写一个选择水果的脚本，将脚本的第一个参数作为要选择的水果
# 有三种水果可选，分别为苹果、香蕉和橘子

fruit=$1

case $fruit in
    apple)
        echo 选择的水果为苹果
        ;;
    banana)
        echo 选择的水果为香蕉
        ;;
    orange)
        echo 选择的水果为橘子
        ;;
    *)
        echo 选择了其他的水果
        ;;
esac
~~~

>
> ​	优势：合理地使用函数，可以将复杂的脚本逻辑分解为可重用和可维护的模块，提高Shell脚本的可读性、可维护性和可扩展性。



> ​	21.1 函数定义和调用
>
> ​	21.2 函数参数
>

>
> ​	21.4 局部变量

## 21.1 函数定义和调用

​	函数定义的基本语法：

~~~shell
function name() 
{
    # 函数体，包含一系列的命令和逻辑
}
~~~

​	要调用函数，只需使用函数名即可：

```bash
name
```



~~~shell
#!/bin/bash

# 构造函数，通过函数打印hello world
function hello()
{
        echo "hello world"
}

hello
~~~



## 21.2 函数参数	

函数可以接受参数，以便在函数内部使用。在函数定义中使用`$1`、`$2`等特殊变量来引用传递给函数的参数。



~~~shell
#!/bin/bash

# 打印函数传递的第一个、第二个参数
function hello()
{
        echo 传入的第一个参数为$1
        echo 传入的第一个参数为$2
}

hello 1 2 
~~~

## 21.3 函数返回值

函数可以通过`return`语句返回一个退出状态码，范围为0-255。在函数中使用`return`语句后，将退出函数并返回指定的状态码，可以通过$?得到返回值

~~~shell
#!/bin/bash

# 打印函数的返回值
function hello()
{
        return 10
}
hello
echo 函数返回值为$?
~~~



## 21.4 局部变量

在函数内部定义的变量默认是全局的，可以在函数外部访问。如果希望在函数内部创建局部变量，可以使用`local`关键字

~~~shell
#!/bin/bash

# 演示局部变量
function hello()
{
    local var=topeet
    echo 局部变量值为$var
}

hello  
echo 局部变量值为$var
~~~
