>

____

我们先来看第二个问题，我们为什么要学习系统编程，在Linux系统中，为了保障系统的安全和稳定，系统将内存划分为**用户空间**和**内核空间**。其中内核空间运行的是操作系统核心功能，他负责管理硬件、内存、文件系统、进程、网络等关键资源，而用户空间是应用程序运行的区域，普通的应用程序无法直接访问内核空间中的资源。

而在嵌入式领域，很多情况下我们都需要对底层相关硬件进行操作，例如对LED、GPIO、蜂鸣器、串口的控制，那要如何在用户空间中通过应用程序实现对上述硬件的控制呢？这就要用到Linux系统编程相关的知识了。

![image-20240921155030180](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202409301422738.png)

- 内核空间：是操作系统核心功能运行的区域，负责管理硬件、内存、文件系统、进程、网络等关键资源。

- 用户空间：是应用程序运行的区域，普通程序无法直接访问内核空间中的资源。

需求：对==LED、GPIO、蜂鸣器、串口==等底层硬件进行操作

____

Linux系统编程实际上是一种编程方式，可以使我们编写的程序能够直接与Linux系统“内核”进行交互。通过系统编程，程序能够通过受控的方式从用户空间请求内核空间的服务。这种请求通常是通过系统调用（system calls）实现的。系统调用提供了一系列接口，使得用户程序能够安全地让内核执行特定的操作，例如，当应用程序需要打开一个文件时，它会通过`open()`系统调用向内核请求该文件的访问权限，而如果程序需要对一个文件进行读写时，则会分别使用`read` `write`系统调用向内核请求该文件的读写权限，然后进行读取和写入。

除了系统调用之外，Linux还提供了库函数，库函数可以看作是对系统调用的封装，属于操作系统开发人员预先定义好的工具，可以帮助开发者更方便地与内核进行交互，简化系统调用的使用。

系统编程：可以使我们编写的程序能够直接与Linux系统“内核”进行交互的一种编程方式。

##  2.1 系统调用

系统调用是由操作系统内核提供的一组API接口，允许用户空间的应用程序与内核进行直接交互。是用户程序进入内核模式的唯一途径，用于请求操作系统内核执行特定任务。系统调用可以访问受操作系统管理的底层资源，如文件系统、设备、进程、内存和网络。

==系统调用:是操作系统提供的一组API接口，允许用户空间的应用程序与内核进行直接交互。是用户程序进入内核模式的唯一途径，用于请求操作系统内核执行特定任务。==

____

然后来对常见的系统调用进行简单的介绍，常见的系统调用可以分为文件操作、内存管理、设备管理和网络通信四个类别，不论要编写的应用程序多么简单，文件操作相关的系统调用都是必须的，open表示打开文件、read和write分别表示从文件中读取数据、和从文件中写入数据、最后使用close关闭文件，在接下来的章节中会编写关于上述四个函数的实际例程，而内存管理mmap、设备管理ioctl以及网络通信socket在后面的实战章节中会进行详细的讲解，从而加深大家对各个函数的认识。

| 分类     | 系统调用   | 功能描述                                                   |
| -------- | ---------- | ---------------------------------------------------------- |
| 文件操作 | `open()`   | 打开文件，返回文件描述符。                                 |
|          | `read()`   | 从文件描述符读取数据。                                     |
|          | `write()`  | 向文件描述符写入数据。                                     |
|          | `close()`  | 关闭文件描述符。                                           |
| 内存管理 | `mmap()`   | 将文件或设备映射到进程的地址空间中，进行内存映射文件 I/O。 |
| 设备管理 | `ioctl()`  | 控制设备操作，提供更灵活的设备操作接口。                   |
| 网络通信 | `socket()` | 创建一个网络通信的套接字，用于网络连接和数据传输。         |

## 2.2 C 库函数

然后来看第二部分C语言库函数，C 库函数是C 语言提供的一组标准化函数，用于执行常见的任务，如文件 I/O、字符串处理、数学运算、内存分配等。C 库函数往往会对底层系统调用进行封装，从而提供更高层次的接口，开发者可以通过这些函数轻松实现日常开发任务。

==C 库函数（C Library Functions）是 C 语言提供的一组标准化函数，用于执行常见的任务，如文件 I/O、字符串处理、数学运算、内存分配等。C 库函数往往对底层系统调用进行了封装，提供了更高层次的接口，开发者可以通过这些函数轻松实现日常开发任务。==

____

与系统调用相比，使用C 库函数进行编程具有以下优点，首先是可以简化编程，C 库函数封装了复杂的系统调用和操作系统功能，使得开发者不必直接调用底层系统调用，只需调用更高层次的函数，第二个优点为可以跨平台兼容，C 库函数通常设计为跨平台的，它们的接口和行为在不同操作系统中保持一致。这使得使用 C 库函数编写的代码更具可移植性，最后使用C库函数可以提供缓冲和优化，C 库函数通常带有缓冲机制，可以通过缓冲区来提高 I/O 性能。

- **简化编程**：C 库函数封装了复杂的系统调用和操作系统功能，使得开发者不必直接调用底层系统调用，只需调用更高层次的函数。
- **跨平台兼容**：C 库函数通常设计为跨平台的，它们的接口和行为在不同操作系统中保持一致。这使得使用 C 库函数编写的代码更具可移植性。
- **提供缓冲和优化**：C 库函数通常带有缓冲机制。例如，标准输入输出函数（如 `printf()`、`fread()` 等）通过缓冲区来提高 I/O 性能。

## 2.3 系统调用与 C 库函数的对比

经过前面两个内容的学习，我们现在已经对系统调用和C语言库函数有了一个基本的认识，从介绍中可以发现，一些系统调用和C语言库函数实现的功能是相似的，那他们分别适用于哪些情况呢，除此之外，二者又有哪些地方的不同呢？具体可以来看这个表格

|            | 系统调用                                           | C 库函数                                       |
| ---------- | -------------------------------------------------- | ---------------------------------------------- |
| 抽象层次   | 底层接口，直接与操作系统内核交互                   | 高层封装，简化开发者操作                       |
| 性能与效率 | 高效，但涉及用户模式和内核模式的上下文切换         | 通过缓冲优化性能，可能带来额外开销             |
| 使用场景   | 与操作系统核心功能直接交互，精确控制硬件、设备文件 | 处理常见文件、字符串、数学操作，跨平台应用开发 |
| 可移植性   | 依赖于特定操作系统，较难移植                       | 跨平台标准接口，具有更好的可移植性             |
| 错误处理   | 基础错误处理，开发者需自行处理所有错误             | 提供高级错误处理机制，处理部分错误             |

系统调用和C库函数第一个不同点为抽象层次，系统调用是底层接口，它可以直接与操作系统内核交互。而C 库函数则是系统调用的高层封装，提供了便利接口，简化开发者的操作，从而使得编程更加方便。第二个性能与效率的不同，系统调用需要在用户模式和内核模式之间切换，这种上下文切换会带来一定的开销，但是由于没有中间缓冲或抽象层，系统调用通常适用于高性能场景。而C 库函数通过缓冲和其他机制优化了性能，尤其是在文件 I/O 操作中，C 库函数通过减少系统调用的频率来提高效率。但是，这种缓冲机制可能在处理设备文件或特殊场景时带来问题，然后来看使用场景，系统调用适用于与操作系统核心功能直接交互，或需要精确控制硬件、设备文件的场景。例如，与设备文件交互、内存映射等任务必须使用系统调用，而C 库函数适合处理常见的文件、字符串、数学等操作，开发者通常使用库函数完成绝大多数常规编程任务。然后是可移植性，系统调用依赖于特定操作系统，因此在不同的操作系统之间，系统调用的接口和行为可能有所不同。直接使用系统调用的代码通常较难移植到不同的平台。而C 库函数则是跨平台的标准接口，它们在不同平台上有一致的行为，因此使用 C 库函数的代码具有更好的可移植性。

至此，关于系统调用和C库函数的对比就讲解完成了，现在来对本节视频讲解的内容进行回顾，首先在第一部分中介绍了系统调用，系统调用是由操作系统内核提供的一组API接口，允许用户空间的应用程序与内核进行直接交互。是用户程序进入内核模式的唯一途径，然后列举了部分常用的系统调用，例如open、read、write、close，然后在第二部分中介绍了C语言库函数，C 库函数是C 语言提供的一组标准化函数，C 库函数往往会对底层系统调用进行封装，从而提供更高层次的接口，开发者可以通过这些函数轻松实现日常开发任务，然后对常见的C库函数，例如输入输出函数、字符串操作函数等进行了介绍，最后在第三部分中对系统调用和C库函数进行了对比，这里重点关注点在使用常见，系统调用适用于与操作系统核心功能直接交互，或需要精确控制硬件、设备文件的场景。而C 库函数适合处理常见的文件、字符串、数学等操作，后面会对常用的系统调用进行讲解，而在进行真正的编程之前，首先需要搭建相应的编程环境，所以在下个小节中将会讲解学习环境的搭建。

| 编译器   | 将高级语言（如 C 语言）编写的源代码翻译成机器语言的工具，使得计算机能够理解并执行这些指令。 |

 ##  编译器的安装和使用
