# 1 初识系统编程

在Linux系统中，为了保障系统的安全和稳定，系统将内存划分为**用户空间**和**内核空间**。用户空间是应用程序运行的区域，普通程序无法直接访问内核空间中的资源。而内核空间则是操作系统核心功能运行的地方，负责管理硬件、内存、文件系统、进程、网络等关键资源，而在嵌入式领域，很多情况下我们都需要对相关底层硬件进行操作，例如对LED、GPIO、蜂鸣器、串口的控制，那要如何在用户空间中实现对上述硬件的控制呢？这就要用到Linux系统编程相关的知识了。

Linux系统编程是一种编程方式，使我们编写的程序能够直接与Linux系统“内核”进行交互。通过系统编程，程序能够通过受控的方式从用户空间请求内核空间的服务。这种请求通常是通过系统调用（system calls）实现的。系统调用提供了一系列接口，使得用户程序能够安全地让内核执行特定的操作，比如文件操作、进程控制、内存分配等。

例如，当程序需要打开一个文件时，它会通过`open()`系统调用向内核请求该文件的访问权限，而如果程序需要对一个文件进行读写时，则会分别使用`read` `write`系统调用告诉内核对该文件进行读取和写入。

除了系统调用之外，Linux还提供了**库函数**，库函数可以看作是对系统调用的封装，属于操作系统开发人员预先定义好的工具，可以帮助开发者更方便地与内核进行交互，简化系统调用的使用.

### 一、系统调用

#### 1. **什么是系统调用？**

系统调用（System Call）是操作系统提供的一组接口，允许用户空间的应用程序与内核直接交互。它们是用户程序进入内核模式的唯一途径，用于请求操作系统内核执行特定任务。系统调用可以访问受操作系统管理的底层资源，如文件系统、设备、进程、内存和网络。

#### 2. **常见的系统调用**

| 分类     | 系统调用   | 功能描述                                                   |
| -------- | ---------- | ---------------------------------------------------------- |
| 文件操作 | `open()`   | 打开文件，返回文件描述符。                                 |
|          | `read()`   | 从文件描述符读取数据。                                     |
|          | `write()`  | 向文件描述符写入数据。                                     |
|          | `close()`  | 关闭文件描述符。                                           |
| 内存管理 | `mmap()`   | 将文件或设备映射到进程的地址空间中，进行内存映射文件 I/O。 |
| 设备管理 | `ioctl()`  | 控制设备操作，提供更灵活的设备操作接口。                   |
| 网络通信 | `socket()` | 创建一个网络通信的套接字，用于网络连接和数据传输。         |

#### 3. **系统调用的工作机制**

系统调用的工作机制大致如下：
- **用户程序发起调用**：用户程序通过调用标准库函数（如 `open()` 或 `write()`）发起系统调用请求。
- **切换模式**：操作系统通过陷阱指令（如 `syscall`）切换到内核模式。
- **内核处理请求**：操作系统查找对应的系统调用号，找到对应的内核函数，并执行所请求的操作。
- **返回结果**：内核执行完系统调用后，将结果返回给用户程序，并切换回用户模式，程序继续执行。

### 二、 C 库函数

#### 1. **什么是 C 库函数？**

C 库函数（C Library Functions）是 C 语言提供的一组标准化函数，用于执行常见的任务，如文件 I/O、字符串处理、数学运算、内存分配等。C 库函数往往对底层系统调用进行了封装，提供了更高层次的接口，开发者可以通过这些函数轻松实现日常开发任务。

#### 2. **C 库函数的作用**

- **简化编程**：C 库函数封装了复杂的系统调用和操作系统功能，使得开发者不必直接调用底层系统调用，只需调用更高层次的函数。
- **跨平台兼容**：C 库函数通常设计为跨平台的，它们的接口和行为在不同操作系统中保持一致。这使得使用 C 库函数编写的代码更具可移植性。
- **提供缓冲和优化**：C 库函数通常带有缓冲机制。例如，标准输入输出函数（如 `printf()`、`fread()` 等）通过缓冲区来提高 I/O 性能。

#### 3. **常见的 C 库函数**

| 类别           | 函数名     | 描述                           |
| -------------- | ---------- | ------------------------------ |
| 输入/输出函数  | `printf()` | 格式化输出到标准输出           |
|                | `scanf()`  | 格式化输入，从标准输入获取数据 |
|                | `fopen()`  | 打开一个文件，返回文件指针     |
|                | `fread()`  | 从文件中读取数据到内存中       |
|                | `fwrite()` | 将数据写入文件                 |
| 字符串操作函数 | `strlen()` | 计算字符串的长度               |
|                | `strcpy()` | 复制字符串                     |
|                | `strcmp()` | 比较两个字符串                 |
| 内存管理函数   | `malloc()` | 动态分配内存                   |
|                | `calloc()` | 动态分配并初始化内存           |
|                | `free()`   | 释放内存                       |
| 数学函数       | `sqrt()`   | 计算平方根                     |
|                | `pow()`    | 计算幂                         |
| 其他实用函数   | `time()`   | 获取当前时间                   |
|                | `rand()`   | 生成随机数                     |

这个表格清晰地列出了每个函数的类别和描述，方便您快速查找和使用。

#### 4. **C 库函数的工作机制**
C 库函数的机制通常是对系统调用进行封装，隐藏底层细节并提供更高级的功能：
- **封装系统调用**：例如，`fopen()` 是 C 库中的文件打开函数，底层它可能会调用 `open()` 系统调用，但它还会提供文件指针管理、缓冲等额外功能。
- **提供缓冲**：C 库的 I/O 函数通常带有缓冲机制，如 `printf()` 或 `fwrite()` 使用缓冲区来提高效率，减少直接 I/O 操作的频率。
- **跨平台支持**：C 库函数往往在不同平台上都有一致的接口，这使得使用库函数编写的代码具备良好的跨平台性。

### 三、系统调用与 C 库函数的对比

|            | 系统调用                                           | C 库函数                                       |
| ---------- | -------------------------------------------------- | ---------------------------------------------- |
| 抽象层次   | 底层接口，直接与操作系统内核交互                   | 高层封装，简化开发者操作                       |
| 性能与效率 | 高效，但涉及用户模式和内核模式的上下文切换         | 通过缓冲优化性能，可能带来额外开销             |
| 使用场景   | 与操作系统核心功能直接交互，精确控制硬件、设备文件 | 处理常见文件、字符串、数学操作，跨平台应用开发 |
| 可移植性   | 依赖于特定操作系统，较难移植                       | 跨平台标准接口，具有更好的可移植性             |
| 错误处理   | 基础错误处理，开发者需自行处理所有错误             | 提供高级错误处理机制，处理部分错误             |

#### 1. **抽象层次**

- **系统调用**：系统调用是底层接口，它直接与操作系统内核交互。开发者使用系统调用可以直接控制文件、内存、设备等资源，但同时需要处理复杂的底层细节，如文件描述符、内存地址等。
- **C 库函数**：C 库函数则是系统调用的高层封装，简化了开发者的操作。库函数为常见任务提供了便利接口，并且通常附加了缓冲、错误处理等额外功能，使得编程更加方便。

#### 2. **性能与效率**

- **系统调用**：系统调用的效率取决于其直接与内核交互的特性。系统调用需要在用户模式和内核模式之间切换，这种上下文切换会带来一定的开销，但由于没有中间缓冲或抽象层，系统调用通常适用于高性能场景。
- **C 库函数**：C 库函数通过缓冲和其他机制优化了性能，尤其是在文件 I/O 操作中，C 库函数通过减少系统调用的频率来提高效率。然而，这种缓冲机制可能在处理设备文件或特殊场景时带来问题。

#### 3. **使用场景**

- **系统调用**：当开发者需要与操作系统核心功能直接交互，或需要精确控制硬件、设备文件时，系统调用是必不可少的。例如，与设备文件交互、内存映射等任务必须使用系统调用。
- **C 库函数**：C 库函数适合处理常见的文件、字符串、数学等操作，尤其适用于跨平台应用开发。开发者通常使用库函数完成绝大多数常规编程任务。

#### 4. **可移植性**

- **系统调用**：系统调用依赖于特定操作系统，因此在不同的操作系统之间，系统调用的接口和行为可能有所不同。直接使用系统调用的代码通常较难移植到不同的平台。
- **C 库函数**：C 库函数则是跨平台的标准接口，它们在不同平台上有一致的行为，因此使用 C 库函数的代码具有更好的可移植性。

#### 5. **错误处理**

- **系统调用**：系统调用的错误处理相对基础，通常通过返回负数并设置 `errno` 来表示错误。开发者需要自己处理所有的错误和异常情况。
- **C 库函数**：C 库函数往往为开发者处理了一部分错误，并且提供了更高级的错误处理机制。例如，`fopen()` 返回 `NULL` 表示文件打开失败，并且可以使用 `perror()` 打印错误信息。

# 4 编译器的安装和使用

| 概念     | **描述**                                                     |
| -------- | ------------------------------------------------------------ |
| C 语言   | 高级编程语言，代码是为人类编写和理解的，具有较强的可读性。   |
| 机器语言 | 低级语言，计算机能够直接执行的语言，指令以二进制形式表示。   |
| 编译器   | 将高级语言（如 C 语言）编写的源代码翻译成机器语言的工具，使得计算机能够理解并执行这些指令。 |

# 5.常用文件io操作



系统编程中的**I/O 操作**是 Linux/Unix 系统中的重要内容。它们提供了与文件、设备、套接字等交互的能力。在 Linux 系统编程中，最常用的 I/O 操作包括：`open()`、`read()`、`write()`、`close()` 和 `lseek()`。这些操作通过系统调用直接与操作系统的内核交互，以执行文件和设备的 I/O 操作。

下面是对每个系统调用的**详细讲解**，包括它们的作用、用法和注意事项。

## 1 文件描述符

### 优化后的描述：

**文件描述符**（File Descriptor, FD）是 Linux 系统中用于表示已打开的文件、设备、管道、套接字等 I/O 资源的**抽象标识符**。它是操作系统为每个进程分配的唯一整数标识，用于管理和操作打开的 I/O 资源。在 Linux 中，文件描述符通常由 `int` 类型表示，所有与 I/O 相关的操作（如读写文件、关闭文件）都依赖文件描述符来进行。

每当进程打开一个文件或 I/O 资源时，系统会分配一个文件描述符，该文件描述符将在后续的 I/O 操作中作为引用使用。由于每个文件占用一定的内存资源，Linux 系统限制了每个进程可以同时打开的文件数量。默认情况下，文件描述符的值范围为 0 到 1023（即最多允许 1024 个打开的文件）。如果进程打开的文件数超过限制，内核会向进程发送警告信号，导致进程终止。

#### 文件描述符的常见使用：

1. **打开文件**：使用 `open()` 系统调用打开文件时，系统会返回一个文件描述符，该文件描述符唯一标识该文件，后续的文件操作（如读写）都需要该文件描述符作为参数。

2. **读写文件**：一旦文件打开，可以通过 `read()` 和 `write()` 系统调用从文件中读取数据或向文件写入数据。读写操作都依赖文件描述符来指定文件。

3. **关闭文件**：在完成文件操作后，必须使用 `close()` 系统调用关闭文件，释放占用的文件描述符和系统资源。未及时关闭文件描述符可能导致资源泄漏。

---

Linux 系统中，预定义了三个**标准文件描述符**：

- **标准输入（stdin）**：文件描述符 `0`，默认指向用户的输入设备（通常是键盘）。
- **标准输出（stdout）**：文件描述符 `1`，默认指向输出设备（通常是终端显示器）。
- **标准错误（stderr）**：文件描述符 `2`，默认指向错误输出设备（通常也是终端显示器）。

---

文件描述符是 Linux 系统中进行 I/O 操作的基础机制，开发人员需要理解文件描述符的概念及其使用方法，以便编写高效、稳定的 Linux 应用程序。通过正确管理文件描述符，可以确保资源的合理分配和释放，避免不必要的资源浪费和程序崩溃。

## 2. 打开和关闭文件

在 Linux 系统中，`open()` 和 `close()` 是两个非常重要的系统调用，分别用于**打开**和**关闭**文件描述符。通过文件描述符，程序可以对文件、设备、管道、套接字等 I/O 资源进行操作。接下来将对这两个函数进行详细讲解。

#### 1. `open()` 函数的头文件和函数原型

要使用 `open()` 函数，需要包含以下头文件：

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
```

`open()` 函数有两个原型：

```c
int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);
```

- **返回值**：调用成功时，`open()` 返回文件描述符（`int` 类型），用于标识打开的文件。失败时返回 `-1`，并设置 `errno` 表示错误原因。

##### 参数说明：

| 参数名称   | 含义                                                         |
| ---------- | ------------------------------------------------------------ |
| `pathname` | 文件路径和文件名，用于标识要打开或创建的文件。               |
| `flags`    | 文件的打开方式，可以通过按位或 `|` 设置多个标志位。          |
| `mode`     | 文件权限（仅在 `O_CREAT` 标志使用时有效），用于指定文件的读写执行权限。 |

##### `flags` 参数详解

- **主参数**（互斥使用）：

| 标志       | 作用                   |
| ---------- | ---------------------- |
| `O_RDONLY` | 以只读模式打开文件     |
| `O_WRONLY` | 以只写模式打开文件     |
| `O_RDWR`   | 以可读可写模式打开文件 |

- **副参数**（可组合使用）：

| 标志         | 作用                                                |
| ------------ | --------------------------------------------------- |
| `O_CREAT`    | 如果文件不存在则创建文件。                          |
| `O_EXCL`     | 与 `O_CREAT` 一起使用，若文件存在则 `open()` 失败。 |
| `O_APPEND`   | 以追加模式打开文件。                                |
| `O_NONBLOCK` | 以非阻塞模式打开文件。                              |
| `O_DIRECT`   | 使用直接 I/O。                                      |
| `O_SYNC`     | 同步 I/O。                                          |
| `O_ASYNC`    | 使文件读写时产生信号，适用于终端或套接字。          |
| `O_TRUNC`    | 如果文件存在，将文件长度截断为 0。                  |

##### `mode` 参数详解

`mode` 参数用于指定文件的权限，通常在 `O_CREAT` 标志位启用时使用。权限使用八进制数表示：

| 字母 | 权限     | 二进制 | 十进制 |
| ---- | -------- | ------ | ------ |
| `r`  | 读权限   | 100    | 4      |
| `w`  | 写权限   | 010    | 2      |
| `x`  | 执行权限 | 001    | 1      |

#### 2. `close()` 函数的头文件和函数原型

`close()` 函数用于关闭打开的文件描述符，释放系统资源。需要包含以下头文件：

```c
#include <unistd.h>
```

`close()` 函数的原型：

```c
int close(int fd);
```

- **返回值**：调用成功时返回 `0`，失败时返回 `-1`，并设置 `errno`。

##### 参数说明：

| 参数名称 | 含义                               |
| -------- | ---------------------------------- |
| `fd`     | 文件描述符，标识要关闭的文件或资源 |

在使用 `close()` 函数时，需要确保文件描述符已被成功打开。通过及时关闭文件描述符，能够释放占用的系统资源，防止资源泄漏。

#### 3. 实验：使用 `open()` 和 `close()` 函数

5. 使用 `ls` 命令检查 `test` 文件是否已成功创建：

   ```bash
   ls
   ```

至此，`open()` 和 `close()` 的实验已完成。

## 3.读文件

`read()` 函数用于从文件或设备中读取数据，是 Linux 系统编程中常用的 I/O 操作。它的功能较为单一，但非常重要，可以通过它从文件、管道、套接字等读取数据。以下是 `read()` 函数的详细讲解及实验示例。

#### 1. `read()` 函数的头文件和函数原型

要使用 `read()` 函数，需要包含以下头文件：

```c
#include <unistd.h>
```

`read()` 函数的原型如下：

```c
ssize_t read(int fd, void *buf, size_t count);
```

- **返回值**：`read()` 函数返回读取到的字节数：
  - **大于 0**：表示实际读取到的字节数。
  - **等于 0**：在阻塞模式下表示已到达文件末尾（EOF），没有更多数据可读。
  - **等于 -1**：表示读取出错，并设置 `errno` 值以标识具体错误原因。

#### 2. `read()` 函数的参数

| 参数名称 | 含义                                       |
| -------- | ------------------------------------------ |
| `fd`     | 文件描述符，指定从哪个文件或设备读取数据。 |
| `buf`    | 缓冲区，用于存储读取到的数据。             |
| `count`  | 要读取的字节数，即最多读取多少字节的数据。 |

`read()` 函数将尝试从文件描述符 `fd` 指定的文件中读取最多 `count` 个字节的数据，存储在 `buf` 中，并返回读取到的字节数。

---

#### 3. 实验：使用 `read()` 函数读取文件内容

**实验要求**：从 `test` 文件中读取数据，并打印读取的文件内容和返回的字节数。

1. 在终端中创建一个名为 `demo04_read.c` 的文件：

   ```bash
   vim demo04_read.c
   ```

2. 在文件中添加以下代码：

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main(int argc, char *argv[])
{
    int fd; // 定义文件描述符
    char buf[32] = {0}; // 定义缓冲区，用于存储读取到的数据
    ssize_t ret; // 定义读取到的字节数

    // 以 O_CREAT | O_RDWR 的方式打开文件，权限为 0666，如果文件不存在则创建
    fd = open("test", O_CREAT | O_RDWR, 0666);
    if (fd < 0) // 打开文件失败
    {
        perror("open is error\n"); // 输出错误信息
        return -1; // 返回错误码 -1
    }

    printf("fd is %d\n", fd); // 输出文件描述符

    // 读取文件内容到缓冲区中，最多读取 32 个字节
    ret = read(fd, buf, 32);
    if (ret < 0) // 读取文件失败
    {
        perror("read is error\n"); // 输出错误信息
        return -2; // 返回错误码 -2
    }

    printf("buf is %s\n", buf); // 输出读取到的内容
    printf("ret is %ld\n", ret); // 输出读取到的字节数

    close(fd); // 关闭文件
    return 0; // 返回正常退出码 0
}
```

- **解释**：
  - 该代码通过 `open()` 函数打开名为 `test` 的文件，如果文件不存在则创建它，并设置权限为 `0666`（即所有用户都可以读写该文件）。
  - 使用 `read()` 函数从文件中读取最多 32 个字节的数据，并存储到缓冲区 `buf[32]` 中。
  - 程序会打印读取到的数据内容和实际读取的字节数，最后关闭文件描述符。

3. 保存并退出后，使用以下命令进行编译：

   ```bash
   gcc -o demo04_read demo04_read.c
   ```

4. 运行程序：

   ```bash
   ./demo04_read
   ```

- 如果 `test` 文件为空，程序的输出结果将是：
  - `buf` 为空字符串，因为没有数据可读。
  - `ret` 为 `0`，表示已到达文件末尾（EOF）。

5. 接下来，编辑 `test` 文件并添加一些内容，例如 `hello world`：

   ```bash
   echo "hello world" > test
   ```

6. 再次运行程序：

   ```bash
   ./demo04_read
   ```

- 运行成功后，程序将输出：
  - `buf is hello world`，表示成功读取到文件中的内容。
  - `ret is 12`，表示实际读取到 12 个字节（包括字符串中的空格）。

`read()` 函数是用于从文件或设备中读取数据的系统调用。它将数据存储在指定的缓冲区中，并返回实际读取到的字节数。在实验中，通过 `read()` 函数读取文件内容并打印，进一步理解了其在文件操作中的重要作用。若文件为空，`read()` 返回 `0`，表示已到达文件末尾；若读取失败，则返回 `-1` 并设置 `errno`。至此，关于 `read()` 函数的实验已完成。

## 4.写文件

`write()` 函数用于将数据写入文件或设备，是 Linux 系统中常见的 I/O 操作之一。其功能较为简单，通过指定的文件描述符将缓冲区的数据写入到指定的文件或设备中。

#### 1. `write()` 函数的头文件和函数原型

要使用 `write()` 函数，需要包含以下头文件：

```c
#include <unistd.h>
```

`write()` 函数的原型如下：

```c
ssize_t write(int fd, const void *buf, size_t count);
```

- **返回值**：`write()` 函数的返回值为写入的字节数：
  - **大于或等于 0**：表示写入成功，并返回实际写入的字节数。
  - **等于 -1**：表示写入失败，并设置 `errno` 来指示错误原因。

#### 2. `write()` 函数的参数

| 参数名称 | 含义                                     |
| -------- | ---------------------------------------- |
| `fd`     | 文件描述符，指定要写入数据的文件或设备。 |
| `buf`    | 缓冲区，存储要写入的数据。               |
| `count`  | 写入的字节数，表示要写入多少字节的数据。 |

## 5 lseek

`lseek()` 函数用于设置文件读写指针的位置。该函数允许我们在文件中移动读写指针，以便从指定位置读取或写入数据。下面是 `lseek()` 函数的详细说明。

#### 1. `lseek()` 函数的头文件和函数原型

要使用 `lseek()` 函数，需要包含以下头文件：

```c
#include <sys/types.h>
#include <unistd.h>
```

`lseek()` 函数的原型如下：

```c
off_t lseek(int fd, off_t offset, int whence);
```

- **返回值**：`lseek()` 执行成功时返回当前的文件偏移量，失败时返回 `-1` 并设置 `errno` 表示具体错误原因。

#### 2. `lseek()` 函数的参数

| 参数名称 | 含义                                                         |
| -------- | ------------------------------------------------------------ |
| `fd`     | 文件描述符，指定要操作的文件。                               |
| `offset` | 偏移量，单位是字节。可以为正数或负数，表示从 `whence` 指定的位置向前或向后移动。 |
| `whence` | 当前位置的基点，指定偏移量相对于文件的哪个位置进行计算。     |

#### 3. `whence` 参数的取值

| 值         | 含义                               |
| ---------- | ---------------------------------- |
| `SEEK_SET` | 偏移量相对于文件的开头计算。       |
| `SEEK_CUR` | 偏移量相对于当前文件指针位置计算。 |
| `SEEK_END` | 偏移量相对于文件末尾计算。         |

#### 4. 常见使用示例

- **将文件指针设置到偏移量为 5**：  

  ```c
  lseek(fd, 5, SEEK_SET);
  ```

- **将文件指针设置到文件末尾**：  

  ```c
  lseek(fd, 0, SEEK_END);
  ```

- **获取当前文件指针位置**：  

  ```c
  lseek(fd, 0, SEEK_CUR);
  ```

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main(int argc, char *argv[])
{
    int fd;                         // 文件描述符
    char buf[32] = {0};             // 读写缓冲区
    ssize_t ret;                    // 读取的字节数

    // 打开或创建文件 "test"，以读写方式打开
    fd = open("test", O_CREAT | O_RDWR, 0666);
    if (fd < 0)                     // 打开文件失败
    {
        perror("open is error\n");  // 打印错误信息
        return -1;                  // 退出程序
    }

    // 读取文件的前 32 个字节
    ret = read(fd, buf, 32);
    if (ret < 0)                    // 读取失败
    {
        perror("read is error\n");  // 打印错误信息
        return -2;                  // 退出程序
    }
    printf("buf is %s\n", buf);     // 打印读取到的字符串
    printf("ret is %ld\n", ret);    // 打印读取到的字节数

    // 使用 lseek 将文件指针移动到偏移量为 5 的位置
    lseek(fd, 5, SEEK_SET);

    // 再次读取文件内容
    ret = read(fd, buf, 32);
    printf("buf is %s\n", buf);     // 打印偏移后读取到的字符串
    printf("ret is %ld\n", ret);    // 打印读取到的字节数

    close(fd);                      // 关闭文件
    return 0;                       // 程序正常结束
}
```

`lseek()` 函数用于调整文件的读写指针位置，使得可以从文件的任意位置进行读取或写入。在实验中，通过 `lseek()` 将文件指针从文件开头移动到偏移量 `5` 处，并对比了两次 `read()` 的结果，验证了文件指针的移动效果。理解并合理使用 `lseek()`，可以实现文件的随机访问操作，灵活处理文件读写需求。至此，关于 `lseek()` 函数的实验已经完成。

# 6 文件属性的判断 - access() 函数详解

`access()` 函数用于检查当前进程对指定文件的访问权限。它可以检查文件或目录是否存在以及是否具有读、写、执行权限。

##### 头文件和函数原型

```c
#include <unistd.h>

int access(const char *pathname, int mode);
```

##### 返回值

- **返回 0**：表示文件存在且具有相应的权限。
- **返回 -1**：表示文件不存在或没有相应的权限，并设置 `errno` 表示错误原因。

##### `access()` 函数参数含义

| 参数名称   | 含义                               |
| ---------- | ---------------------------------- |
| `pathname` | 文件或目录的路径名                 |
| `mode`     | 要检查的权限模式，使用以下取值之一 |

##### `mode` 参数取值

| 取值   | 含义                     |
| ------ | ------------------------ |
| `F_OK` | 检查文件或目录是否存在   |
| `R_OK` | 检查文件或目录是否可读   |
| `W_OK` | 检查文件或目录是否可写   |
| `X_OK` | 检查文件或目录是否可执行 |

---

### 实验：使用 `access()` 检查文件权限

#### 实验要求

```c
{

    ret = access("./test", F_OK);  // 检查文件是否存在
    if (-1 == ret)  // 文件不存在
    {
        printf("file does not exist.\n");
        return -1;
    }
    else
        printf("file exist\n");  // 文件存在

    // 检查文件读权限
    ret = access("./test", R_OK);
    if (!ret)
        printf("是否有读权限: 有\n");
    else
        printf("是否有读权限: 无\n");
```

# led 应用编程

本章节将通过系统编程知识，使用应用程序对开发板上的 LED 进行实际控制。

####  应用层控制硬件的两种方式

在 Linux 操作系统中，应用层可以通过以下两种方式控制硬件设备：

1. **通过 `/dev` 目录下的设备节点控制硬件设备**：
   - 设备节点是 Linux 系统中的一种虚拟文件，代表硬件设备或驱动程序，为用户提供标准化的接口。每个设备节点都对应一个硬件设备或设备驱动程序，用户可以像操作普通文件一样，通过 `open()`、`read()`、`write()`、`close()` 等函数对设备节点进行操作，从而实现硬件设备的访问和控制。
   - 示例：`/dev/sda` 代表第一个硬盘，`/dev/ttyS0` 代表第一个串口设备。

2. **通过 `sysfs` 文件系统控制硬件设备**：
   - `sysfs` 是 Linux 内核提供的虚拟文件系统，向用户空间公开内核对象的属性和状态。应用程序可以通过 `sysfs` 读取和修改内核对象的属性，进而控制硬件设备。
   - 在 `sysfs` 中，每个硬件设备都有一个目录，目录下的文件包含设备的状态信息和控制接口。对于简单设备（如 LED、GPIO），可以通过 `sysfs` 控制。

   **`sysfs` 的特点：**

   | 特点     | 描述                                                  |
   | -------- | ----------------------------------------------------- |
   | 可读写   | `sysfs` 中大多数文件可读写，允许读取和修改设备属性    |
   | 动态更新 | 随着设备状态的变化，文件会动态更新                    |
   | 轻量级   | 资源消耗少，适合快速读取和修改内核对象属性            |
   | 易于使用 | 用户空间可以使用标准文件操作来读取、写入 `sysfs` 文件 |
   | 可扩展性 | `sysfs` 可以随着内核对象的增加动态扩展                |

# gpio应用编程

在本章节中，我们将使用 GPIO（通用输入输出）接口来控制开发板上的 LED。通过对设备树的修改和系统编程，掌握如何使用 GPIO 控制硬件。

#### 修改设备树

为了使用 GPIO 控制 LED，需要注释掉设备树中与 LED 节点相关的内容。我们已将修改后的内核设备树放置在目录 `iTOP-3568开发板/03_【iTOP-RK3568开发板】指南教程/01_系统编程配套程序/62` 下。你可以根据手册“4.2 单独烧写 Linux 固件”中的指引，将提供的 `boot.img` 文件烧录到开发板中，完成后即可继续本章节的学习。

###  应用层如何操控 GPIO

与 LED 设备一样，GPIO 也是通过 `sysfs` 文件系统进行控制。首先进入 `/sys/class/gpio` 目录，使用以下命令：
```bash
cd /sys/class/gpio
```
你将看到 `export`、`unexport` 文件和多个 `gpiochipX` 目录。`gpiochipX` 目录表示当前 SoC 中的 GPIO 控制器（iTOP-RK3568 开发板一共有 5 个 GPIO 控制器），分别对应 `gpiochip0`、`gpiochip32`、`gpiochip64`、`gpiochip96`、`gpiochip128`。每个 `gpiochipX` 目录用来管理一组 GPIO。

#### GPIO 目录文件说明

每个 `gpiochipX` 目录包含以下重要文件：

| 文件名  | 描述                                           |
| ------- | ---------------------------------------------- |
| `base`  | 当前 GPIO 控制器管理的 GPIO 引脚的最小编号     |
| `label` | GPIO 控制器标签（例如 `gpiochip0` 对应的标签） |
| `ngpio` | 当前控制器所管理的 GPIO 引脚数量               |

#### GPIO 引脚计算
每个 GPIO 引脚都有一个编号。引脚编号的计算公式如下：
- **pin = bank * 32 + number**
  - `bank` 表示组号，`number` 表示小组编号。

例如，对于 LED9 用户灯的 `GPIO0_PB7` 引脚：
- **bank = 0**
- **group = 1** (因为 `B = 1`)
- **X = 7**

计算得出：
- **number = group * 8 + X = 1 * 8 + 7 = 15**
- **pin = bank * 32 + number = 0 * 32 + 15 = 15**

#### 导出 GPIO 引脚

使用 `export` 文件将 GPIO 引脚导出：
```bash
echo 15 > export
```
导出成功后，在 `/sys/class/gpio` 目录下会生成 `gpio15` 文件夹，用于管理 GPIO 引脚。

#### 删除 GPIO 引脚

使用 `unexport` 文件删除导出的 GPIO 引脚：
```bash
echo 15 > unexport
```
删除后，`gpio15` 文件夹会消失。

#### GPIO 控制文件

导出 GPIO 引脚后，可以通过以下文件对 GPIO 引脚进行控制：

| 文件名       | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| `direction`  | 配置 GPIO 为输入或输出模式，值为 `in` 或 `out`。             |
| `active_low` | 控制极性：`0` 表示正常极性，`1` 表示反向极性。               |
| `value`      | 读取或写入 GPIO 电平值，`1` 为高电平，`0` 为低电平。         |
| `edge`       | 控制中断触发模式，值为 `none`、`rising`、`falling` 或 `both`。 |

# 输入设备应用编程

###  输入设备介绍

输入设备是指可以接收用户输入的设备，例如键盘、鼠标、触摸屏等。这些设备通过 Linux 内核的输入子系统与应用程序进行交互。输入子系统是一个内核模块，负责管理输入设备与应用程序之间的通信。

在 Linux 系统中，每个输入设备都表示为一个字符设备文件，这些文件通常位于 `/dev/input` 目录下。如下图所示：

当输入设备产生一个事件时，它将事件数据写入该设备的输入队列中。应用程序通过读取该设备的输入队列获取事件信息，事件信息包括按键按下、松开、鼠标移动、触摸屏触摸等。

###  Input 子系统

Input 子系统负责管理输入设备，提供了一组用于读取和控制输入设备的 API 和机制，包括设备注册、事件读取和处理等。在 Linux 系统中，所有输入设备都被视为事件源，如键盘、鼠标、触摸屏等设备。这些设备产生各种事件，如按键按下/松开、鼠标移动、触摸屏触摸等。Input 子系统将这些事件收集并传递给用户空间应用程序进行处理。

**Input 子系统的核心部分是 Input 驱动程序**，它负责与硬件通信，读取输入事件并将其转换为标准格式的输入事件，并将其发送到内核的 Input 子系统。每个输入设备需要有相应的驱动程序管理和处理。例如，USB 鼠标需要加载 HID 驱动程序，触摸屏则需要加载相应的触摸屏驱动。

在用户空间，可以通过读取 `/dev/input/eventX` 文件（其中 X 是输入设备编号）获取输入事件。如下图所示：

###  读取输入设备

要读取某个输入设备的数据，流程如下：

| 步骤 | 描述                                                         |
| ---- | ------------------------------------------------------------ |
| 1    | 应用程序打开 `/dev/input/eventX` 设备文件，X 是设备编号。    |
| 2    | 应用程序使用 `read` 函数从设备文件中读取数据。如果没有数据可读，`read` 将阻塞，直到有数据可用。 |
| 3    | 读取的数据是一个 `struct input_event` 结构体，包含事件类型、事件码、事件值。事件类型表明事件的分类，如按键事件、鼠标事件；事件码表明具体事件，如按键按下/松开；事件值则表示事件的状态（如按键按下/释放）。 |
| 4    | 应用程序处理事件数据，根据事件类型和码执行相应操作。         |
| 5    | 使用完输入设备后，调用 `close` 关闭设备文件。                |

每次 `read` 操作都会获取一个 `struct input_event` 结构体类型的数据。该结构体定义在 `<linux/input.h>` 头文件中：

```c
struct input_event {
    struct timeval time;   // 事件发生的时间
    __u16 type;            // 事件类型
    __u16 code;            // 具体事件
    __s32 value;           // 事件取值
};
```

#### 结构体字段详解

- **type**：表示事件类型，如按键、鼠标移动、触摸屏事件等。常用的事件类型如下：
  
  ```c
  #define EV_SYN    0x00  // 同步事件
  #define EV_KEY    0x01  // 按键事件
  #define EV_REL    0x02  // 相对位移事件（鼠标移动）
  #define EV_ABS    0x03  // 绝对位移事件（触摸屏）
  ```

- **code**：表示该类型事件中的具体事件。例如，对于按键事件（`EV_KEY`），`code` 表示哪个键被按下。常用按键码如下：

  ```c
  #define KEY_ESC  1  // ESC 键
  #define KEY_1    2  // 数字键 1
  #define KEY_2    3  // 数字键 2
  ```

- **value**：表示事件的状态或取值。在按键事件中，`value` 为 1 表示按下，为 0 表示松开；在位移事件中，`value` 表示鼠标或触摸屏的位移或坐标值。

# 串口应用编程

###  串口介绍

串口是嵌入式开发中最常用的外设之一，通过串口可以调试程序、连接外设（如串口打印机、蓝牙、Wi-Fi、GPS、GPRS 等等）。串口通信通过串行传输数据，避免了多条线路的干扰和延迟问题。

#### 1. 数据传输方式
串口通信使用串行传输，通过两条数据线（TXD，RXD）和一条地线（GND）进行数据传输。

#### 2. 数据格式
串口通信使用异步传输，将数据分成固定大小的数据帧。每个数据帧由起始位、数据位、校验位和停止位组成。

#### 3. 波特率
波特率表示每秒传输的二进制位数，常见的波特率有 9600bps、115200bps 等。

#### 4. 流控制
流控制确保数据传输的准确性，分为硬件流控制（通过 RTS 和 CTS）和软件流控制（通过 XON 和 XOFF）。

#### 5. 错误检测
串口通信可能出现校验错误、帧错误等问题，使用校验位和缓冲区可检测和避免这些错误。

#### 6. 串口编程
在 Linux 系统中，串口通过文件 I/O 和 `ioctl` 操作进行编程，控制流、读取和设置串口参数。

###  串口设备节点介绍

在嵌入式设备中，串口设备通常以字符设备节点的形式出现，位于 `/dev` 目录下。常见的设备节点以 `tty` 开头：

- `/dev/ttyX`：本地终端设备节点。
- `/dev/ttyFIQ0`：iTOP-3568 开发板的 UART2 调试终端。
- `/dev/ttyS4`、`/dev/ttyS7`、`/dev/ttyS9`：其他 UART 端口。
- `/dev/ttyGS0`：USB 虚拟串口。

###  串口的使用步骤

串口使用的基本步骤如下：

1. 打开串口设备文件 (`open`)
2. 获取当前串口配置 (`tcgetattr`)
3. 修改串口配置 (`tcsetattr`)
4. 设置新的配置 (`tcsetattr`)
5. 读写串口数据 (`read` / `write`)
6. 关闭串口设备文件 (`close`)

#### 获取当前串口配置
使用 `tcgetattr` 函数获取串口配置，包括波特率、数据位、停止位、校验位等。

#### 修改串口配置
使用 `tcsetattr` 函数设置新的配置，包括数据位、停止位、校验位等。

###  串口模式设置

#### 1 输入模式

输入模式控制接收的数据在传递给应用程序之前的处理方式。可用的标志如 `IGNBRK`（忽略输入终止条件）、`INPCK`（启用奇偶校验）等。

#### 2 输出模式

输出模式控制发送字符的处理方式，例如 `OPOST`（启用输出处理）、`ONLCR`（将换行符转换为回车符）。

#### 3 控制模式

控制模式主要用于设置串口的硬件特性，包括波特率、数据位、校验位、停止位等。常见波特率有 `B9600`、`B115200` 等。

#### 4 本地模式

本地模式控制终端数据的处理方式，例如 `ICANON`（启用规范模式）。

#### 5 特殊控制字符

特殊控制字符（如 `Ctrl+C`、`Ctrl+Z`）通过 `c_cc` 数组定义，用于处理终端的特殊行为。

### 串口配置

#### 1 设置波特率

使用 `cfsetspeed` 函数设置波特率。常用波特率有 `B9600`、`B115200` 等。

#### 2 设置数据位

通过修改 `c_cflag` 设置数据位数，常见的有 7 位和 8 位。

#### 3 设置奇偶校验

通过 `PARENB` 和 `INPCK` 设置奇偶校验。可以选择奇校验、偶校验或无校验。

#### 4 设置停止位

通过 `CSTOPB` 设置停止位，1 位或 2 位停止位。

### 6 串口实验

#### 实验要求

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <fcntl.h>
#include <termios.h>
#include <string.h>

#define BUFFER_SIZE 256

void set_serial_port(int fd, int baud_rate, int data_bits, int stop_bits, int parity) {
    struct termios options;
    tcgetattr(fd, &options);
    options.c_cflag &= ~CSIZE;
    options.c_lflag &= ~(ICANON | ECHO | ECHOE | ISIG);
    options.c_oflag &= ~OPOST;
    options.c_iflag &= ~(IXON | IXOFF | IXANY);
    options.c_cc[VTIME] = 0;
    options.c_cc[VMIN] = 0;

    // 设置波特率
    switch (baud_rate) {
        case 115200:
            cfsetispeed(&options, B115200);
            cfsetospeed(&options, B115200);
            break;
        // 其他波特率设置
    }

    options.c_cflag |= CLOCAL | CREAD;
    options.c_cflag &= ~CRTSCTS;

    // 设置数据位
    if (data_bits == 8)
        options.c_cflag |= CS8;

    // 设置停止位
    if (stop_bits == 2)
        options.c_cflag |= CSTOPB;

    // 设置校验位
    if (parity == 'N')
        options.c_cflag &= ~PARENB;

    tcsetattr(fd, TCSANOW, &options);
}

int main(int argc, char *argv[]) {
    if (argc != 3) {
        printf("Usage: %s [serial device] [send|recv]\n", argv[0]);
        return -1;
    }

    int fd;
    char buffer[BUFFER_SIZE];

    if (strcmp(argv[2], "send") == 0) {
        fd = open(argv[1], O_WRONLY | O_NOCTTY | O_SYNC);
        if (fd < 0) {
            printf("Error opening serial device\n");
            return -1;
        }
        set_serial_port(fd, 115200, 8, 1, 'N');
        while (1) {
            printf("Enter message: ");
            fgets(buffer, BUFFER_SIZE, stdin);
            write(fd, buffer, strlen(buffer));
        }
    } else if (strcmp(argv[2], "recv") == 0) {
        fd = open(argv[1], O_RDONLY | O_NOCTTY | O_SYNC);
        if (fd < 0) {
            printf("Error opening serial device\n");
            return -1;
        }
        set_serial_port(fd, 115200, 8, 1, 'N');
        while (1) {
            int bytes_read = read(fd, buffer, BUFFER_SIZE);
            if (bytes_read > 0) {
                buffer[bytes_read] = '\0';
                printf("Received message: %s", buffer);
            }
        }
    } else {
        printf("Invalid option\n");
        return -1;
    }

    return 0;
}
```

### PWM 介绍

**PWM（脉冲宽度调制，Pulse Width Modulation）**是一种通过改变信号脉冲宽度来控制电路输出的方法。在 PWM 中，信号的周期保持不变，而脉冲宽度随时间变化，从而调节电路的输出功率或电压。

**PWM 的主要特点：**

1. **周期固定，占空比可变**：PWM 信号的周期不变，但脉冲宽度（占空比）可以调整，占空比的范围通常为 0 到 1。
2. **高效控制**：通过调整占空比，可以高效地控制电机转速、LED 亮度等，无需复杂的模拟电路。
3. **硬件与软件实现**：
   - **硬件 PWM**：由专用的 PWM 模块生成，通常具有固定频率和可配置的占空比。
   - **软件 PWM**：通过软件定时器和控制程序实现，占用更多的 CPU 资源，但灵活性更高。

**PWM 的应用场景：**

- **调节电机转速**
- **控制 LED 亮度**
- **驱动电磁阀**
- **音频信号生成**

1. **创建并编辑 `demo67_pwm.c` 文件**

   ```bash
   vim demo67_pwm.c
   ```

2. **添加以下代码**

   ```c
   #include <stdio.h>
   #include <sys/types.h>
   #include <sys/stat.h>
   #include <fcntl.h>
   #include <unistd.h>
   #include <string.h>

   #define PWM0_PATH "/sys/class/pwm/pwmchip0/pwm0"
   #define BUFFER_SIZE 100

   // PWM 控制函数，arg 为控制参数，val 为参数值
   int pwm_ctrl(const char *arg, const char *val) {
       char file_path[256];
       int fd, len, ret;

       // 拼接文件路径
       snprintf(file_path, sizeof(file_path), "%s/%s", PWM0_PATH, arg);

       // 打开文件
       fd = open(file_path, O_WRONLY);
       if (fd < 0) {
           printf("打开文件 %s 失败\n", file_path);
           return -1;
       }

       // 写入参数值
       len = strlen(val);
       ret = write(fd, val, len);
       if (ret < 0) {
           printf("写入文件 %s 失败\n", file_path);
           close(fd);
           return -1;
       }

       // 关闭文件
       close(fd);
       return 0;
   }

   int main() {
       int fd, ret;
       char buf[BUFFER_SIZE];

       // 检查 PWM0 是否已导出，不存在则导出
       if (access(PWM0_PATH, F_OK)) {
           fd = open("/sys/class/pwm/pwmchip0/export", O_WRONLY);
           if (fd < 0) {
               printf("打开文件 /sys/class/pwm/pwmchip0/export 失败\n");
               return -1;
           }

           // 导出 PWM0
           ret = write(fd, "0", 1);
           if (ret < 0) {
               printf("写入文件 /sys/class/pwm/pwmchip0/export 失败\n");
               close(fd);
               return -1;
           }

           close(fd);
       }

       // 设置 PWM 参数
       pwm_ctrl("period", "1000000");      // 设置周期为1,000,000纳秒（1毫秒）
       pwm_ctrl("duty_cycle", "500000");   // 设置占空比为500,000纳秒（50%）
       pwm_ctrl("enable", "1");            // 启用 PWM

       // 实现呼吸灯效果
       while (1) {
           // 增加占空比，LED 逐渐变亮
           for (int i = 0; i <= 1000000; i += 10000) {
               snprintf(buf, sizeof(buf), "%d", i);
               pwm_ctrl("duty_cycle", buf);
               usleep(5000); // 延时5毫秒
           }

           // 减少占空比，LED 逐渐变暗
           for (int i = 1000000; i >= 0; i -= 10000) {
               snprintf(buf, sizeof(buf), "%d", i);
               pwm_ctrl("duty_cycle", buf);
               usleep(5000); // 延时5毫秒
           }
       }

       return 0;
   }
   ```

   **代码说明：**

   - **pwm_ctrl 函数**：用于控制 PWM 参数，通过写入相应的 `sysfs` 文件实现。
   - **main 函数**：
     - 检查 PWM0 是否已导出，若未导出则通过写入 `export` 文件导出 PWM0。
     - 设置 PWM 的周期为 1 毫秒（1,000,000 纳秒），占空比为 50%（500,000 纳秒），并启用 PWM 输出。
     - 通过两个循环实现占空比的逐渐增加和减少，形成呼吸灯效果。

###  看门狗介绍

看门狗（Watchdog）是一种用于检测和处理系统异常的硬件或软件机制。它主要用于确保系统在出现异常（如程序崩溃、死锁、系统停滞等）时能够自动恢复。看门狗通常由一个计时器或电路组成，定期监控系统的运行状态。如果系统在规定时间内未能正确响应，看门狗将采取预设的处理措施，如重新启动系统、发出警报等。

在嵌入式系统中，看门狗扮演着非常重要的角色，因为它可以保障系统的稳定性和可靠性。RK3568 芯片集成了两个看门狗定时器：WDOG1 和 WDOG2。WDOG1 是一个常规的看门狗，支持产生中断和复位 CPU；而 WDOG2 则用于安全目的。

在 Linux 系统中，看门狗设备会注册为字符设备，并在 `/dev` 目录下生成相应的设备节点，如 `/dev/watchdog` 和 `/dev/watchdogX`。通过这些设备节点，用户可以直接控制看门狗设备。

### ioctl 函数

`ioctl()` 是 Unix-like 操作系统中用于设备控制的系统调用。它的主要作用是向设备驱动程序发送控制命令和参数，以便对设备进行配置或查询设备状态。该函数的原型如下：

```c
#include <sys/ioctl.h>
int ioctl(int fd, unsigned long request, ...);
```

- **fd**: 打开的设备文件描述符。
- **request**: 请求的命令码（通常由宏定义）。
- **...**: 可选的命令参数。

`ioctl()` 在设备驱动中通过不同的 `request` 参数来执行特定操作。在看门狗设备的控制中，主要用到一系列定义在 `<linux/watchdog.h>` 中的 `ioctl` 请求码。

###  看门狗的使用步骤

使用看门狗时，通常按以下步骤进行：

1. **打开看门狗设备文件**：使用 `open()` 函数打开 `/dev/watchdog` 设备，获取文件描述符。
2. **设置超时时间**：通过 `ioctl()` 函数使用 `WDIOC_SETTIMEOUT` 命令设置看门狗的超时时间。
3. **启动看门狗**：使用 `ioctl()` 函数调用 `WDIOC_SETOPTIONS` 命令启动看门狗。
4. **定时喂狗**：在超时时间内，使用 `WDIOC_KEEPALIVE` 命令定时喂狗，防止看门狗超时复位。
5. **停止看门狗**：在不需要看门狗时，调用 `WDIOC_SETOPTIONS` 命令关闭看门狗。
6. **关闭设备文件**：使用 `close()` 函数关闭看门狗设备。

#### 常用的 `ioctl` 宏定义

这些宏定义在 `<linux/watchdog.h>` 头文件中，不同的宏定义对应不同的设备操作：

- **WDIOC_SETTIMEOUT**：设置看门狗超时时间。
- **WDIOC_GETTIMEOUT**：获取当前的超时时间。
- **WDIOC_KEEPALIVE**：喂狗操作，防止看门狗复位。
- **WDIOC_SETOPTIONS**：启动或关闭看门狗。
  

例如，设置看门狗超时时间为 10 秒：

```c
unsigned int timeout = 10;
ioctl(fd, WDIOC_SETTIMEOUT, &timeout);
```

喂狗操作：

```c
ioctl(fd, WDIOC_KEEPALIVE, NULL);
```

# can设备应用编程

以下是关于 CAN 设备编程的整理优化：

---

### CAN 设备编程简介

CAN（Controller Area Network）是一种多主机局域网络协议，广泛应用于汽车、工业自动化等领域，用于在设备之间传输数据。CAN 设备通过发送和接收消息帧进行通信，具有高可靠性和实时性。

在 Linux 系统中，CAN 设备可以通过 SocketCAN 进行访问和控制。SocketCAN 是 Linux 中 CAN 总线的协议栈，它提供了一套类似于网络套接字的接口，使得开发者能够轻松地对 CAN 设备进行编程和调试。

---

### CAN 设备基本概念

在 CAN 总线中，通信的基本单位是消息帧。消息帧包含以下信息：

- **帧 ID**：标识消息的优先级和类型。
- **数据长度码（DLC）**：指示数据段的长度。
- **数据**：传输的数据段，长度为 0 到 8 字节。
- **校验位**：用于检测消息的传输错误。

CAN 总线的特点如下：
- **多主控制**：CAN 总线允许多个设备同时发送数据，依靠帧 ID 决定优先级。
- **差分信号传输**：通过两根信号线（CAN_H 和 CAN_L）进行通信，抗干扰能力强。
- **位定时同步**：各个节点使用同步位计时，确保消息同步。

---

### CAN 设备编程步骤

在 Linux 中，CAN 设备通常以字符设备形式呈现，可以通过 SocketCAN 接口进行访问。SocketCAN 通过套接字接口提供了对 CAN 总线的操作支持，使得 CAN 设备的编程与网络编程类似。

#### 1. 硬件设备检查

在操作 CAN 设备之前，首先需要检查系统中是否有 CAN 设备。可以使用以下命令列出系统中的网络接口，包括 CAN 接口：

```bash
ip link show
```

通常，CAN 设备的名称为 `can0`、`can1` 等。如果设备未启用，可以使用以下命令启用 CAN 设备：

```bash
sudo ip link set can0 up type can bitrate 500000
```

这会启用 `can0` 设备，并设置 CAN 的波特率为 500 kbps。

#### 2. 配置 CAN 设备

在启用 CAN 设备后，还可以使用 `ifconfig` 命令查看 CAN 设备的配置信息：

```bash
ifconfig can0
```

你可以通过 `ip link` 设置 CAN 设备的各种参数，例如：

- 设置 CAN 的模式为环回模式（用于测试）：
  
  ```bash
  sudo ip link set can0 up type can bitrate 500000 loopback on
  ```

- 关闭 CAN 设备：

  ```bash
  sudo ip link set can0 down
  ```

#### 3. 使用 SocketCAN 进行编程

SocketCAN 提供了一种标准的 Linux 网络套接字接口，用于与 CAN 设备通信。以下是使用 SocketCAN 进行 CAN 编程的步骤：

##### 3.1 打开 CAN 套接字

使用 `socket()` 函数创建一个 CAN 套接字：

```c
#include <linux/can.h>
#include <linux/can/raw.h>
#include <sys/socket.h>

int s = socket(PF_CAN, SOCK_RAW, CAN_RAW);
```

- `PF_CAN`：协议族，表示使用 CAN 协议。
- `SOCK_RAW`：套接字类型，表示使用原始套接字。
- `CAN_RAW`：表示 CAN 原始协议。

##### 3.2 绑定 CAN 设备

使用 `bind()` 函数将套接字绑定到具体的 CAN 设备（例如 `can0`）：

```c
struct sockaddr_can addr;
struct ifreq ifr;

strcpy(ifr.ifr_name, "can0");
ioctl(s, SIOCGIFINDEX, &ifr); // 获取 can0 的接口索引

addr.can_family = AF_CAN;
addr.can_ifindex = ifr.ifr_ifindex;

bind(s, (struct sockaddr *)&addr, sizeof(addr));
```

##### 3.3 发送 CAN 消息

创建 CAN 帧并使用 `write()` 函数发送：

```c
struct can_frame frame;
frame.can_id = 0x123;  // 设置 CAN ID
frame.can_dlc = 8;     // 设置数据长度为 8 字节
memcpy(frame.data, "ABCDEFGH", 8);  // 设置数据内容

write(s, &frame, sizeof(frame));  // 发送 CAN 帧
```

##### 3.4 接收 CAN 消息

使用 `read()` 函数从 CAN 设备读取数据帧：

```c
struct can_frame frame;
int nbytes = read(s, &frame, sizeof(struct can_frame));

if (nbytes > 0) {
    printf("Received CAN frame with ID 0x%X\n", frame.can_id);
    printf("Data: ");
    for (int i = 0; i < frame.can_dlc; i++) {
        printf("%02X ", frame.data[i]);
    }
    printf("\n");
}
```

##### 3.5 关闭套接字

操作完成后，关闭 CAN 套接字：

```c
close(s);
```

2. **编写 CAN 设备控制代码**

```c
#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <fcntl.h>
#include <linux/can.h>
#include <linux/can/raw.h>
#include <sys/ioctl.h>
#include <sys/socket.h>
#include <net/if.h>

int main() {
    int s;
    struct sockaddr_can addr;
    struct ifreq ifr;

    // 创建 CAN 套接字
    s = socket(PF_CAN, SOCK_RAW, CAN_RAW);
    if (s < 0) {
        perror("Socket");
        return 1;
    }

    // 绑定到 can0 设备
    strcpy(ifr.ifr_name, "can0");
    ioctl(s, SIOCGIFINDEX, &ifr);

    addr.can_family = AF_CAN;
    addr.can_ifindex = ifr.ifr_ifindex;

    if (bind(s, (struct sockaddr *)&addr, sizeof(addr)) < 0) {
        perror("Bind");
        return 1;
    }

    // 发送 CAN 帧
    struct can_frame frame;
    frame.can_id = 0x123;
    frame.can_dlc = 8;
    strcpy((char *)frame.data, "HelloCAN");

    if (write(s, &frame, sizeof(frame)) != sizeof(frame)) {
        perror("Write");
        return 1;
    }

    // 接收 CAN 帧
    int nbytes = read(s, &frame, sizeof(struct can_frame));
    if (nbytes > 0) {
        printf("Received CAN frame with ID 0x%X\n", frame.can_id);
        printf("Data: ");
        for (int i = 0; i < frame.can_dlc; i++) {
            printf("%02X ", frame.data[i]);
        }
        printf("\n");
    }

    // 关闭 CAN 套接字
    close(s);
    return 0;
}
```

# socker应用编程

###  Socket 介绍

Socket 是计算机网络编程中的一个重要概念，它提供了应用层与传输层之间的一种抽象接口，用于在网络中进行数据交换。Socket 允许程序员通过一种通用接口来使用底层传输协议（如 TCP 和 UDP）进行网络通信。

- **Socket API** 提供了创建、配置、连接、发送/接收数据、关闭等一系列网络操作函数。
- **流套接字（SOCK_STREAM）** 提供面向连接的可靠数据传输，主要使用 TCP 协议。
- **数据报套接字（SOCK_DGRAM）** 提供无连接、不可靠的数据传输，通常基于 UDP 协议。

#### 常见的 Socket 编程术语

- **IP 地址**：网络中标识主机的唯一地址。
- **端口号**：用于标识主机上特定应用程序的16位数字。
- **套接字**：网络连接的终端，包含 IP 地址和端口号。
- **服务器**：提供服务的主机。
- **客户端**：请求服务的主机。

#### Socket 编程模型

Socket 编程通常分为服务器端和客户端：
- **服务器端**：监听特定端口，等待客户端连接。可同时处理多个客户端。
- **客户端**：创建套接字并连接到服务器。连接建立后，客户端通过套接字与服务器通信。

---

###  Socket 编程步骤

Socket 编程主要包括以下步骤和对应的系统调用：

| 步骤          | 介绍                   | 系统调用                                                     |
| ------------- | ---------------------- | ------------------------------------------------------------ |
| 1. 创建套接字 | 创建一个套接字         | `socket(domain, type, protocol)`                             |
| 2. 绑定套接字 | 将套接字与 IP/端口绑定 | `bind(sockfd, addr, addrlen)`                                |
| 3. 监听连接   | 监听来自客户端的连接   | `listen(sockfd, backlog)`                                    |
| 4. 接受连接   | 接受客户端连接         | `accept(sockfd, addr, addrlen)`                              |
| 5. 数据收发   | 发送和接收数据         | `recv(sockfd, buf, len, flags)` / `send(sockfd, buf, len, flags)` |
| 6. 关闭套接字 | 关闭连接               | `close(sockfd)`                                              |

#### 1 创建套接字

创建套接字是构建网络应用的第一步。使用 `socket()` 系统调用，创建套接字并指定协议族、类型及协议：

```c
#include <sys/types.h>
#include <sys/socket.h>

int sockfd = socket(AF_INET, SOCK_STREAM, 0);
```

- **AF_INET**：使用 IPv4 协议。
- **SOCK_STREAM**：流套接字，使用 TCP 协议。

#### 2 绑定套接字

将创建的套接字与特定 IP 地址和端口号绑定，使用 `bind()` 系统调用：

```c
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

struct sockaddr_in servaddr;
memset(&servaddr, 0, sizeof(servaddr));
servaddr.sin_family = AF_INET;
servaddr.sin_addr.s_addr = htonl(INADDR_ANY);
servaddr.sin_port = htons(port);

if (bind(sockfd, (struct sockaddr *)&servaddr, sizeof(servaddr)) < 0) {
    perror("bind failed");
    exit(EXIT_FAILURE);
}
```

- **INADDR_ANY**：监听所有可用的网络接口。
- **htons()/htonl()**：将主机字节序转换为网络字节序。

#### 3 监听连接

服务器端需要调用 `listen()` 开始监听客户端的连接请求：

```c
if (listen(sockfd, SOMAXCONN) < 0) {
    perror("listen failed");
    exit(EXIT_FAILURE);
}
```

- **SOMAXCONN**：最大连接队列的长度。

#### 4 接受连接

使用 `accept()` 来接受客户端的连接请求：

```c
struct sockaddr_in cliaddr;
socklen_t clilen = sizeof(cliaddr);
int connfd = accept(sockfd, (struct sockaddr *)&cliaddr, &clilen);

if (connfd < 0) {
    perror("accept failed");
    exit(EXIT_FAILURE);
}
```

- **connfd**：新的套接字描述符，用于与客户端通信。

#### 5 数据接收和发送

使用 `recv()` 接收数据：

```c
char buffer[1024];
ssize_t n = recv(connfd, buffer, sizeof(buffer), 0);

if (n < 0) {
    perror("recv failed");
    exit(EXIT_FAILURE);
}
```

使用 `send()` 发送数据：

```c
ssize_t n = send(connfd, buffer, strlen(buffer), 0);

if (n < 0) {
    perror("send failed");
    exit(EXIT_FAILURE);
}
```

#### 6 关闭套接字

当不再需要连接时，使用 `close()` 关闭套接字：

```c
close(sockfd);
```

### 3 Socket 编程实验

#### 实验要求

服务端接收客户端发来的数据，并打印接收到的数据和客户端 IP。

#### 1 服务端程序

创建服务器端程序 `demo72_server.c`：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define PORT 8888
#define BUFFER_SIZE 1024

int main() {
    int sockfd, connfd;
    struct sockaddr_in servaddr, cliaddr;
    socklen_t cliaddr_len = sizeof(cliaddr);
    char buffer[BUFFER_SIZE];

    sockfd = socket(AF_INET, SOCK_STREAM, 0);
    if (sockfd == -1) {
        perror("socket");
        exit(EXIT_FAILURE);
    }

    memset(&servaddr, 0, sizeof(servaddr));
    servaddr.sin_family = AF_INET;
    servaddr.sin_addr.s_addr = htonl(INADDR_ANY);
    servaddr.sin_port = htons(PORT);

    if (bind(sockfd, (struct sockaddr *)&servaddr, sizeof(servaddr)) == -1) {
        perror("bind");
        exit(EXIT_FAILURE);
    }

    if (listen(sockfd, 10) == -1) {
        perror("listen");
        exit(EXIT_FAILURE);
    }

    printf("Server started.\n");

    connfd = accept(sockfd, (struct sockaddr *)&cliaddr, &cliaddr_len);
    if (connfd == -1) {
        perror("accept");
        exit(EXIT_FAILURE);
    }

    printf("Client connected: %s:%d\n", inet_ntoa(cliaddr.sin_addr), ntohs(cliaddr.sin_port));

    while (1) {
        memset(buffer, 0, BUFFER_SIZE);
        ssize_t numBytes = recv(connfd, buffer, BUFFER_SIZE - 1, 0);
        if (numBytes == -1) {
            perror("recv");
            exit(EXIT_FAILURE);
        } else if (numBytes == 0) {
            printf("Client closed connection.\n");
            break;
        } else {
            printf("Received from client: %s", buffer);
        }
    }

    close(connfd);
    close(sockfd);

    return 0;
}
```

#### 2 客户端程序

创建客户端程序 `demo72_client.c`：

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

#define PORT 8888
#define BUFFER_SIZE 1024

int main(int argc, char *argv[]) {
    int sockfd;
    struct sockaddr_in servaddr;
    char buffer[BUFFER_SIZE];

    if (argc < 2) {
        fprintf(stderr, "Usage: %s server_ip\n", argv[0]);
        exit(EXIT_FAILURE);
    }

    sockfd = socket(AF_INET, SOCK_STREAM, 0);
    if (sockfd == -1) {
        perror("socket");
        exit(EXIT_FAILURE);
    }

    memset(&servaddr, 0, sizeof(servaddr));
    servaddr.sin_family = AF_INET;
    servaddr.sin_addr.s_addr = inet_addr(argv[1]);
    servaddr.sin_port = htons(PORT);

    if (connect(sockfd, (struct sockaddr *)&servaddr, sizeof(servaddr)) == -1) {
        perror("connect");
        exit(EXIT_FAILURE);
    }

    while (fgets(buffer, BUFFER_SIZE, stdin) != NULL) {
        if (send(sockfd, buffer, strlen(buffer), 0) == -1) {
            perror("send");
            exit(EXIT_FAILURE);
        }
    }

    close(sockfd);

    return 0;
}
```
