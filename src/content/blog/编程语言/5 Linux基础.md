24|16|16|# Linux 目录结构
25|17|17|
26|18|18|## 1.1 Linux目录结构
27|19|19|
28|20|20|​	在Linux系统中，文件系统采用了树型结构，以一个称为根目录的起始点开始，所有文件和目录都位于根目录之下。根目录在Linux中表示为`/`（斜线），是整个文件系统的顶级目录。从根目录向下，可以创建更多的目录和文件，形成一个层次结构。
29|21|21|
30|22|22|​	在Windows系统中，每个逻辑驱动器通常会被分配一个盘符，如C盘、D盘、E盘等。每个盘符可以拥有自己的根目录和文件系统。这使得在Windows系统中可以直接访问不同的逻辑驱动器，并且它们之间是相互独立的。
31|23|23|
32|24|24|## 1.2 Linux路径描述方式
33|25|25|
34|26|26|​	在Linux系统中，路径之间的层级关系使用斜线（/）来表示。例如，`/home/user/Documents` 表示位于根目录下的 `home` 目录中的 `user` 目录中的 `Documents` 目录。
35|27|27|
36|28|28|在Windows系统中，路径之间的层级关系使用反斜线（\）来表示。例如，`C:\Users\Username\Documents` 表示位于 C 盘下的 `Users` 目录中的 `Username` 目录中的 `Documents` 目录。
37|29|29|
38|30|30|在Linux系统中，斜线（/）具有以下含义：
39|31|31|
40|32|32|1. 根目录：当斜线（/）出现在路径的开头时，表示根目录。根目录是整个文件系统的顶级目录，是文件系统的起点。所有其他目录和文件都位于根目录之下。例如，`/home/user/Documents` 表示位于根目录的 `home` 目录中的 `user` 目录中的 `Documents` 目录。
41|33|33|2. 层次关系：当斜线（/）出现在路径的中间或末尾时，表示路径中不同层级之间的关系。斜线用于分隔不同的目录或文件名，并表示它们之间的层次关系。例如，`/home/user/Documents` 中的斜线表示 `home` 目录位于根目录下，`user` 目录位于 `home` 目录下，`Documents` 目录位于 `user` 目录下。
42|34|34|
43|35|35|#  相对路径绝对路径
44|36|36|
45|37|37|绝对路径是一种路径写法，它以根目录作为起点来描述文件或目录的位置。绝对路径始终以斜线（/）开头。通过绝对路径，您可以准确地指定文件或目录在文件系统中的位置，不受当前工作目录的影响。例如，`/home/user/Documents/file.txt` 是一个绝对路径，它从根目录开始，描述了文件 `file.txt` 在 `Documents` 目录中的位置。
46|38|38|
47|39|39|相对路径是另一种路径写法，它以当前工作目录为起点来描述文件或目录的位置。相对路径不以斜线开头，而是从当前工作目录开始描述路径的层级关系。相对路径的解析依赖于当前工作目录的位置。例如，如果当前工作目录是 `/home/user`，那么相对路径 `Documents/file.txt` 表示文件 `file.txt` 在 `Documents` 目录中的位置，而该目录位于当前工作目录下的 `user` 目录中。
48|40|40|
49|41|41|相对路径的优势在于更加简洁，不需要显式地指定根目录。它适用于当前工作目录已知的情况，并且可以相对于当前位置快速定位文件或目录。
50|42|42|
51|43|43|总结起来，绝对路径以根目录为起点，以斜线开头来描述文件或目录的位置。相对路径以当前工作目录为起点，不以斜线开头来描述文件或目录的位置，路径描述相对于当前位置。
52|44|44|
53|45|45|#  特殊路径符
54|46|46|
55|47|47|1. `.` （点）：表示当前目录。使用单个点表示当前所在的目录。例如，`./file.txt` 表示当前目录下的文件 `file.txt`。
56|48|48|
57|49|49|2. `..` （点点）：表示上一级目录。使用两个点表示当前目录的父目录。例如，`../dir/file.txt` 表示当前目录的上一级目录中的 `dir` 目录下的文件 `file.txt`。
58|50|50|
59|51|51|3. `~` （波浪线）：表示当前用户的主目录。波浪线后面可以跟其他路径来表示主目录下的子目录或文件。例如，`~/Documents` 表示当前用户主目录下的 `Documents` 目录。
60|52|52|
61|53|53|4. `/` （斜线）：表示根目录。根目录是整个文件系统的顶级目录。以斜线开头的路径表示从根目录开始的绝对路径。例如，`/usr/bin` 表示根目录下的 `usr` 目录中的 `bin` 目录。
62|54|54|
63|55|55|5. `-` （短横线）：表示前一个工作目录。使用短横线可以回到上一个工作目录。这在需要快速切换目录时很有用。

65|57|57|   # 普通用户和系统管理员用户
66|58|58|
67|59|59|   1. 在 Linux 系统中，root 用户和普通用户是两种不同的用户身份。下面详细讲解它们的区别和特点：
68|60|60|
69|61|61|      1. Root 用户：
70|62|62|         - 超级用户：root 用户是系统的超级用户，也称为管理员用户。它拥有系统中的最高权限，可以执行所有操作，包括修改系统配置、安装软件、管理用户和文件等。
71|63|63|         - UID 和 GID：root 用户的用户标识号（UID）和组标识号（GID）都为 0。
72|64|64|         - 根目录：root 用户的主目录为根目录（/），在大多数 Linux 发行版中，其主目录路径为 "/root"。
73|65|65|         - 安全性注意：由于 root 用户具有最高权限，使用 root 用户时需要谨慎操作。推荐使用 sudo 命令以普通用户身份执行需要特权的任务，以提高系统安全性。
74|66|66|         - 特权访问：root 用户可以直接登录系统，或通过其他用户登录后切换到 root 用户。默认情况下，远程登录方式禁止直接使用 root 用户登录。
75|67|67|      2. 普通用户：
76|68|68|         - 一般权限：普通用户是系统中除 root 用户以外的用户。它们具有有限的权限，并受到权限限制，无法执行对系统关键部分的修改和访问。
77|69|69|         - UID 和 GID：普通用户具有非零的用户标识号（UID）和组标识号（GID），用于唯一标识用户和分配权限。
78|70|70|         - 主目录：每个普通用户都有一个主目录，用于存储其个人文件和配置。主目录的路径通常是 "/home/username"，其中 "username" 是用户的用户名。
79|71|71|         - 限制：普通用户受到许多权限限制，例如无法修改系统配置、安装软件包或访问其他用户的私有文件等。这些限制增加了系统的安全性，防止误操作和恶意行为。
80|72|72|
81|73|73|      总结起来，root 用户是系统的超级用户，具有最高权限和特权，能够执行所有操作。普通用户是系统中的常规用户，受到权限限制，无法执行对系统关键部分的修改和访问。使用 root 用户需要谨慎，而普通用户则更适合日常任务和常规操作，以增加系统安全性。

83|75|75|      # su和exit
84|76|76|
85|77|77|su命令是用于在 Linux 系统中切换用户的命令，它允许当前用户切换到其他用户账户，包括 root 用户。而exit命令用于退出当前用户的登录会话。
86|78|78|
87|79|79|以下是对su命令和exit命令的详细解释：
88|80|80|
89|81|81|1. su命令：
90|82|82|   - 语法：su [-] [用户名]
91|83|83|   - - 符号是可选的，表示是否在切换用户后加载目标用户的环境变量。如果使用 - 符号，会加载目标用户的环境变量，包括其配置文件（如 .bashrc）中定义的环境变量。如果不使用 - 符号，则不会加载环境变量。
92|84|84|   - 用户名参数是要切换到的目标用户的用户名。如果省略用户名，su 命令默认切换到 root 用户。
93|85|85|   - 在切换用户时，如果当前用户不是 root 用户，通常需要输入目标用户的密码。只有在当前用户是 root 用户时，才可以直接切换到其他用户而无需密码。
94|86|86|2. exit命令：
95|87|87|   - exit命令用于退出当前用户的登录会话或终端窗口。
96|88|88|   - 当使用 su 命令切换到其他用户时，可以使用 exit 命令返回到之前的用户。执行 exit 命令后，会关闭当前用户的登录会话或终端窗口，并回到上一个用户的会话或窗口。
97|89|89|   - 另外，可以使用快捷键 Ctrl + D 来执行与 exit 命令相同的操作，即退出当前用户的登录会话。
98|90|90|
99|91|91|需要注意的是，su 命令和 exit 命令都需要在命令行中使用，而不是在图形界面中使用。su 命令要求输入目标用户的密码，而 exit 命令不需要任何参数，直接执行即可退出当前用户的登录会话。这些命令在管理 Linux 系统时非常有用，可以在不同的用户之间切换，以便执行特定的任务和操作。
100|92|92|
101|93|93|# sudo命令
102|94|94|
103|95|95|sudo命令是在 Linux 系统中用于临时以 root 身份执行命令的工具。它允许普通用户在执行需要特权的操作时，临时获取 root 用户的权限，而无需切换到 root 用户。
104|96|96|
105|97|97|以下是对sudo命令的详细解释：
106|98|98|
107|99|99|1. 语法：sudo 其他命令
108|100|100|   - 其他命令是要以 root 权限执行的实际命令。在其他命令之前添加 sudo 前缀，即可为该命令临时赋予 root 用户的权限。
109|101|101|   - 举例来说，如果要以 root 权限安装软件包，可以使用命令 sudo apt install package-name。
110|102|102|2. 配置 sudo 认证：
111|103|103|   - 并非所有用户都有权使用 sudo 命令。在 Linux 系统中，需要配置 sudo 来授权普通用户使用该命令。
112|104|104|   - 在配置 sudo 时，可以为特定的用户或用户组分配 sudo 认证。
113|105|105|   - sudo 配置文件位于 /etc/sudoers，可以使用 visudo 命令来编辑该文件，visudo 命令会检查 sudoers 文件的语法错误。
114|106|106|   - 在 sudoers 文件中，可以使用特定的语法为用户或用户组添加授权，例如：
115|107|107|     - user1 ALL=(ALL) ALL：为 user1 用户授予所有命令的 sudo 权限。
116|108|108|     - %group1 ALL=(ALL) ALL：为 group1 用户组授予所有命令的 sudo 权限。
117|109|109|3. sudo 命令的工作方式：
118|110|110|   - 当用户使用 sudo 执行命令时，系统会要求用户输入自己的密码进行身份验证。这是为了确保执行特权操作的用户是经过授权的。
119|111|111|   - 默认情况下，sudo 会在一段时间内缓存用户的密码，以便在此期间内执行多次 sudo 命令时无需重复输入密码。
120|112|112|   - sudo 还提供了一些选项，例如：
121|113|113|     - -i：以 root 用户的身份登录并获取 root 用户的环境变量。
122|114|114|     - -u user：以指定用户的身份执行命令。

124|116|116|# 查看 Linux 文件的权限管控信息
125|117|117|
126|118|118|​	在 Linux 系统中，可以使用 ls 命令结合选项来查看文件的权限管控信息。ls 命令用于列出目录中的文件和子目录，并显示其相关属性。
127|119|119|
128|120|120|- 语法：ls -l [文件或目录]
129|121|121|- -l 选项指示 ls 命令以长格式显示文件信息，包括权限、所有者、组、大小、时间戳等。
130|122|122|- 文件或目录参数是要查看权限信息的文件或目录的路径。如果省略该参数，默认为当前目录。
131|123|123|
132|124|124|输出结果中权限信息由 10 个字符组成，分别表示文件类型和文件权限。前三个字符表示所有者权限，接下来三个字符表示所属组权限，最后三个字符表示其他用户权限。
133|125|125|
134|126|126|# 读、写、执行三种权限的含义
135|127|127|
136|128|128|​	在 Linux 系统中，每个文件都有三种基本权限：读取权限、写入权限和执行权限。
137|129|129|
138|130|130|- 读取权限（r）：读取权限允许用户查看文件的内容。对于目录来说，读取权限允许用户列出目录中的文件和子目录。
139|131|131|- 写入权限（w）：写入权限允许用户修改文件的内容、重命名文件或删除文件。对于目录来说，写入权限允许用户在目录中创建、删除或重命名文件和子目录。
140|132|132|- 执行权限（x）：执行权限允许用户执行可执行文件、脚本或进入目录。
141|133|133|
142|134|134|权限字符的含义：
143|135|135|
144|136|136|- r 表示读取权限，用数字 4 表示。
145|137|137|- w 表示写入权限，用数字 2 表示。
146|138|138|- x 表示执行权限，用数字 1 表示。
147|139|139|- "-" 表示没有相应的权限。
148|140|140|
149|141|141|在权限字符的表示中，可以使用数字表示权限组合。例如，rwx 表示读取、写入和执行权限的组合，其数字表示为 7。r-x 表示具有读取和执行权限，但没有写入权限，其数字表示为 5。
150|142|142|
151|143|143|通过对文件和目录设置适当的权限，可以实现对系统资源的安全管控。正确设置权限可以保护重要文件的机密性、完整性和可用性，并防止未经授权的访问和修改。

153|145|145|![image-20240131225129893](https://chai-1301855619.cos.ap-beijing.myqcloud.com/202401312251927.png)
154|146|146|
155|147|147|- 第一个字符（-、d、l）表示文件类型：
156|148|148|  - "-" 表示普通文件。
157|149|149|  - "d" 表示目录（文件夹）。
158|150|150|  - "l" 表示符号链接（软链接）。
159|151|151|- 剩下的九个字符表示权限，按照权限分为三组：
160|152|152|  1. 所属用户权限（User）：
161|153|153|     - 第一个字符表示所属用户对文件或目录的权限：
162|154|154|       - "r" 表示具有读取权限。
163|155|155|       - "-" 表示没有读取权限。
164|156|156|     - 第二个字符表示所属用户对文件或目录的写入权限：
165|157|157|       - "w" 表示具有写入权限。
166|158|158|       - "-" 表示没有写入权限。
167|159|159|     - 第三个字符表示所属用户对文件或目录的执行权限：
168|160|160|       - "x" 表示具有执行权限。
169|161|161|       - "-" 表示没有执行权限。
170|162|162|  2. 所属用户组权限（Group）：
171|163|163|     - 第四个字符表示所属用户组对文件或目录的权限，含义与所属用户权限相同。
172|164|164|     - 第五个字符表示所属用户组对文件或目录的写入权限。
173|165|165|     - 第六个字符表示所属用户组对文件或目录的执行权限。
174|166|166|  3. 其他用户权限（Other）：
175|167|167|     - 第七个字符表示其他用户对文件或目录的权限。
176|168|168|     - 第八个字符表示其他用户对文件或目录的写入权限。
177|169|169|     - 第九个字符表示其他用户对文件或目录的执行权限。
178|170|170|
179|171|171|这种权限字符的排列顺序是：文件类型 + 所属用户权限 + 所属用户组权限 + 其他用户权限。
180|172|172|
181|173|173|权限字符 "r"、"w" 和 "x" 分别表示读取、写入和执行权限。"-" 表示没有相应的权限。
182|174|174|
183|175|175|通过读取这些权限字符，可以了解特定文件或目录对不同用户的访问权限。正确设置这些权限可以确保文件和目录的安全性和可访问性。

188|180|180|
189|181|181|rwx是UNIX和类UNIX系统中用于表示文件或目录权限的符号。它是由三个字符组成，分别代表读权限（r），写权限（w）和执行权限（x）。
190|182|182|
191|183|183|- r（读权限）表示允许用户读取文件的内容或查看目录中的文件列表。对于文件来说，这意味着可以读取文件的内容；对于目录来说，这意味着可以列出目录中的文件。
192|184|184|- w（写权限）表示允许用户修改文件的内容或在目录中创建、删除和重命名文件。对于文件来说，这意味着可以修改文件的内容；对于目录来说，这意味着可以在目录中进行写操作。
193|185|185|- x（执行权限）表示允许用户执行文件或将工作目录更改为目录。对于文件来说，这意味着可以将文件作为可执行程序来运行；对于目录来说，这意味着可以进入该目录并在其中执行操作。
