54|# 01-shell的介绍
55|
56|简单的工作用简单语言来开发。
57|
58|==对Linux有一个更深层次的认识==
59|
60|Shell在交互式模式下，用户可以逐行输入命令，Shell会立即解释执行并输出结果，这种方式称为交互式Shell。这样用户可以根据需要逐步操作和获取结果。
61|
62|另一种执行命令的方式是批处理模式，用户事先编写好一个Shell脚本（也称为脚本文件），其中包含多条命令。用户只需执行该脚本，Shell会按顺序读取脚本文件中的命令，并一次性执行完所有命令，而无需手动逐行输入。这种方式称为批处理Shell。
63|
64|Shell脚本类似于编程语言，它可以包含变量、流程控制语句（如条件判断和循环），以及其他功能。然而，与编程语言不同的是，Shell脚本是解释执行的，不需要进行编译过程。Shell程序会逐行读取脚本中的命令，并将其解释执行，就好像用户一行一行地在Shell提示符下输入并执行这些命令一样。
65|
66|通过编写Shell脚本，用户可以将一系列常用的命令组合起来，实现自动化、批量化的操作，提高工作效率。
67|
68| 不同的bash版本：
69|
70|1. sh (Bourne Shell): 由Steve Bourne开发，几乎所有UNIX系统都默认配备了sh。它是一种简单的Shell，不支持一些高级功能，如作业控制、命令历史和命令行编辑。
71|2. csh (C Shell): 由Bill Joy开发，随BSD UNIX发布。它的流程控制语句类似于C语言，支持Bourne Shell不支持的一些功能。然而，csh也有一些限制，并不是在所有UNIX系统上都广泛使用。
72|3. ksh (Korn Shell): 由David Korn开发，旨在向后兼容sh，并添加了csh引入的新功能。ksh是许多UNIX系统的标准配置Shell，因此在这些系统上，/bin/sh通常是指向/bin/ksh的符号链接。
73|4. tcsh (TENEX C Shell): 是csh的增强版本，引入了命令补全等功能。在FreeBSD、MacOS X等系统上，tcsh替代了csh。
74|5. bash (Bourne Again Shell): 由GNU开发的Shell，目标是与POSIX标准保持一致，并兼容sh。bash从csh和ksh借鉴了许多功能。它是各种Linux发行版的标准配置Shell，因此在Linux系统上，/bin/sh通常是指向/bin/bash的符号链接。尽管如此，bash和sh仍然存在一些差异。一方面，bash扩展了某些命令和参数，另一方面，bash并不完全兼容sh，某些行为可能不一致。因此，当以sh为程序名启动bash时，bash可以模拟sh的行为，不识别扩展命令，并保持与sh的行为一致。

80|# 2 shell内建命令
81|
82|在命令行中，用户输入的命令通常由Shell执行。但是Shell的内建命令是一个例外，它们不会创建新的进程，而是在Shell进程中直接执行，类似于调用Shell内部的一个函数。
83|
84|一些常见的Shell内建命令包括umask、exit、cd、alias、man等。这些命令在执行时不会启动新的进程，而是直接在当前Shell进程中执行相应的功能。
85|
86|对于一些不是内建命令的普通命令，可以使用`which`命令来查找其可执行文件的位置。而内建命令没有对应的可执行文件，因此无法用`which`命令查找到它们的位置。
87|
88|要查看Shell内建命令的详细信息，可以使用`man bash-builtins`命令（对于Bash Shell）。这将显示有关Shell内建命令的手册页面，包括命令的用法、选项和示例。
89|
90|虽然内建命令不创建新的进程，但它们执行结束后也会有一个退出状态码（Exit Status）。通常，执行成功的内建命令会返回0作为状态码，而执行失败的则返回非零值。这样的状态码可以通过特殊变量`$?`来读取，以便在Shell脚本中进行后续的处理。
91|
92|# 3 执行脚本
93|
94|可以看到一个简单的脚本test.sh，其中包含以下内容：
95|
96|```shell
97|#!/bin/sh
98|echo HelloWorld
99|cd..
100|```
101|
102|这是一个简单的Shell脚本示例，使用了Shebang（#!）来指定解释器为`/bin/sh`。脚本的功能是输出"HelloWorld"并尝试切换到上一级目录。
103|
104|要执行这个脚本，可以通过以下方式之一：
105|
106|1. 给脚本文件添加可执行权限并执行：
107|
108|   ```shell
109|   $ chmod +x test.sh
110|   $ ./test.sh
111|   ```
112|
113|   这样Shell会fork一个子进程并调用exec执行`./test.sh`，由于指定了Shebang，子进程会用`/bin/sh`解释器的代码段替换当前进程，并从解释器的开始执行。因此，脚本中的命令会被执行。
114|
115|2. 直接使用解释器执行脚本：
116|
117|   ```shell
118|   $ /bin/sh ./test.sh
119|   ```
120|
121|   这种方式不需要脚本文件具有可执行权限，直接通过指定解释器执行脚本。
122|
123|无论使用哪种方式执行脚本，都会输出"HelloWorld"并尝试切换到上一级目录。需要注意的是，脚本中的`cd..`命令存在错误，正确的写法应该是`cd ..`，使用空格分隔目录名和`cd`命令。
124|
125|另外，提到了使用括号和`source`命令来执行Shell脚本。这种方式不会创建子Shell，而是直接在当前交互式Shell中逐行执行脚本中的命令。例如：
126|
127|```shell
128|$ (cd ..; ls -l)
129|```
130|
131|这将在当前Shell中执行括号内的命令，不会影响当前Shell的PWD。而使用`source`命令可以直接在当前Shell中执行脚本，例如：
132|
133|```shell
134|$ source ./test.sh
135|```
136|
137|这种方式也不会创建子Shell，脚本中的命令会在当前Shell中逐行执行。
138|
139|source 和.的作用以及最终效果相同：
140|
141|~~~shell
142|. test.sh
143|~~~

145|# 4.变量的使用
146|
147|​	在Shell中，变量通常以字母加下划线开头，并由字母、数字和下划线组成。变量的定义或赋值可以使用等号（=）进行，例如`VARNAME=value`。需要注意的是，等号两边不能有空格，否则空格会被解释成命令和参数的一部分。
148|
149|在使用变量时，可以使用美元符号（$）加上变量名来引用变量的值，例如`echo $VARNAME`。如果变量名后面紧跟着其他字符（例如后缀），为了避免将变量名和后缀当作一个整体，可以使用花括号将变量名分隔出来，例如`echo ${VARNAME}suffix`。
150|
151|# 5.变量的分类
152|
153|可以将Shell中的变量分为两种不同的分类：
154|
155|1. 环境变量（Environment Variables）：
156|
157|   环境变量是操作系统自带的变量，每个进程都有自己的环境变量集合。当启动一个子进程时，子进程会从父进程继承一份相同的环境变量副本。这意味着子进程拥有与父进程相同的初始环境变量。
158|
159|   子进程对环境变量的修改不会影响父进程的环境变量。环境变量是单向传递的，子进程可以修改自己的环境变量，但不会影响到父进程或其他兄弟进程的环境变量。
160|
161|   在Shell中，可以使用`export`命令将一个变量导出为环境变量，例如`export varname=value`。另外，也可以直接在当前Shell进程中定义一个变量，并将其同时导出为环境变量，例如`varname=value`和`export varname`两个命令组合使用。
162|
163|2. 本地变量（Local Variables）：
164|
165|   **全局变量（Global Variables）:**
166|   全局变量是在Shell的解析环境中存在的变量，不需要任何修饰符进行声明。无论是在函数内部还是函数外部，全局变量的作用范围都是整个Shell进程。全局变量的生命周期从声明语句开始，一直持续到Shell脚本结束。
167|
168|   **局部变量（Local Variables）:**
169|   局部变量使用`local`关键字进行修饰，只能在函数内部声明。局部变量的作用范围仅限于声明它的函数内部，在函数结束后，局部变量将不再可用。局部变量的生命周期从声明语句开始，一直持续到函数执行结束。
170|
171|   Shell内部变量只能在当前Shell进程中使用，如果跨越了进程边界，例如在子进程中或从子进程返回到父进程，那么变量的可见性将受到限制。
172|
173|环境变量是进程间共享的概念，而本地变量是Shell特有的概念。在Shell中，环境变量和本地变量的定义和使用方式相似。
174|
175|可以使用`export`命令将本地变量导出为环境变量。一般情况下，定义和导出环境变量可以一步完成，例如`export VARNAME=value`。也可以分两步完成，先定义本地变量，然后使用`export`命令将其导出为环境变量。
176|
177|
178|
179|3. 删除环境变量
180|   使用`unset`命令来删除Shell中的变量，无论是环境变量还是普通的Shell内部变量。
181|
182|使用`unset`命令的语法是：`unset 变量名`。
183|
184|例如，如果要删除一个环境变量，可以执行如下命令：
185|
186|```
187|unset VARNAME
188|```
189|
190|如果要删除一个普通的Shell内部变量，同样可以使用`unset`命令：
191|
192|```
193|unset varname
194|```
195|
196|执行`unset`命令后，相应的变量将被从当前Shell进程中移除，不再可用。
197|
198|需要注意的是，`unset`命令只能删除当前Shell进程中的变量，不会影响其他Shell进程或操作系统级别的环境变量。
199|
200|# 6.文件名替换和参数扩展
201|
202|文件名代换（Globbing）是一种在Shell中用于匹配文件名的机制，通常使用通配符来指定匹配模式。以下是一些常用的通配符和它们的具体含义：
203|
204|- `*`：匹配零个或多个任意字符。
205|- `?`：匹配一个任意字符。
206|- `[]`：匹配方括号中的任意一个字符，可以指定多个字符或字符范围。
207|
208|下面是一些示例说明如何使用这些通配符：
209|
210|1. `ls /dev/ttyS*`：匹配以`/dev/ttyS`开头的文件名，后面可以是任意字符。
211|2. `ls ch0?.doc`：匹配以`ch0`开头，后面跟着一个任意字符，然后以`.doc`结尾的文件名。
212|3. `ls ch0[0-2].doc`：匹配以`ch0`开头，后面跟着一个字符，该字符是0、1或2，然后以`.doc`结尾的文件名。
213|4. `Is ch[012] [0-9].doc`：这个示例可能存在一些打字错误。如果将其理解为`Is ch[012] [0-9].doc`，那么它将匹配以`ch0`、`ch1`或`ch2`开头，后面跟着一个数字，然后以空格和一个数字，最后以`.doc`结尾的文件名。
214|
215|需要注意的是，Globbing所匹配的文件名是Shell在参数传递给命令之前进行展开的。因此，在执行诸如`ls`或`Is`这样的命令之前，Shell会将匹配的文件名展开为实际的文件名，并将这些文件名作为命令的参数传递。例如，在使用`ls ch0[012].doc`命令时，如果当前目录下存在`ch00.doc`和`ch02.doc`这两个文件，那么实际传递给`ls`命令的参数将是这两个文件名，而不是一个匹配字符串。 
216|
217|# 7.命令代换
218|
219|命令代换（Command Substitution）是一种在Shell中执行命令并将其输出结果直接嵌入到另一个命令或上下文中的机制。在Shell中，可以使用反引号（`）或美元符号加圆括号（$()）来表示命令代换。
220|
221|以下是一些示例说明如何使用命令代换：
222|
223|1. 使用反引号（`）表示命令代换：
224|
225|   ```
226|   DATE=`date`
227|   echo $DATE
228|   ```
229|
230|   在上述示例中，`date`命令被执行，并将其输出结果赋给变量`DATE`。然后，使用`echo`命令打印出`DATE`变量的值，即当前的日期和时间。
231|
232|2. 使用美元符号加圆括号（$()）表示命令代换：
233|
234|   ```
235|   DATE=$(date)
236|   echo $DATE
237|   ```
238|
239|   这个示例与前一个示例的功能相同，只是使用了不同的语法形式。`$(date)`会执行`date`命令，并将输出结果赋给变量`DATE`。
240|
241|```shell
242|#!/bin/bash
243|# 获取当前日期和时间
244|dateTime=$(date)
245|echo "dateTime is $dateTime"
246|
247|# 在当前路径下创建1.txt文件
248|touch 1.txt
249|
250|# 获取当前脚本所在路径
251|curPath=$(cd "$(dirname "$0")" && pwd)
252|touch "$curPath/1.txt"
253|```
254|
255|`dateTime=$(date)`使用命令代换获取当前日期和时间，并将结果存储在`dateTime`变量中，然后使用`echo`命令打印出该变量的值。
256|
257|在创建文件部分，`touch 1.txt`直接在当前路径下创建了1.txt文件。
258|
259|获取当前脚本所在路径的部分，使用了`cd`命令结合命令代换和`pwd`命令来实现。`cd "$(dirname "$0")"`将当前路径切换到脚本所在路径，然后使用`pwd`命令获取该路径，并将结果存储在`curPath`变量中。最后，使用`touch`命令在该路径下创建1.txt文件。
260|
261|# 8.算数代换
262|
263|算术代换（Arithmetic Substitution）是一种在Shell中进行数值计算的机制，使用双圆括号`((...))`来表示。在算术代换中，Shell会将双圆括号中的表达式作为数值进行计算，并返回计算结果。
264|
265|以下是一些示例说明如何使用算术代换：
266|
267|1. 使用双圆括号进行算术计算：
268|
269|   ```bash
270|   VAR=45
271|   echo $((VAR + 3))
272|   ```
273|
274|   在上述示例中，`$((VAR + 3))`会将变量`VAR`的值（45）与3相加，并返回计算结果。
275|
276|2. 使用不同进制进行数值解释：
277|
278|   ```bash
279|   echo $((8#10 + 11))
280|   echo $((16#10 + 11))
281|   ```
282|
283|   在上述示例中，`8#10`表示将后面的数字按照八进制进行解释，`16#10`表示将后面的数字按照十六进制进行解释。然后，进行相应的数值计算并返回结果。
284|
285|需要注意的是，算术代换中只能使用`+`、`-`、`*`、`/`和`()`等算术运算符，且只能进行整数运算。
286|
287|在示例中，`s((SVAR+3))`和`[VAR+3]`是不正确的写法。正确的写法应该是`((VAR+3))`或`echo $((VAR+3))`。
288|
289|# 9.转义字符
290|
291|在Shell中，反斜杠（`\`）可以用作转义字符，用于去除紧跟其后的单个字符的特殊意义。换句话说，反斜杠告诉Shell将紧跟其后的字符视为字面值而不是特殊字符。
292|
293|以下是关于反斜杠的一些示例：
294|
295|1. 转义特殊字符：
296|
297|   ```bash
298|   #!/bin/bash
299|   echo \$SHELL
300|   $SHELL
301|   ```
302|
303|   
304|
305|   ~~~bash
306|   echo \\
307|   ~~~
308|
309|   
310|
311|   在上述示例中，`\$SHELL`和`\\`使用反斜杠进行转义，将其后的字符视为字面值。因此，`\S`被解释为"S"，而`\)`被解释为")"。
312|
313|2. 创建文件名包含特殊字符的文件：
314|
315|   ```
316|   touch \s\ \S
317|   ```
318|
319|   在上述示例中，通过使用反斜杠转义空格字符，可以创建一个文件名为"S "（S后面有一个空格）的文件。
320|
321|对于以连字符（减号/破折号）开头的文件名，由于Shell会将以连字符开头的命令行参数视为选项，而不是文件名，所以创建以连字符开头的文件名会遇到问题。如果您非要处理以连字符开头的文件名，有两种常见的方法：
322|
323|使用相对路径或绝对路径：
324|
325|```
326|touch ./-filename
327|touch /path/to/-filename
328|```
329|
330|在上述示例中，通过在文件名前添加`./`（当前目录）或者使用绝对路径来创建以连字符开头的文件。
331|
332|使用`--`来表示选项的结束：
333|
334|```
335|touch -- -filename
336|```
337|
338|在上述示例中，`--`告诉Shell选项结束，后面的参数将被视为文件名而不是选项。

340|# 10 单引号和双引号
341|
342|在Shell脚本中，单引号（'）和双引号（"）都用作字符串的界定符，但它们有不同的行为方式。
343|
344|1. 单引号（'）：
345|
346|   - 单引号用于创建保持字符串的字面值的字符串。在单引号字符串中，所有字符都被视为普通字符，不进行变量扩展、命令替换或特殊字符转义。
347|
348|   - 例如：
349|
350|     ```
351|     echo 'SHEIJ'
352|     # 输出：SHEIJ
353|     
354|     echo 'ABC\'
355|     # 输出：ABC\
356|     ```
357|
358|   - 在单引号字符串中，无法直接嵌入单引号字符。如果需要在单引号字符串中使用单引号，可以通过拼接字符串或转义字符的方式实现。
359|
360|2. 双引号（"）：
361|
362|   - 双引号用于创建字符串，并支持变量扩展和特殊字符转义。
363|
364|   - 在双引号字符串中，变量会被展开为其对应的值，命令替换会执行替换操作，而特殊字符可以通过转义字符进行转义。
365|
366|   - 例如：
367|
368|     ```
369|     VAR="World"
370|     echo "Hello, $VAR!"
371|     # 输出：Hello, World!
372|     
373|     echo "ABC\$"
374|     # 输出：ABC$
375|     ```
376|
377|   - 在双引号字符串中，变量会被展开为其对应的值（通过`$`进行标识），而特殊字符可以通过反斜杠进行转义。
378|
379|总结起来，单引号保持字符串的字面值，不进行变量扩展、命令替换和特殊字符转义。双引号支持变量扩展和特殊字符转义。
380|
381|​	作为有经验的Shell程序员，确实在使用变量作为参数传递时，习惯性地将变量用双引号括起来是一个很好的做法，特别是当变量的值中包含空格或其他特殊字符时。
382|
383|​	双引号的使用可以确保传递的参数被当作一个整体进行处理，而不会被解析为多个独立的片段。这对于保留变量值中的空格或其他特殊字符的完整性非常重要。
384|
385|​	以下是一些示例来说明这个问题：
386|
387|```
388|var='a b'
389|rm  $var        # 假设这是一个命令，参数是变量var的值
390|rm "$var"   # 假设这是一个命令，参数是变量var的值
391|```
392|
393|在上述示例中，如果没有使用双引号括起变量`$var`，那么Shell会将其解析为两个独立的参数："a"和"b"。这可能导致命令无法按预期工作，或者产生错误。
394|
395|通过使用双引号将变量括起来，Shell会将整个变量作为一个参数传递给命令，确保参数的完整性。在上述示例中，`Svar`和`rm`命令将正确处理变量`var`的值。

397|# 11 条件测试
398|
399|在Shell中，test命令用于测试条件是否成立，并返回相应的退出状态码。如果测试结果为真，则该命令的退出状态为0，如果测试结果为假，则退出状态为1。
400|
401|方括号（[）是test命令的等效形式，只是使用了不同的语法。方括号命令的参数与test命令相同，只是需要在方括号前后加上空格，并在方括号内部使用空格将各个参数分隔开。
402|
403|以下是对您提供的示例进行解释：
404|
405|```
406|var=2
407|test $var -gt 1
408|echo $?
409|```
410|
411|在这个示例中，`test $var -gt 1`用于测试变量`var`是否大于1。如果是，则`test`命令返回退出状态码0，否则返回退出状态码1。接下来，`echo $?`用于打印上一条命令的退出状态码。
412|
413|同样地，`[ $var -gt 3 ]`使用方括号形式进行条件测试，判断变量`var`是否大于3。如果是，则方括号命令返回退出状态码0，否则返回退出状态码1。
414|
415|总结起来，test命令和方括号（[）用于在Shell中进行条件测试。它们可以用于检查各种条件，包括数值比较、字符串比较、文件检查等。请注意保持参数之间的空格，并根据测试结果检查命令的退出状态码。
416|
417|- `(EXPRESSION)`：表达式为真。
418|- `!EXPRESSION`：表达式为假。
419|- `EXPRESSION1 -a EXPRESSION2`：同时满足EXPRESSION1和EXPRESSION2。
420|- `EXPRESSION1 -o EXPRESSION2`：满足EXPRESSION1或EXPRESSION2中的任意一个。
421|- `-n STRING`：字符串STRING的长度非零。
422|- `STRING`：等同于`-n STRING`，即字符串STRING的长度非零。
423|- `-z STRING`：字符串STRING的长度为零。
424|- `STRING1 = STRING2`：字符串STRING1和STRING2相等。
425|- `STRING1 != STRING2`：字符串STRING1和STRING2不相等。
426|- `INTEGER1 -eq INTEGER2`：整数INTEGER1等于INTEGER2。
427|- `INTEGER1 -ge INTEGER2`：整数INTEGER1大于或等于INTEGER2。
428|- `INTEGER1 -gt INTEGER2`：整数INTEGER1大于INTEGER2。
429|- `INTEGER1 -le INTEGER2`：整数INTEGER1小于或等于INTEGER2。
430|- `INTEGER1 -lt INTEGER2`：整数INTEGER1小于INTEGER2。
431|- `INTEGER1 -ne INTEGER2`：整数INTEGER1不等于INTEGER2。
432|- `FILE1 -ot FILE2`：判断FILE1是否比FILE2旧（修改时间早于文件2）。
433|- `-b FILE`：判断FILE是否为块设备。
434|- `-C FILE`：判断FILE是否为字符设备。
435|- `-d FILE`：判断FILE是否为目录。
436|- `-e FILE`：判断FILE是否存在（存在则为真）。
437|- `-f FILE`：判断FILE是否为普通文件。
438|- `-h FILE` 或 `-L FILE`：判断FILE是否为符号链接。
439|- `-k FILE`：判断FILE的粘着位是否已设置。
440|- `-O FILE`：判断当前用户是否为FILE的所有者。
441|- `-0 FILE`：判断FILE是否存在（存在则为真）。
442|
443|中括号是test的一个等效命令，所以[ ]两端要有空格，后面的参数都是该命令的参数，

445|# 12 if分支结构
446|
447|在Shell编程中，条件判断是非常常见和重要的部分，而if语句是一种用于实现条件判断和分支控制的结构。if语句允许根据条件的真假执行不同的代码块。下面我将详细讲解Shell编程中的if分支结构。
448|
449|if语句的基本语法如下：
450|
451|```bash
452|if condition
453|then
454|    # 代码块1，条件为真时执行的代码
455|else
456|    # 代码块2，条件为假时执行的代码（可选）
457|fi
458|```
459|
460|在这个语法中，`condition`是一个条件表达式，用于判断真假。如果`condition`为真，则执行`then`后的代码块1，否则执行可选的`else`后的代码块2。
461|
462|另一种形式的if语句是使用elif（即else if）来实现多个条件判断，例如：
463|
464|```bash
465|if condition1
466|then
467|    # 代码块1，条件1为真时执行的代码
468|elif condition2
469|then
470|    # 代码块2，条件2为真时执行的代码
471|else
472|    # 代码块3，以上条件都为假时执行的代码（可选）
473|fi
474|```
475|
476|在这个语法中，首先判断`condition1`，如果为真则执行代码块1；如果为假，则继续判断`condition2`，如果为真则执行代码块2；如果以上条件都为假，则执行可选的代码块3。
477|
478|下面是一个示例，演示了if语句的使用：
479|
480|```bash
481|#!/bin/bash
482|
483|score=80
484|
485|if [ $score -ge 90 ]; then
486|    echo "优秀"
487|elif [ $score -ge 80 ]; then
488|    echo "良好"
489|elif [ $score -ge 60 ]; then
490|    echo "及格"
491|else
492|    echo "不及格"
493|fi
494|```
495|
496|在这个示例中，根据变量`score`的值进行成绩判断。根据条件的真假，分别输出不同的结果。
497|
498|在if语句中，条件表达式可以使用各种比较运算符（如`-eq`、`-ne`、`-lt`、`-gt`、`-le`、`-ge`）来比较数字，也可以使用字符串比较运算符（如`=`、`!=`）来比较字符串。此外，还可以使用逻辑运算符（如`-a`、`-o`）进行多个条件的组合。
499|
500|在if语句中，代码块可以包含任意Shell命令、其他条件语句、循环结构等。
