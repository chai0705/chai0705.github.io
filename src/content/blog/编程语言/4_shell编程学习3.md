# 第 1 章 bash入门

​	shell 无非是另一个程序罢了，它的工作就是为用户执行其他程序。

​	将 shell 与操作系统自身分离使得软件开发人员（例如 bash 之父 Brian Fox 以及 bash 当前的开发者和维护者 Chet Ramey）更容易编写出更好的 shell，你也可以在不改动操作系统的前提下编写出新的 shell。

​	**为什么是bash？**

​	GNU 项目旨在创建一款彻头彻尾的 POSIX 系统，这个系统自然需要一个 POSIX shell，bash 最初正是该项目的一部分。

​	学习 bash 准没错。最常见的默认 shell 就是古老的 Bourne shell和 bash，后者基本上兼容前者。任何现代的主流 Unix 或类 Unix 操作系统肯定安装了其一。

​	**什么是bash？**

​	bash 其实就是一种用于执行命令的语言，你输入的命令（如 ls、cd、grep、cat）从某种意义上来说就是 bash 命令。这些命令有些是 bash 内建的，有些是独立的程序。就目前而言，这种差异并不重要。

​	**使用shell引用**

~~~shell
$ echo A coffee is $5?!
A coffee is ?!
$ echo "A coffee is $5?!"
-bash: !": event not found
$ echo 'A coffee is $5?!'
A coffee is $5?!
~~~

第一个例子要对 $5 进行变量扩展，但因为该变量并不存在，所以扩展结果为空。第二个例子要遵循同样的规则，但我们没有看到任何命令输出，这是因为对 ! 进行历史替换时失败了，原因是其不匹配任何历史记录。第三个例子的结果符合我们的预期。

**确定是否处于交互模式**

~~~shell
#!/usr/bin/env bash
# 文件名: interactive
case "$-" in
 *i*) # 在交互式shell中运行的代码位于此处
 ;;
 *) # 在非交互式shell中运行的代码位于此处
 ;;
esac
~~~

变量 $- 中保存了一个字符串，其中列出了当前所有的 shell 选

项。如果 shell 处于交互模式，则其中会包含 i。

# 第 2 章 标准输出

**将输出和错误消息发送到不同文件**

~~~shell
myprogram 1> messages.out 2> message.err
~~~

在 1> 和 2> 中，数字表示文件 描述符。1 代表标准输出（STDOUT），2 代表标准错误（STDERR）。起始编号 0 对应标准输入（STDIN）。如果不指定数字，则假定为 STDOUT。

**将输出和错误消息发送到同一文件**

首选：

~~~shell
both >& outfile
~~~

或者：

~~~shell
both &> outfile
~~~

**丢弃输出**

将输出重定向到 /dev/null，如下所示

~~~shell
find / -name myfile -print 2> /dev/null
noisy > /dev/null 2>&1
~~~

# 第 3 章 标准输入

​	**将数据与脚本存放在一起**

使用 <<（here-document）从命令行而非文件重定向输入文本。如果放在 shell 脚本中，则脚本文件可以同时包含数据与代码。以下是名为 ext 的 shell 脚本示例：

~~~shell
$ cat ext
#
# 下面是here-document
#
grep $1 <<EOF
mike x.123
joe x.234
sue x.555
pete x.818
sara x.822
bill x.919
EOF
$
~~~

**获取用户输入**

使用 read 语句：

~~~shell
read
~~~

或者：

~~~shell
read -p "answer me this " ANSWER
~~~

或者：

~~~shell
read -t 3 -p "answer quickly: " ANSWER
~~~

又或者：

~~~shell
read PRE MID POST
~~~

不带参数的 read 语句会读取用户输入并将其保存在 shell 变量REPLY 中，这是 read 的最简形式。如果希望 bash 在读取用户输入前先输出提示信息，可以使用 -p 选项。-p 之后的单词就是提示信息，如果想提供多个单词，可以将其引用起来。记住，要在提示信息结尾处加上标点符号或空格，因为光标会停在那里等待输入

**选择选项列表**

​	使用 bash 内建的 select 来生成菜单，随后用户可以输入选项数字进行选择



# 第 4 章 执行命令

**运行程序**

bash 使用名为 $PATH 的 shell 变量来定位可执行文件。$PATH 变量包含了一个目录列表。各个目录之间以冒号（:）分隔。bash 在这些目录中查找命令行上指定的可执行文件。目录的顺序很重要：bash按照变量中所列出的目录顺序依次查找，选择所找到的第一个同名的可执行文件。

~~~shell
$ echo $PATH
/bin:/usr/bin:/usr/local/bin:.
~~~

**依次执行多个命令**

将命令一个接一个敲进去即可

将这些命令写入文件，然后让 bash 执行该文件（简单的 shell 脚本）中的命令。

第三种是依次执行每个命令，只需要用分号将其隔开

如果只想在上一个程序成功运行的情况下运行下一个程序，并且所有的程序都正确设置了退出码，那么可以用 && 将其隔开：

~~~shell
long && medium && short
~~~

**同时执行多个命令**

​	在命令末尾添加一个 & 符号，在后台运行该命令

~~~shell
$ long &
[1] 4592
$ medium &
[2] 4593
$ short
$
~~~

或者，更好的做法是在单个命令行中完成所有操作

~~~shell
$ long & medium & short
[1] 4592
[2] 4593
$
~~~

​	　如果执行某个命令，但发觉其完成时间比预想的更长，那么可以用 Ctrl-Z 暂停该命令，返回到提示符下。接着输入 bg 来恢复作业，并在后台继续运行。这么做的效果相当于事前在命令尾部加上 & 符号

**了解命令是否成功运行**

​	如果编写命令或 shell 脚本的程序员遵循既定约定，那么 shell 变量 $? 会在命令失败时被设置为非 0 值。

**仅当一个命令运行成功后才执行下一个命令**

​	可以用 cd 命令的退出状态（$?）配合 if 语句来实现仅当 cd 命令成功时执行 rm 命令。

~~~shell
cd mytmp
if (( $? == 0 )); then rm * ; fi
~~~

　以下写法更好，它更清晰地展现并诠释了我们的意图。

~~~shell
if cd mytmp; then rm * ; fi
~~~

**&& 运算符**

​	在 bash 中使用 && 运算符，根据条件执行命令

~~~shell
cd mytmp && rm *
~~~

​	用 && 分隔两个命令，以此告诉 bash 先执行第一个命令，如果该命令成功（退出状态为 0），再执行第二个命令。这非常类似于用 if语句检查第一个命令的退出状态，从而判断是否执行第二个命令，可以写成这个样子：
~~~shell
cd mytmp
if (( $? == 0 )); then rm * ; fi
~~~



**set -e**

要想彻底检查错误，但又不想到处出现 if 语句，可以设置 -e 标记，这样的话，只要脚本中有任何命令（排除在 while 循环和 if语句中，因为其本身就要用到退出状态）出现错误（退出状态为非0），bash 就会退出

~~~shell
set -e
cd mytmp
rm *
~~~

设置了 -e 标记之后，shell 会在命令失败时退出。如果本例中的cd 命令失败，脚本则直接退出，不再执行 rm * 命令。

**|| 运算符**

|| 告诉 bash 如果第一个表达式为真（成功），则不再评估第二个表达式。

# 第 5 章 脚本编程基础：shell变量

​	bash 脚本中的变量名称通常采用全大写，但这并非强制性的，只是一种常见做法而已。变量不用事先声明，直接使用就行了。变量基本上都是字符串类型，不过有些运算符能够将变量内容视为数字。变量的实际用法如下所示。

~~~shell
# 使用shell变量的普通脚本
# （不过好歹也注释过了）
MYVAR="something"
echo $MYVAR
# 写法类似，但没有引号
MY_2ND=anotherone
echo $MY_2ND
# 这里需要使用引号：
MYOTHER="more stuff to echo"
echo $MYOTHER
~~~

​	bash 变量的语法有两处要点，但可能不那么一目了然。首先，赋值语法 *name=value* 看起来相当直观，但 = 两侧不能有任何空白字符。



**导出变量**

有时你的确想在脚本之间传递信息。在这种情况下，你可以导出变量，使变量的值能够传递给脚本所调用的其他程序。

用 env（或 export -p）命令查看那些被导出的、可用于子 shell的变量。

**在shell脚本中使用参数**

${1}会显示在命令行上被调用时所指定的第一个参数。其他参数的可用形式分别为 ${2}、${3}、${4}、${5} 等。单个数位的数字用不着花括号

 $*  表示深入的全部参数列表

"$@" 得到的不是一个字符串，而是与各个参数对应的带有引号的字符串列表。

 $#：表示输入参数数量

# 第 6 章 shell逻辑与算术

**条件分支**

~~~shell
if [ $# -lt 3 ]
then
 printf "%b" "Error. Not enough arguments.\n"
 printf "%b" "usage: myscript file1 op file2\n"
 exit 1
fi
~~~





~~~shell
if (( $# < 3 ))
then
 printf "%b" "Error. Not enough arguments.\n"
 printf "%b" "usage: myscript file1 op file2\n"
 exit 1
elif (( $# > 3 ))
then
 printf "%b" "Error. Too many arguments.\n"
 printf "%b" "usage: myscript file1 op file2\n"
 exit 2
else
 printf "%b" "Argument count correct. Proceeding...\n"
fi
~~~

**运算符**

| 运算符   | 描述                                       |
| :------- | :----------------------------------------- |
| -b       | 块设备文件（如 /dev/hda1）                 |
| -c       | 字符设备文件（如 /dev/tty）                |
| -d       | 目录文件                                   |
| -e       | 文件存在                                   |
| -f       | 普通文件                                   |
| -g       | 文件设置了 set-group-ID（setgid）位        |
| -h 或 -L | 符号链接文件                               |
| -G       | 有效组 ID（effective group ID）拥有的文件  |
| -k       | 文件设置了粘滞位                           |
| -N       | 文件自上次读取后被修改过                   |
| -O       | 有效用户 ID（effective user ID）拥有的文件 |
| -p       | 具名管道文件                               |
| -r       | 可读文件                                   |
| -s       | 文件大小不为空                             |
| -S       | 套接字文件                                 |
| -u       | 文件设置了 set-user-ID（setuid）位         |
| -w       | 可写文件                                   |
| -x       | 可执行文件                                 |



**数值比较运算符：**

| 运算符 | 含义       |
| :----- | :--------- |
| -lt    | 小于       |
| -le    | 小于或等于 |
| -gt    | 大于       |
| -ge    | 大于或等于 |
| -eq    | 等于       |
| -ne    | 不等于     |

**字符串比较运算符：**

| 运算符  | 含义   |
| :------ | :----- |
| = 或 == | 等于   |
| !=      | 不等于 |

​	这些运算符用于在条件语句（如if语句）中比较数值或字符串的大小或相等性。例如，您可以使用 `-lt` 运算符来检查一个数值是否小于另一个数值，并根据结果执行相应的操作。使用字符串比较运算符时，可以使用 `=` 或 `==` 来比较字符串是否相等，使用 `!=` 来比较字符串是否不相等。

可能最好的办法就是始终在 (( )) 语法中进行数值测试，在 [[ ]]语法中进行字符串比较。

**创建简单的菜单**

用 select 语句创建简单的字符型屏幕菜单

~~~shell
#!/usr/bin/env bash
# 实例文件：dbinit.1
#
DBLIST=$(sh ./listdb | tail -n +2)
select DB in $DBLIST
do
 echo Initializing database: $DB
 mysql -u user -p $DB <myinit.sql
done
~~~

# 第 10 章 脚本编程的附加特性

~~~

**代码重用**

用 bash shell 的 source 命令或 POSIX 的点号（.）读取配置文件的内容。

**使用函数：参数和返回值**

在调用函数时放置参数就和调用 shell 脚本一样。参数无非就是另一种命令行单词罢了。与命令行参数一样，在函数中引用参数也是使用 $1、$2 等。但是，$0 保持不变，其中包含的还是所调用脚本的名称。一旦从函数中返回，$1、$2 等就恢复原状，仍旧引用脚本参数。

**中断陷阱**

使用实用工具 trap 来设置信号处理程序。

time 以多种方式报告某个进程或管道所花费的时间。
