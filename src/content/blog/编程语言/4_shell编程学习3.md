22|# 第 1 章 bash入门
23|
24|​	shell 无非是另一个程序罢了，它的工作就是为用户执行其他程序。
25|
26|​	将 shell 与操作系统自身分离使得软件开发人员（例如 bash 之父 Brian Fox 以及 bash 当前的开发者和维护者 Chet Ramey）更容易编写出更好的 shell，你也可以在不改动操作系统的前提下编写出新的 shell。
27|
28|​	**为什么是bash？**
29|
30|​	GNU 项目旨在创建一款彻头彻尾的 POSIX 系统，这个系统自然需要一个 POSIX shell，bash 最初正是该项目的一部分。
31|
32|​	学习 bash 准没错。最常见的默认 shell 就是古老的 Bourne shell和 bash，后者基本上兼容前者。任何现代的主流 Unix 或类 Unix 操作系统肯定安装了其一。
33|
34|​	**什么是bash？**
35|
36|​	bash 其实就是一种用于执行命令的语言，你输入的命令（如 ls、cd、grep、cat）从某种意义上来说就是 bash 命令。这些命令有些是 bash 内建的，有些是独立的程序。就目前而言，这种差异并不重要。
37|
38|​	**使用shell引用**
39|
40|~~~shell
41|$ echo A coffee is $5?!
42|A coffee is ?!
43|$ echo "A coffee is $5?!"
44|-bash: !": event not found
45|$ echo 'A coffee is $5?!'
46|A coffee is $5?!
47|~~~
48|
49|第一个例子要对 $5 进行变量扩展，但因为该变量并不存在，所以扩展结果为空。第二个例子要遵循同样的规则，但我们没有看到任何命令输出，这是因为对 ! 进行历史替换时失败了，原因是其不匹配任何历史记录。第三个例子的结果符合我们的预期。
50|
51|**确定是否处于交互模式**
52|
53|~~~shell
54|#!/usr/bin/env bash
55|# 文件名: interactive
56|case "$-" in
57| *i*) # 在交互式shell中运行的代码位于此处
58| ;;
59| *) # 在非交互式shell中运行的代码位于此处
60| ;;
61|esac
62|~~~
63|
64|变量 $- 中保存了一个字符串，其中列出了当前所有的 shell 选
65|
66|项。如果 shell 处于交互模式，则其中会包含 i。

70|**将输出和错误消息发送到不同文件**
71|
72|~~~shell

74|~~~

76|在 1> 和 2> 中，数字表示文件 描述符。1 代表标准输出（STDOUT），2 代表标准错误（STDERR）。起始编号 0 对应标准输入（STDIN）。如果不指定数字，则假定为 STDOUT。

80|首选：
81|
82|~~~shell
83|both >& outfile
84|~~~

89|both &> outfile
90|~~~

157|# 第 4 章 执行命令
158|
159|**运行程序**
160|
161|bash 使用名为 $PATH 的 shell 变量来定位可执行文件。$PATH 变量包含了一个目录列表。各个目录之间以冒号（:）分隔。bash 在这些目录中查找命令行上指定的可执行文件。目录的顺序很重要：bash按照变量中所列出的目录顺序依次查找，选择所找到的第一个同名的可执行文件。
162|
163|~~~shell
164|$ echo $PATH
165|/bin:/usr/bin:/usr/local/bin:.
166|~~~
167|
168|**依次执行多个命令**
169|
170|将命令一个接一个敲进去即可
171|
172|将这些命令写入文件，然后让 bash 执行该文件（简单的 shell 脚本）中的命令。
173|
174|第三种是依次执行每个命令，只需要用分号将其隔开
175|
176|如果只想在上一个程序成功运行的情况下运行下一个程序，并且所有的程序都正确设置了退出码，那么可以用 && 将其隔开：
177|
178|~~~shell
179|long && medium && short
180|~~~
181|
182|**同时执行多个命令**
183|
184|​	在命令末尾添加一个 & 符号，在后台运行该命令
185|
186|~~~shell
187|$ long &
188|[1] 4592
189|$ medium &
190|[2] 4593
191|$ short
192|$
193|~~~
194|
195|或者，更好的做法是在单个命令行中完成所有操作
196|
197|~~~shell
198|$ long & medium & short
199|[1] 4592
200|[2] 4593
201|$
202|~~~
203|
204|​	　如果执行某个命令，但发觉其完成时间比预想的更长，那么可以用 Ctrl-Z 暂停该命令，返回到提示符下。接着输入 bg 来恢复作业，并在后台继续运行。这么做的效果相当于事前在命令尾部加上 & 符号
205|
206|**了解命令是否成功运行**
207|
208|​	如果编写命令或 shell 脚本的程序员遵循既定约定，那么 shell 变量 $? 会在命令失败时被设置为非 0 值。
209|
210|**仅当一个命令运行成功后才执行下一个命令**
211|
212|​	可以用 cd 命令的退出状态（$?）配合 if 语句来实现仅当 cd 命令成功时执行 rm 命令。
213|
214|~~~shell
215|cd mytmp
216|if (( $? == 0 )); then rm * ; fi
217|~~~
218|
219|　以下写法更好，它更清晰地展现并诠释了我们的意图。
220|
221|~~~shell
222|if cd mytmp; then rm * ; fi
223|~~~
224|
225|**&& 运算符**
226|
227|​	在 bash 中使用 && 运算符，根据条件执行命令
228|
229|~~~shell
230|cd mytmp && rm *
231|~~~
232|
233|​	用 && 分隔两个命令，以此告诉 bash 先执行第一个命令，如果该命令成功（退出状态为 0），再执行第二个命令。这非常类似于用 if语句检查第一个命令的退出状态，从而判断是否执行第二个命令，可以写成这个样子：
234|~~~shell
235|cd mytmp
236|if (( $? == 0 )); then rm * ; fi
237|~~~
238|
239|
240|
241|**set -e**
242|
243|要想彻底检查错误，但又不想到处出现 if 语句，可以设置 -e 标记，这样的话，只要脚本中有任何命令（排除在 while 循环和 if语句中，因为其本身就要用到退出状态）出现错误（退出状态为非0），bash 就会退出
244|
245|~~~shell
246|set -e
247|cd mytmp
248|rm *
249|~~~
250|
251|设置了 -e 标记之后，shell 会在命令失败时退出。如果本例中的cd 命令失败，脚本则直接退出，不再执行 rm * 命令。
252|
253|**|| 运算符**
254|
255||| 告诉 bash 如果第一个表达式为真（成功），则不再评估第二个表达式。

257|# 第 5 章 脚本编程基础：shell变量
258|
259|​	bash 脚本中的变量名称通常采用全大写，但这并非强制性的，只是一种常见做法而已。变量不用事先声明，直接使用就行了。变量基本上都是字符串类型，不过有些运算符能够将变量内容视为数字。变量的实际用法如下所示。
260|
261|~~~shell
262|# 使用shell变量的普通脚本
263|# （不过好歹也注释过了）
264|MYVAR="something"
265|echo $MYVAR
266|# 写法类似，但没有引号
267|MY_2ND=anotherone
268|echo $MY_2ND
269|# 这里需要使用引号：
270|MYOTHER="more stuff to echo"
271|echo $MYOTHER
272|~~~
273|
274|​	bash 变量的语法有两处要点，但可能不那么一目了然。首先，赋值语法 *name=value* 看起来相当直观，但 = 两侧不能有任何空白字符。
275|
276|
277|
278|**导出变量**
279|
280|有时你的确想在脚本之间传递信息。在这种情况下，你可以导出变量，使变量的值能够传递给脚本所调用的其他程序。
281|
282|用 env（或 export -p）命令查看那些被导出的、可用于子 shell的变量。
283|
284|**在shell脚本中使用参数**
285|
286|${1}会显示在命令行上被调用时所指定的第一个参数。其他参数的可用形式分别为 ${2}、${3}、${4}、${5} 等。单个数位的数字用不着花括号
287|
288| $*  表示深入的全部参数列表
289|
290|"$@" 得到的不是一个字符串，而是与各个参数对应的带有引号的字符串列表。

296|**条件分支**
297|
298|~~~shell
299|if [ $# -lt 3 ]
300|then
301| printf "%b" "Error. Not enough arguments.\n"
302| printf "%b" "usage: myscript file1 op file2\n"
303| exit 1
304|fi
305|~~~
306|
307|
308|
309|
310|
311|~~~shell
312|if (( $# < 3 ))
313|then
314| printf "%b" "Error. Not enough arguments.\n"
315| printf "%b" "usage: myscript file1 op file2\n"
316| exit 1
317|elif (( $# > 3 ))
318|then
319| printf "%b" "Error. Too many arguments.\n"
320| printf "%b" "usage: myscript file1 op file2\n"
321| exit 2
322|else
323| printf "%b" "Argument count correct. Proceeding...\n"
324|fi
325|~~~
